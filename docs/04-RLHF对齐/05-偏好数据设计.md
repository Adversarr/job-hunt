# 偏好数据设计：DPO/SFT 数据配比与多轮采样策略

## 一句话结论
DPO 数据配比为**单轮 40-60% + 多轮 40-60%**，多轮采样按轮次（早/中/晚 40%/40%/20%）、错误类型（事实/指令/安全 40%/30%/15%）、长度三维控制；第一轮对话需独立构造三元组，System Prompt 视场景保留，Rejected 样本需注入多样性避免奖励黑客。

## 核心定义/公式

### DPO Loss 公式
```python
# DPO Loss: 直接从偏好数据学习，无需训练奖励模型
def dpo_loss(policy_chosen_logp, policy_rejected_logp, 
             ref_chosen_logp, ref_rejected_logp, beta=0.1):
    """
    L_DPO = -log(σ(beta * (log(π(y_c|x)/π_ref(y_c|x)) - log(π(y_r|x)/π_ref(y_r|x)))))
    
    policy_chosen_logp: 策略模型对 chosen 回复的对数概率
    policy_rejected_logp: 策略模型对 rejected 回复的对数概率
    ref_chosen_logp: 参考模型对 chosen 回复的对数概率
    ref_rejected_logp: 参考模型对 rejected 回复的对数概率
    beta: KL 散度控制系数
    """
    # 计算 log-ratio
    chosen_logratios = policy_chosen_logp - ref_chosen_logp
    rejected_logratios = policy_rejected_logp - ref_rejected_logp
    
    # DPO loss
    logits = beta * (chosen_logratios - rejected_logratios)
    loss = -F.logsigmoid(logits).mean()
    
    return loss
```

### 数据配比经验公式
```python
# 单轮 vs 多轮配比决策因素
def compute_data_ratio(coverage_score, difficulty_score, noise_score, cost_score):
    """
    coverage_score: 场景覆盖率 (0-1)
    difficulty_score: 任务难度 (0-1，越高越难)
    noise_score: 数据噪声水平 (0-1，越高越噪)
    cost_score: 标注成本 (0-1，越高越贵)
    """
    # 单轮数据权重：噪声低、成本低 → 权重高
    single_turn_weight = (1 - noise_score) * (1 - cost_score) * 0.5
    
    # 多轮数据权重：覆盖广、难度高 → 权重高
    multi_turn_weight = coverage_score * difficulty_score * 0.5
    
    # 归一化
    total = single_turn_weight + multi_turn_weight
    return {
        "single_turn_ratio": single_turn_weight / total,
        "multi_turn_ratio": multi_turn_weight / total
    }
```

## 为什么（3 个因果链）

### 1. 为什么需要平衡单轮和多轮 DPO 数据？

**现象**：只用单轮数据训练的模型在多轮对话场景表现差，只用多轮数据又导致标注成本过高且噪声大。

**根因**：
- **单轮数据优势**：标注简单、质量可控、成本低（$1-2/对 vs 多轮 $5-10/对）、噪声低（独立判断，无上下文依赖）
- **单轮数据劣势**：场景覆盖不足（无追问、无澄清、无上下文切换）、难度偏低（无多步推理）
- **多轮数据优势**：真实场景覆盖（指令修正、信息补充、任务拆解）、难度多样（上下文依赖、状态跟踪）
- **多轮数据劣势**：标注复杂（需理解完整对话链）、质量难控（局部错误传播）、成本高

**结果**：单轮数据保证基础能力和标注质量，多轮数据覆盖真实场景和复杂任务，两者互补达到最佳效果。推荐配比 **单轮 40-60% + 多轮 40-60%**。

### 2. 为什么多轮 DPO 样本采样需按轮次、错误类型、长度三个维度？

**现象**：随机采样多轮数据导致模型在特定场景（长对话、特定错误类型）表现不稳定。

**根因**：
- **按轮次采样**：
  - 早轮（1-2 轮）：测试指令理解、意图识别能力
  - 中轮（3-5 轮）：测试上下文跟踪、信息整合能力
  - 末轮（>5 轮）：测试任务完成、总结归纳能力
  - 实际场景中不同轮次频率不同，需按分布采样（早轮 40% + 中轮 40% + 末轮 20%）

- **按错误类型采样**：
  - 事实错误（幻觉、编造）：最致命，权重最高
  - 指令偏离（未遵循用户要求）：高频错误
  - 安全问题（有害内容）：关键但低频
  - 格式错误（输出格式不规范）：低权重但需覆盖
  - 按错误类型分层采样确保模型全面学习

- **按对话长度分桶**：
  - 短上下文（<512 tokens）：基础场景，占比 30%
  - 中上下文（512-2048 tokens）：主流场景，占比 50%
  - 长上下文（>2048 tokens）：边缘但关键场景，占比 20%
  - 长对话暴露 KV cache、位置编码、注意力衰减问题

**结果**：三维采样确保模型在轮次分布、错误类型、对话长度三个维度都有充分训练，避免某类场景性能崩塌。

### 3. 为什么"第一轮对话做 DPO"需特别处理 prompt 和曝光偏差？

**现象**：直接用第一轮对话做 DPO，模型容易过拟合特定 prompt 格式，泛化能力差。

**根因**：
- **Prompt 定义问题**：
  - 第一轮对话的 prompt 通常是独立的用户问题，缺少对话上下文
  - 直接使用导致模型只在"冷启动"场景表现好，多轮场景弱
  - 需要注入多样化 prompt 模板（不同表述、不同指令风格）

- **System Prompt 处理**：
  - 是否保留 system prompt 取决于业务场景
  - 助手类任务（对话、问答）：保留 system prompt，定义角色和能力边界
  - 指令类任务（翻译、摘要）：可省略 system prompt，减少依赖
  - 关键是统一处理策略，避免训练/推理不一致

- **曝光偏差（Exposure Bias）**：
  - 训练时模型只看到高质量 chosen 回复，推理时可能生成低质量内容
  - 多轮对话中，后续轮次的 prompt 包含模型自身生成的回复
  - 如果训练数据中 chosen 回复风格过于一致，模型难以处理分布外的自身生成内容

**结果**：需构造多样化 prompt 模板、选择性保留 system prompt、引入 rejected 样本的多样性（不同错误类型），缓解曝光偏差。

```python
def sample_by_turn(dialogue_pool, turn_weights=[0.4, 0.4, 0.2]):
    """
    按对话轮次权重采样
    turn_weights: [早轮权重, 中轮权重, 末轮权重]
    """
    # 按轮次分桶
    early_turn_dialogues = [d for d in dialogue_pool if d["turn"] <= 2]
    mid_turn_dialogues = [d for d in dialogue_pool if 2 < d["turn"] <= 5]
    late_turn_dialogues = [d for d in dialogue_pool if d["turn"] > 5]
    
    # 加权采样
    n_samples = len(dialogue_pool)
    early_samples = random.sample(
        early_turn_dialogues, 
        int(n_samples * turn_weights[0])
    )
    mid_samples = random.sample(
        mid_turn_dialogues,
        int(n_samples * turn_weights[1])
    )
    late_samples = random.sample(
        late_turn_dialogues,
        int(n_samples * turn_weights[2])
    )
    
    return early_samples + mid_samples + late_samples
```

**配置建议**：
- 早轮（1-2 轮）：40% 权重，测试基础能力

```python
def sample_by_error_type(dialogue_pool, error_weights=None):
    """
    按错误类型分层采样
    """
    if error_weights is None:
        error_weights = {
            "factual_error": 0.4,      # 事实错误（最致命）
            "instruction_deviation": 0.3,  # 指令偏离
            "safety_issue": 0.15,      # 安全问题
            "format_error": 0.15       # 格式错误
        }
    
    # 按错误类型分类
    error_buckets = defaultdict(list)
    for dialogue in dialogue_pool:
        error_type = classify_error(dialogue["rejected"])
        error_buckets[error_type].append(dialogue)
    
    # 分层采样
    sampled_dialogues = []
    for error_type, weight in error_weights.items():
        n_samples = int(len(dialogue_pool) * weight)
        sampled_dialogues.extend(
            random.sample(error_buckets[error_type], min(n_samples, len(error_buckets[error_type])))
        )
    
    return sampled_dialogues

def classify_error(rejected_response):
    """错误类型分类器"""
    if contains_hallucination(rejected_response):
        return "factual_error"
    elif not follow_instruction(rejected_response):
        return "instruction_deviation"
    elif contains_harmful_content(rejected_response):
        return "safety_issue"
    else:
        return "format_error"
```

- 事实错误：40% 权重（最致命，需重点学习）
- 指令偏离：30% 权重（高频错误）
- 安全问题：15% 权重（低频但关键）
- 格式错误：15% 权重（基础能力）

```python
def sample_by_length(dialogue_pool, length_bins=None):
    """
    按对话长度分桶采样
    """
    if length_bins is None:
        length_bins = [
            (0, 512, 0.3),       # 短上下文：30%
            (512, 2048, 0.5),    # 中上下文：50%
            (2048, float('inf'), 0.2)  # 长上下文：20%
        ]
    
    # 按长度分桶
    length_buckets = defaultdict(list)
    for dialogue in dialogue_pool:
        length = count_tokens(dialogue["prompt"])
        for min_len, max_len, _ in length_bins:
            if min_len <= length < max_len:
                length_buckets[(min_len, max_len)].append(dialogue)
                break
    
    # 分桶采样
    sampled_dialogues = []
    for (min_len, max_len), weight in [(k, v[2]) for k, v in length_bins]:
        bucket = length_buckets[(min_len, max_len)]
        n_samples = int(len(dialogue_pool) * weight)
        sampled_dialogues.extend(
            random.sample(bucket, min(n_samples, len(bucket)))
        )
    
    return sampled_dialogues
```

**配置建议**：
- 短上下文（<512）：30%，基础场景
- 中上下文（512-2048）：50%，主流场景
- 长上下文（>2048）：20%，边缘场景但关键

```python
    构造第一轮对话 DPO 数据
    
    raw_dialogues: 原始多轮对话数据
    prompt_templates: 多样化 prompt 模板
    keep_system: 是否保留 system prompt
    """
    dpo_data = []
    
    for dialogue in raw_dialogues:
        # 提取第一轮对话
        first_turn = extract_first_turn(dialogue)
        
        # 构造 prompt
        if keep_system and dialogue.get("system"):
            prompt = [
                {"role": "system", "content": dialogue["system"]},
                {"role": "user", "content": first_turn["user"]}
            ]
        else:
            # 应用多样化 prompt 模板
            template = random.choice(prompt_templates)
            prompt = template.format(question=first_turn["user"])
        
        # 构造 chosen/rejected
        chosen = first_turn["chosen"]
        rejected = first_turn["rejected"]
        
        # 控制曝光偏差：注入 rejected 多样性
        if random.random() < 0.3:  # 30% 概率使用自动生成的 rejected
            auto_rejected = generate_rejected_response(prompt, error_type="random")
            rejected = auto_rejected
        
        dpo_data.append({
            "prompt": prompt,
            "chosen": chosen,
            "rejected": rejected
        })
    
    return dpo_data
```

```python
def clean_dpo_data(dpo_pairs):
    """DPO 数据清洗流水线"""
    cleaned_data = []
    
    for pair in dpo_pairs:
        # 1. 去重
        if is_duplicate(pair, cleaned_data):
            continue
        
        # 2. 长度过滤
        chosen_len = len(tokenizer.encode(pair["chosen"]))
        rejected_len = len(tokenizer.encode(pair["rejected"]))
        if chosen_len < 20 or chosen_len > 4096:
            continue
        if rejected_len < 10:  # rejected 不能过短
            continue
        
        # 3. 质量差异过滤
        chosen_score = score_response(pair["chosen"], pair["prompt"])
        rejected_score = score_response(pair["rejected"], pair["prompt"])
        if chosen_score - rejected_score < 0.2:  # 差异过小无效
            continue
        
        # 4. 格式一致性检查
        if not validate_format(pair["chosen"]):
            continue
        
        # 5. 去除有害内容（可选，取决于业务需求）
        if contains_harmful_content(pair["chosen"]):
            continue
        
        cleaned_data.append(pair)
    
    return cleaned_data

def is_duplicate(pair, existing_pairs, similarity_threshold=0.9):
    """基于语义相似度去重"""
    pair_embedding = get_embedding(pair["prompt"])
    for existing in existing_pairs:
        existing_embedding = get_embedding(existing["prompt"])
        similarity = cosine_similarity(pair_embedding, existing_embedding)
        if similarity > similarity_threshold:
            return True
    return False
```

### 完整代码示例：DPO 数据构造 Pipeline

```python
import random
from collections import defaultdict
from typing import List, Dict, Optional
import torch
import torch.nn.functional as F

class DPODataConstructor:
    def __init__(
        self,
        single_turn_ratio=0.5,
        multi_turn_ratio=0.5,
        turn_weights=[0.4, 0.4, 0.2],
        error_weights={"factual": 0.4, "instruction": 0.3, "safety": 0.15, "format": 0.15},
        length_bins=[(0, 512, 0.3), (512, 2048, 0.5), (2048, float('inf'), 0.2)]
    ):
        self.single_turn_ratio = single_turn_ratio
        self.multi_turn_ratio = multi_turn_ratio
        self.turn_weights = turn_weights
        self.error_weights = error_weights
        self.length_bins = length_bins
    
    def construct_dataset(
        self,
        single_turn_pool: List[Dict],
        multi_turn_pool: List[Dict],
        n_samples: int = 10000
    ) -> List[Dict]:
        """
        构造完整的 DPO 数据集
        
        Args:
            single_turn_pool: 单轮对话池
            multi_turn_pool: 多轮对话池
            n_samples: 目标样本数
        """
        # 1. 计算样本数
        n_single = int(n_samples * self.single_turn_ratio)
        n_multi = n_samples - n_single
        
        # 2. 单轮数据采样
        single_turn_data = random.sample(single_turn_pool, min(n_single, len(single_turn_pool)))
        
        # 3. 多轮数据采样（三维采样）
        multi_turn_data = self._sample_multi_turn(multi_turn_pool, n_multi)
        
        # 4. 合并并打乱
        dataset = single_turn_data + multi_turn_data
        random.shuffle(dataset)
        
        # 5. 质量控制
        dataset = self._quality_control(dataset)
        
        return dataset
    
    def _sample_multi_turn(self, pool: List[Dict], n_samples: int) -> List[Dict]:
        """三维采样：轮次 × 错误类型 × 长度"""
        
        # 按轮次采样
        turn_sampled = self._sample_by_turn(pool, n_samples)
        
        # 按错误类型采样
        error_sampled = self._sample_by_error_type(turn_sampled, n_samples)
        
        # 按长度分桶
        length_sampled = self._sample_by_length(error_sampled, n_samples)
        
        return length_sampled[:n_samples]
    
    def _sample_by_turn(self, pool: List[Dict], n_samples: int) -> List[Dict]:
        """按轮次权重采样"""
        early = [d for d in pool if d.get("turn", 1) <= 2]
        mid = [d for d in pool if 2 < d.get("turn", 1) <= 5]
        late = [d for d in pool if d.get("turn", 1) > 5]
        
        n_early = int(n_samples * self.turn_weights[0])
        n_mid = int(n_samples * self.turn_weights[1])
        n_late = n_samples - n_early - n_mid
        
        sampled = []
        sampled.extend(random.sample(early, min(n_early, len(early))))
        sampled.extend(random.sample(mid, min(n_mid, len(mid))))
        sampled.extend(random.sample(late, min(n_late, len(late))))
        
        return sampled
    
    def _sample_by_error_type(self, pool: List[Dict], n_samples: int) -> List[Dict]:
        """按错误类型分层采样"""
        error_buckets = defaultdict(list)
        for d in pool:
            error_type = d.get("error_type", "instruction")
            error_buckets[error_type].append(d)
        
        sampled = []
        for error_type, weight in self.error_weights.items():
            n = int(n_samples * weight)
            bucket = error_buckets[error_type]
            sampled.extend(random.sample(bucket, min(n, len(bucket))))
        
        return sampled
    
    def _sample_by_length(self, pool: List[Dict], n_samples: int) -> List[Dict]:
        """按对话长度分桶采样"""
        length_buckets = defaultdict(list)
        for d in pool:
            length = len(d.get("prompt", ""))
            for min_len, max_len, _ in self.length_bins:
                if min_len <= length < max_len:
                    length_buckets[(min_len, max_len)].append(d)
                    break
        
        sampled = []
        for (min_len, max_len), (_, _, weight) in zip(
            [(b[0], b[1]) for b in self.length_bins],
            self.length_bins
        ):
            n = int(n_samples * weight)
            bucket = length_buckets[(min_len, max_len)]
            sampled.extend(random.sample(bucket, min(n, len(bucket))))
        
        return sampled
    
    def _quality_control(self, data: List[Dict]) -> List[Dict]:
        """数据质量控制"""
        cleaned = []
        for pair in data:
            # 长度检查
            chosen_len = len(pair.get("chosen", ""))
            if chosen_len < 20 or chosen_len > 4096:
                continue
            
            # 分数差异检查（简化版）
            # 实际应用中可用 reward model 打分
            if pair.get("chosen_score", 1.0) - pair.get("rejected_score", 0.0) < 0.2:
                continue
            
            # 去重（简化版）
            if pair in cleaned:
                continue
            
            cleaned.append(pair)
        
        return cleaned


# 使用示例

        multi_turn_ratio=0.5
```

## 权衡分析

### 单轮 vs 多轮数据配比

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| 单轮数据 100% | 标注成本低（$1-2/对）、噪声低、训练稳定 | 多轮场景表现差、无上下文跟踪能力 | 简单指令任务（翻译、摘要） |
| 多轮数据 100% | 真实场景覆盖好、难度多样、上下文能力强 | 标注成本高（$5-10/对）、噪声大、训练不稳定 | 长对话助手（客服、教育） |
| 单轮 50% + 多轮 50%（推荐） | 平衡成本与效果、基础能力+场景覆盖 | 需精细采样策略、工程复杂度高 | 通用对话助手 |
| 单轮 40% + 多轮 60% | 真实场景覆盖更优、长对话能力强 | 成本略高、需更多多轮数据池 | 业务导向的对话助手 |
| 单轮 60% + 多轮 40% | 成本更低、训练更稳定、基础能力扎实 | 复杂场景覆盖不足 | 任务型指令（翻译、代码） |

### System Prompt 处理策略

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| 保留 System Prompt | 定义角色边界、风格一致、推理可控 | 依赖 prompt engineering、泛化可能受限 | 对话助手、角色扮演 |
| 省略 System Prompt | 减少依赖、泛化性强、数据构造简单 | 风格不一致、角色不明确 | 任务型指令（翻译、摘要） |
| 多样化 System Prompt | 角色多样性强、泛化好 | 数据构造复杂、需更多标注 | 通用模型训练 |

### Rejected 样本生成策略

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| 人工标注 rejected | 质量高、错误真实、学习效果好 | 成本高、速度慢 | 小规模高质量数据集 |
| 模型自动生成 rejected | 成本低、速度快、可扩展 | 错误模式单一、可能不真实 | 大规模数据集、数据增强 |
| 混合策略（30% 自动 + 70% 人工） | 平衡成本与质量 | 需精细控制比例 | 工业界主流方案 |

### 多轮采样策略对比

| 策略 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| 随机采样 | 实现简单、无偏 | 某些场景样本不足、性能不稳定 | 数据充足且分布均匀 |
| 按轮次采样 | 轮次覆盖均匀、能力全面 | 需标注轮次信息、工程复杂 | 多轮对话为主 |
| 按错误类型采样 | 错误学习全面、鲁棒性强 | 需错误分类器、标注成本高 | 对安全性要求高 |
| 三维联合采样（推荐） | 全面覆盖、性能最优 | 实现复杂、需多种标注 | 工业界高质量数据集 |

### 2. Q: DPO 和 SFT 数据如何配比？先做 DPO 还是 SFT？

**A**: 
- **顺序**：先 SFT 后 DPO（标准流程）
  - SFT：建立基础能力（指令遵循、格式规范），DPO 在此基础上优化偏好
  - 先 DPO 后 SFT 会破坏 DPO 学到的偏好，因为 SFT 只学 chosen，会"遗忘" rejected 的区分

- **数据配比**：
  - SFT 数据量：10k-100k（建立基础能力）
  - DPO 数据量：SFT 的 10-30%（1k-30k，精细优化）
  - 比例：SFT:DPO ≈ 3:1 到 10:1

- **例外情况**：
  - 如果模型已有强基础能力（预训练充分），可只用 DPO
  - 如果 SFT 数据质量差，可减少 SFT 比例，增加 DPO

### 3. Q: 多轮 DPO 数据如何构造？prompt/chosen/rejected 如何定义？

**A**:
```python
# 多轮 DPO 数据格式
{
    "prompt": [
        {"role": "system", "content": "你是助手"},
        {"role": "user", "content": "问题1"},
        {"role": "assistant", "content": "回答1"},
        {"role": "user", "content": "问题2"}  # 到当前轮次
    ],
    "chosen": "高质量回复2",   # 当前轮次的高质量回答
    "rejected": "低质量回复2"  # 当前轮次的低质量回答
}

# 关键点：
# 1. prompt 包含完整对话历史（到当前轮次）
# 2. chosen/rejected 只对当前轮次的回复排序
# 3. 不修改历史轮次的内容（只学当前轮次的偏好）
```

### 4. Q: 如何处理数据质量不好但不想用人工标注？

**A**（百度面试题）：
1. **自动化质量评估**：
   - 用 reward model 或 GPT-4 打分，过滤低分样本
   - 用困惑度过滤异常样本（过高/过低）

2. **数据清洗策略**：
   - 去重：基于 embedding 相似度去重
   - 长度过滤：过短（<20 tokens）或过长（>4096 tokens）剔除
   - 格式检查：验证 JSON/Markdown 格式是否正确

3. **合成高质量数据**：
   - 用 GPT-4 重新生成 chosen 回复
   - 用弱模型生成 rejected（注入特定错误）
   - Prompt 变体增强（改写、翻译）

4. **主动学习**：
   - 先用少量人工标注训练 reward model
   - 用 RM 自动标注剩余数据
   - 对不确定样本人工复审

### 5. Q: 如何控制曝光偏差（Exposure Bias）？

**A**:
1. **问题根源**：
   - 训练时只看高质量 chosen，推理时可能生成低质量内容
   - 多轮对话中，后续 prompt 包含模型自身生成内容，训练时未见过

2. **解决方案**：
   - **Scheduled Sampling**：训练时以一定概率用模型自身生成的内容构造 prompt（而非全用 chosen）
   - **Rejected 多样性**：30% 使用自动生成的 rejected，引入不同错误类型
   - **数据增强**：用模型生成多个候选回复，构造不同偏好对

3. **代码示例**：
```python
def control_exposure_bias(dialogue, model, alpha=0.3):
    """
    alpha: 用模型生成内容替换 chosen 的概率
    """
    if random.random() < alpha:
        # 用模型生成的内容替换历史轮次
        synthetic_history = model.generate(
            dialogue["prompt"][:-1],  # 去除最后一轮
            temperature=0.8
        )
        dialogue["prompt"] = synthetic_history + dialogue["prompt"][-1:]
    
    return dialogue
```

### 6. Q: 多轮样本采样中，如何平衡早轮/中轮/末轮？（字节真题）

**A**:
- **早轮（1-2 轮）**：测试指令理解、意图识别，权重 30-40%
- **中轮（3-5 轮）**：测试上下文跟踪、信息整合，权重 40-50%
- **末轮（>5 轮）**：测试任务完成、长对话能力，权重 10-20%

**调整原则**：
- 如果业务场景以短对话为主（<3 轮），增加早轮权重至 50%
- 如果需要强长对话能力（客服、教育），增加末轮权重至 30%
- 根据线上真实对话轮次分布调整权重

### 7. Q: DPO 数据量不足怎么办？（百度真题）

**A**（百度面试题）：
1. **数据增强**：
   - Prompt 变体：改写、翻译、简化/扩展
   - Rejected 生成：用模型自动生成不同错误类型的 rejected
   - 数量提升：每条原始数据生成 3-5 条变体

2. **合成数据**：
   - 用 GPT-4/Claude 生成偏好对
   - 构造对比样本：正确 vs 错误、详细 vs 简略、安全 vs 有害

3. **迁移学习**：
   - 先用开源 DPO 数据（如 UltraFeedback）训练
   - 再用少量领域数据 fine-tune

4. **主动学习**：
   - 用当前模型生成候选回复
   - 人工标注最难区分的样本（信息量最大）

### 8. Q: 如何评估 DPO 数据的质量？

**A**:
1. **自动化指标**：
   - **Chosen-Rejected 分数差异**：用 reward model 打分，差异 < 0.2 视为无效
   - **长度差异**：chosen 不能显著长于 rejected（避免长度偏好）
   - **重复率**：去重后剩余比例应 > 80%

2. **人工抽检**：
   - 随机抽取 100-200 条，人工判断 chosen 是否真的优于 rejected
   - 合格率应 > 90%

3. **训练监控**：
   - 观察 DPO loss 收敛曲线，如果震荡或不下降，数据质量可能有问题
   - 观察 chosen/rejected reward 变化，如果两者都下降，参考"奖励黑客"排查

4. **下游评估**：
   - 在 held-out 测试集上评估模型性能
   - 对比 SFT-only vs SFT+DPO 的提升幅度

### 9. Q: 数据设计中如何避免奖励黑客（Reward Hacking）？

**A**:
**奖励黑客定义**：模型学会欺骗奖励模型，chosen/rejected reward 都下降，但模型输出实际变差。

**数据层面的防范策略**：

1. **Rejected 样本多样性**：
   ```python
   # 避免所有 rejected 都是同一类错误
   error_type_distribution = {
       "factual_error": 0.4,      # 事实错误
       "instruction_deviation": 0.3,  # 指令偏离
       "safety_issue": 0.15,      # 安全问题
       "format_error": 0.15       # 格式错误
   }
   # 如果只学习"事实错误"，模型可能学会"只要不犯错就行"，但格式混乱
   ```

2. **Chosen 样本真实性**：
   - 避免 chosen 过于完美（如全是 GPT-4 生成）
   - 掺杂 20-30% 真实高质量人工回复
   - 避免 chosen 具有可被模型学习的"捷径"模式（如固定开头、特定格式词）

3. **长度平衡**：
   ```python
   # 错误：chosen 总是更长，模型学会"写长就能得分"
   if len(chosen) > len(rejected) * 1.5:
       # 过滤或平衡
       pass
   
   # 正确：长度分布接近
   ```

4. **交叉验证机制**：
   - 用不同版本的 reward model 标注数据
   - 如果不同 RM 对同一对的排序不一致，标记为歧义样本，人工复审
   - 避免模型学习特定 RM 的偏差

5. **对比学习策略**：
   - 对每个 prompt，提供多个 chosen/rejected 对（而非单一对）
   - 模型学习"相对优劣"而非"绝对高分"

**监控指标**：
- 训练中 chosen reward ↑，rejected reward ↓：正常
- 两者都 ↓：疑似奖励黑客，检查数据质量
- 两者都 ↑：可能是数据标注错误

### 10. Q: 单轮和多轮数据配比怎么确定？有没有具体决策依据？（字节真题）

**A**:
**决策框架**：

```python
def decide_single_multi_ratio(business_scenario, data_constraints, model_goal):
    """
    business_scenario: 业务场景类型
    data_constraints: 数据约束（成本、时间、人力）
    model_goal: 模型目标（对话助手 / 任务执行）
    """
    
    # 1. 业务场景判断
    if business_scenario == "多轮对话助手":
        # 客服、教育、医疗：强依赖上下文
        base_ratio = {"single": 0.4, "multi": 0.6}
    elif business_scenario == "任务执行":
        # 翻译、摘要、代码：主要是单轮指令
        base_ratio = {"single": 0.7, "multi": 0.3}
    else:
        # 通用对话
        base_ratio = {"single": 0.5, "multi": 0.5}
    
    # 2. 数据约束调整
    if data_constraints["labeling_budget"] < "limited":
        # 预算有限，增加单轮比例（成本低）
        base_ratio["single"] += 0.1
        base_ratio["multi"] -= 0.1
    
    if data_constraints["timeline"] == "urgent":
        # 时间紧急，增加单轮比例（标注快）
        base_ratio["single"] += 0.15
        base_ratio["multi"] -= 0.15
    
    # 3. 模型目标调整
    if model_goal == "general_capability":
        # 通用能力：保持平衡
        pass
    elif model_goal == "long_context":
        # 长对话能力优先：增加多轮
        base_ratio["multi"] += 0.1
        base_ratio["single"] -= 0.1
    
    # 归一化
    total = base_ratio["single"] + base_ratio["multi"]
    return {
        "single_turn": base_ratio["single"] / total,
        "multi_turn": base_ratio["multi"] / total
    }

# 实际案例
# 字节对话助手：单轮 40% + 多轮 60%
# 阿里翻译任务：单轮 70% + 多轮 30%
# 腾讯通用模型：单轮 50% + 多轮 50%
```

**关键决策因素**：
1. **线上对话轮次分布**：如果 80% 对话 < 3 轮，单轮占比可到 60%
2. **标注成本**：多轮标注成本是单轮的 3-5 倍
3. **数据池大小**：如果多轮数据不足，先用单轮凑数
4. **模型基础能力**：SFT 阶段已有强基础，DPO 可更多用多轮精细优化

**快速决策表**：
| 场景 | 单轮占比 | 多轮占比 | 理由 |
|------|---------|---------|------|
| 客服对话助手 | 40% | 60% | 强依赖上下文，多轮为主 |
| 任务型指令 | 70% | 30% | 单轮指令执行为主 |
| 通用模型 | 50% | 50% | 平衡基础能力与场景覆盖 |
| 预算有限 | 60% | 40% | 单轮成本低，保证数据量 |
| 长对话优先 | 30% | 70% | 特化长对话能力 |

## 常见错误（6 个）

### 1. 错误：单轮/多轮配比极端化

**错误理解**：认为多轮数据更真实，只用多轮数据训练。

**问题**：
- 多轮数据标注成本高，数据量不足
- 多轮数据噪声大，局部错误传播
- 模型基础能力不扎实（单轮场景弱）

**正确做法**：
- 单轮 40-60% + 多轮 40-60%，平衡成本与效果
- 单轮数据建立基础能力，多轮数据覆盖真实场景
- 根据业务场景调整比例（对话助手多轮占比更高）

### 2. 错误：忽略 chosen/rejected 的质量差异

**错误做法**：
```python
# 错误：chosen 和 rejected 质量差异过小
{
    "chosen": "这是一个很好的回答，包含了关键信息。",
    "rejected": "这是一个不错的回答，包含了重要信息。"
}
# 差异仅为"很好 vs 不错"，模型难以学习
```

**正确做法**：
```python
# 正确：chosen 明显优于 rejected
{
    "chosen": "Transformer 的核心是自注意力机制，允许模型在处理序列时捕获全局依赖关系。具体来说，每个 token 可以直接关注到序列中所有其他 token...",
    "rejected": "Transformer 是一种神经网络结构。"  # 过于简略
}

# 或引入明确错误
{
    "chosen": "中国的首都是北京。",
    "rejected": "中国的首都是上海。"  # 事实错误
}
```

**质量控制**：
- 用 reward model 打分，分数差异应 > 0.2
- 人工抽检，确保 chosen 明显优于 rejected

### 3. 错误：多轮 DPO 数据构造时修改历史轮次

**错误做法**：
```python
# 错误：修改历史轮次的内容
{
    "prompt": [
        {"role": "user", "content": "问题1"},
        {"role": "assistant", "content": "修改后的回答1"},  # 修改了历史
        {"role": "user", "content": "问题2"}
    ],
    "chosen": "回答2",
    "rejected": "低质量回答2"
}
```

**问题**：
- 多轮 DPO 只应对当前轮次的回复排序，不应修改历史
- 修改历史会导致对话不一致，模型学习混乱

**正确做法**：
```python
# 正确：保留完整历史，只对当前轮次排序
{
    "prompt": [
        {"role": "user", "content": "问题1"},
        {"role": "assistant", "content": "原始回答1"},  # 保留原始内容
        {"role": "user", "content": "问题2"}
    ],
    "chosen": "高质量回答2",   # 只对当前轮次排序
    "rejected": "低质量回答2"
}
```

### 4. 错误：忽略曝光偏差

**错误理解**：训练数据全是高质量 chosen，推理时模型就能生成高质量内容。

**问题**：
- 训练时只看到高质量回复，推理时生成低质量内容后模型不知道如何处理
- 多轮对话中，后续 prompt 包含模型自身生成内容，训练时未见过这种分布

**正确做法**：
1. **Scheduled Sampling**：30% 概率用模型生成的内容替换 chosen
2. **Rejected 多样性**：引入不同错误类型的 rejected
3. **数据增强**：用模型生成多个候选，构造不同偏好对

```python
# 缓解曝光偏差的代码
def mitigate_exposure_bias(prompt, model, alpha=0.3):
    if random.random() < alpha:
        # 用模型生成的内容替换历史
        generated_history = model.generate(prompt[:-1], temperature=0.8)
        prompt = generated_history + [prompt[-1]]
    return prompt
```

### 5. 错误：DPO 数据量与 SFT 不匹配

**错误做法**：
```python
# 错误：DPO 数据量远大于 SFT
SFT_data = 10k
DPO_data = 100k  # DPO 数据量是 SFT 的 10 倍
```

**问题**：
- DPO 是精细优化，数据量过大可能导致过拟合
- DPO 依赖 SFT 建立的基础能力，数据量不匹配会破坏基础

**正确做法**：
```python
# 正确：DPO 数据量为 SFT 的 10-30%
SFT_data = 50k
DPO_data = 10k  # 20% 的 SFT 数据量

# 推荐比例
# SFT:DPO = 3:1 到 10:1
```

### 6. 错误：数据质量控制不足，导致奖励黑客

**错误做法**：
```python
# 错误：所有 rejected 都是同一类错误
for sample in dataset:
    sample["rejected"] = generate_short_response(sample["prompt"])
    # 所有 rejected 都是"过短"，模型学会"写长就能得分"
```

**问题**：
- Rejected 样本类型单一，模型学到"捷径"而非真实质量差异
- 训练时 chosen/rejected reward 都下降，但模型输出变差（奖励黑客）

**正确做法**：
```python
# 正确：rejected 样本需注入多样性
error_types = ["factual_error", "instruction_deviation", 
               "safety_issue", "format_error", "too_short", "too_long"]

for sample in dataset:
    error_type = random.choice(error_types)
    sample["rejected"] = generate_rejected_with_error(
        sample["prompt"], 
        error_type=error_type
    )

# 监控指标
# 正常：chosen reward ↑，rejected reward ↓
# 异常：两者都 ↓ → 奖励黑客，需检查数据质量
```

**关键检查点**：
- Rejected 样本是否覆盖多种错误类型
- Chosen 样本是否存在可学习的"捷径"模式
- 训练过程中 reward 变化是否符合预期
