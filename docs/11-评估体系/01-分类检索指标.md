# 分类/检索指标

## 一句话结论

样本不均衡时，**accuracy 会严重失真，应使用 precision/recall/F1，并根据业务场景选择 micro 或 macro 聚合方式**；核心在于明确"你更关心哪一类"（少数类还是多数类），选择对应指标。

---

## 核心定义/公式

### 基础指标（二分类）

```python
# 混淆矩阵
TP (True Positive)   = 预测为正且实际为正
TN (True Negative)   = 预测为负且实际为负
FP (False Positive)  = 预测为正但实际为负 (Type I Error)
FN (False Negative)  = 预测为负但实际为正 (Type II Error)
```

### 四大核心指标

```python
# 1. Accuracy - 整体准确率
Accuracy = (TP + TN) / (TP + TN + FP + FN)

# 2. Precision - 精确率（预测为正中真正为正的比例）
Precision = TP / (TP + FP)

# 3. Recall - 召回率（实际为正中被正确预测的比例）
Recall = TP / (TP + FN)  # 又称 Sensitivity, TPR

# 4. F1-Score - 精确率和召回率的调和平均
F1 = 2 * Precision * Recall / (Precision + Recall)

# 补充指标
Specificity = TN / (TN + FP)  # 特异性（负类召回率）
FPR = FP / (FP + TN)  # 假阳率（False Positive Rate）
```

### 多分类聚合方式

```python
# Micro-F1: 全局聚合（所有类别 TP/FP/FN 汇总后计算）
Micro_Precision = sum(TP_i) / sum(TP_i + FP_i)  # i 为类别索引
Micro_Recall = sum(TP_i) / sum(TP_i + FN_i)
Micro_F1 = 2 * Micro_Precision * Micro_Recall / (Micro_Precision + Micro_Recall)

# Macro-F1: 类别独立计算后平均
Macro_Precision = mean(Precision_i)  # 每个类别的 precision 平均
Macro_Recall = mean(Recall_i)
Macro_F1 = mean(F1_i)  # 或 2 * Macro_Precision * Macro_Recall / (Macro_Precision + Macro_Recall)

# Weighted-F1: 按样本量加权平均
Weighted_F1 = sum(F1_i * n_i) / sum(n_i)  # n_i 为第 i 类样本数
```

### ROC-AUC

```python
# ROC 曲线：TPR (Recall) vs FPR 在不同阈值下的轨迹
# AUC: ROC 曲线下面积，衡量分类器整体排序能力

# 计算 AUC（O(n log n)）
from sklearn.metrics import roc_auc_score
auc = roc_auc_score(y_true, y_score)  # y_score 为预测概率

# AUC 直觉：随机抽取一个正样本和一个负样本，
# 模型给正样本更高分数的概率
```

## 为什么（2-3 个因果链）

### 1. 为什么 Accuracy 在样本不均衡时失真？

**现象** → **根因** → **结果**

- **现象**：99% 负样本 + 1% 正样本，模型预测全为负，accuracy = 99%
- **根因**：Accuracy 是**全局视角**指标，被多数类主导；少数类的错误被淹没在多数类的正确中
- **结果**：
  - 模型"看起来很好"但实际对业务毫无价值（正类预测全错）
  - 无法发现模型对少数类的欠拟合
  - 给出错误的优化信号（继续优化全局准确率 → 多数类过拟合）

**数学解释**：
```
∂Accuracy/∂TP = 1/N
∂Accuracy/∂TN = 1/N  
∂Accuracy/∂FP = -1/N
∂Accuracy/∂FN = -1/N

当 N_neg >> N_pos 时，TN 的贡献远大于 TP
```

### 2. 为什么需要 Precision 和 Recall 两个指标？

**原因**：单指标无法同时反映"准确性"和"覆盖性"

- **Precision**：**从预测角度看正类的质量**
  - 高 Precision → 预测为正的样本可信度高
  - 业务含义：推荐系统的"推荐准确性"、风控的"报警质量"
  - 代价：可能漏掉很多正样本（保守预测）

- **Recall**：**从实际角度看正类的覆盖**
  - 高 Recall → 实际正样本被找到的比例高
  - 业务含义：召回系统的"覆盖完整性"、医疗诊断的"漏诊率"
  - 代价：可能误判很多负样本（激进预测）

**冲突关系**：
- 提高阈值 → Precision↑ Recall↓（更保守）
- 降低阈值 → Precision↓ Recall↑（更激进）
- **P-R 权衡是业务决策，不是纯技术问题**

### 3. 为什么 F1 用调和平均而非算术平均？

**根因**：调和平均对极端值更敏感，惩罚"短板"

```python
# 算术平均：P=0, R=100 → 平均 = 50（看起来还行）
Arithmetic = (P + R) / 2

# 调和平均：P=0, R=100 → F1 = 0（正确反映"模型无用"）
Harmonic = 2 * P * R / (P + R)

# 数学本质
# 调和平均 = 倒数的算术平均的倒数
# 对应"每个样本的平均错误贡献"视角
```

**业务含义**：
- Precision=0.01（几乎全是误报），Recall=0.99（几乎全召回）
- 算术平均 = 0.5，误导"模型还行"
- 调和平均 ≈ 0.02，正确反映"模型不可用"

## 怎么做（可落地步骤）

### 1. 指标选择决策流程

```
Step 1: 明确业务目标
├─ 关注正类预测质量（少误报）→ Precision 优先
├─ 关注正类覆盖完整（少漏报）→ Recall 优先  
└─ 两者平衡 → F1

Step 2: 评估样本分布
├─ 均衡（类别比例 1:1 ~ 1:10）→ Accuracy 可用，F1 更稳健
└─ 不均衡（类别比例 > 1:10）→ 禁用 Accuracy，必须看 P/R/F1

Step 3: 选择聚合方式（多分类）
├─ 每个类同等重要 → Macro-F1
├─ 整体性能优先 → Micro-F1  
└─ 关注少数类 → Macro-F1 或 Weighted-F1（按业务权重）
```

```python
import numpy as np
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    roc_auc_score, classification_report, confusion_matrix
)

def evaluate_classifier(y_true, y_pred, y_prob=None):
    """
    完整的分类评估流程
    
    Args:
        y_true: 真实标签
        y_pred: 预测标签（0/1）
        y_prob: 预测概率（用于 AUC）
    """
    # 1. 基础指标
    print("=" * 50)
    print("基础指标:")
    print(f"  Accuracy:  {accuracy_score(y_true, y_pred):.4f}")
    print(f"  Precision: {precision_score(y_true, y_pred):.4f}")
    print(f"  Recall:    {recall_score(y_true, y_pred):.4f}")
    print(f"  F1-Score:  {f1_score(y_true, y_pred):.4f}")
    
    # 2. 混淆矩阵
    print("\n混淆矩阵:")
    cm = confusion_matrix(y_true, y_pred)
    print(f"  TN={cm[0,0]}, FP={cm[0,1]}")
    print(f"  FN={cm[1,0]}, TP={cm[1,1]}")
    
    # 3. 样本分布
    unique, counts = np.unique(y_true, return_counts=True)
    print(f"\n样本分布: {dict(zip(unique, counts))}")
    print(f"类别比例: {counts[0]/counts[1]:.2f}:1")
    
    # 4. 判断是否不均衡
    if counts[0] / counts[1] > 10 or counts[1] / counts[0] > 10:
        print("⚠️  样本严重不均衡！Accuracy 可能失真，重点关注 P/R/F1")
    
    # 5. AUC（如果有概率）
    if y_prob is not None:
        auc = roc_auc_score(y_true, y_prob)
        print(f"\nROC-AUC: {auc:.4f}")
    
    # 6. 详细报告
    print("\n详细报告:")
    print(classification_report(y_true, y_pred, digits=4))

# 示例使用
y_true = np.array([0]*950 + [1]*50)  # 95% 负样本

```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **Accuracy** | 简单直观、易解释 | 不均衡时严重失真 | 类别相对均衡（1:1 ~ 1:3）|
| **Precision** | 控制误报、减少无效干预 | 可能漏掉大量正样本 | 风控、推荐、广告（误报成本高）|
| **Recall** | 覆盖完整、减少漏报 | 可能大量误报 | 医疗、安全、召回（漏报成本高）|
| **F1-Score** | 平衡 P 和 R | 无法反映 P/R 具体表现 | 需要综合评估的场景 |
| **Micro-F1** | 反映整体性能 | 忽略少数类表现 | 关注全局准确率的场景 |
| **Macro-F1** | 关注每个类平等 | 对少数类敏感但忽略样本量 | 每个类同等重要、样本不均衡 |
| **Weighted-F1** | 按样本量加权 | 可能掩盖少数类问题 | 需要平衡样本量影响 |
| **ROC-AUC** | 阈值无关、排序能力 | 不均衡时可能过于乐观 | 关注排序质量、阈值可调 |
| **PR-AUC** | 不均衡场景更稳健 | 解释性稍弱 | 样本严重不均衡、关注正类 |

### Micro-F1 vs Macro-F1 选择

```python
# Micro-F1 适合场景
# 1. 关注整体准确率，不特别关心某一类
# 2. 样本量大的类别更重要
# 3. 例如：搜索引擎整体效果（网页分类）

# Macro-F1 适合场景  
# 1. 每个类同等重要，无论样本量多少
# 2. 需要关注少数类表现
# 3. 例如：医疗诊断、罕见病识别、风控欺诈检测

# 实际案例
# 场景：100 类图像分类，99 类各 100 样本，1 类 10000 样本
# Micro-F1: 大类主导，接近大类的 F1
# Macro-F1: 所有类平等，小类表现差会拉低分数

# 建议
if 业务中每个类都重要:
    使用 Macro-F1
elif 关注整体性能:
    使用 Micro-F1
elif 样本不均衡但需要关注少数类:
    使用 Macro-F1 或 Weighted-F1（按业务权重）
```

## 高频追问（至少 5 个）

### Q1: 如果业务要求 Precision 和 Recall 都高怎么办？

**A**: 两个方向：
1. **模型改进**：更好的特征、更强的模型、集成方法、阈值调优
2. **业务妥协**：
   - 设定最低 Precision 阈值，最大化 Recall
   - 或设定最低 Recall 阈值，最大化 Precision
   - 或优化 F2/F0.5 等加权 F 值（更侧重某一指标）

```python
# F-beta: 可调节 P/R 权重
F_beta = (1 + beta**2) * P * R / (beta**2 * P + R)
# beta > 1: 更看重 Recall
# beta < 1: 更看重 Precision
```

### Q2: Micro-F1 和 Macro-F1 实际计算有什么区别？

**A**: 
- **Micro-F1**：先汇总所有类的 TP/FP/FN，再计算全局 P/R/F1
  - 等价于 Accuracy（多分类场景）
  - 被大类主导
  
- **Macro-F1**：先计算每个类的 P/R/F1，再求平均
  - 小类和大类权重相同
  - 对少数类更敏感

```python
# 三分类示例
# 类 A: TP=90, FP=10, FN=10 → P=0.9, R=0.9, F1=0.9
# 类 B: TP=9,  FP=1,  FN=1  → P=0.9, R=0.9, F1=0.9
# 类 C: TP=1,  FP=9,  FN=9  → P=0.1, R=0.1, F1=0.1

# Micro-F1: TP=100, FP=20, FN=20 → F1≈0.83（大类主导）
# Macro-F1: (0.9 + 0.9 + 0.1) / 3 = 0.63（小类拉低）
```

### Q3: 样本不均衡 1:100，应该怎么处理？

**A**: 分层处理：

1. **评估**：禁用 Accuracy，关注 Precision/Recall/F1，PR-AUC（非 ROC-AUC）
2. **数据**：
   - 过采样（SMOTE）适合正样本极少
   - 欠采样适合负样本极多
   - 混合策略更稳健
3. **训练**：
   - 类权重 `class_weight='balanced'` 或手动设置
   - Focal Loss 降低易分类样本权重
4. **阈值**：不使用默认 0.5，根据 PR 曲线选择
5. **集成**：EasyEnsemble、BalanceCascade 等专门方法

```python
# 推荐组合
from imblearn.pipeline import Pipeline
from imblearn.over_sampling import SMOTE
from sklearn.ensemble import GradientBoostingClassifier

pipeline = Pipeline([
    ('smote', SMOTE(sampling_strategy=0.3)),  # 过采样到 3:10
    ('clf', GradientBoostingClassifier())
])
pipeline.fit(X_train, y_train)
```

### Q4: ROC-AUC 和 PR-AUC 有什么区别？什么时候用哪个？

**A**:

| 维度 | ROC-AUC | PR-AUC |
|------|---------|--------|
| **曲线** | TPR vs FPR | Precision vs Recall |
| **基线** | 0.5（随机猜测）| 正类比例（如 0.01）|
| **不均衡** | 可能过于乐观 | 更稳健 |
| **关注点** | 整体排序能力 | 正类性能 |

**选择建议**：
- **样本均衡**：ROC-AUC 和 PR-AUC 都可以
- **样本不均衡且关注正类**：优先 PR-AUC
- **ROC-AUC 高但 PR-AUC 低**：说明负类预测好，但正类预测差

```python
# 示例：不均衡场景对比
# 数据：990 负样本 + 10 正样本
# 模型：随机猜测

# ROC-AUC = 0.5（看起来还行）
# PR-AUC = 0.01（正确反映模型对正类无用）

# 模型：预测全为负
# ROC-AUC = 0.5（无法区分好坏）
# PR-AUC = 0.01（正确反映问题）
```

### Q5: 多分类场景，类别 3 很重要但样本只有 5%，怎么评估？

**A**: 针对性策略：

1. **分离评估**：
   ```python
   # 将类别 3 vs 其他 作为二分类单独评估
   y_true_binary = (y_true == 3).astype(int)
   y_pred_binary = (y_pred == 3).astype(int)
   
   precision_3 = precision_score(y_true_binary, y_pred_binary)
   recall_3 = recall_score(y_true_binary, y_pred_binary)
   f1_3 = f1_score(y_true_binary, y_pred_binary)
   ```

2. **加权指标**：
   ```python
   # 给类别 3 更高权重
   weights = {0: 1, 1: 1, 2: 1, 3: 10}
   f1_weighted = f1_score(y_true, y_pred, average='weighted', 
                          sample_weight=[weights[y] for y in y_true])
   ```

3. **Per-Class 报告**：
   ```python
   print(classification_report(y_true, y_pred, digits=4))
   # 重点关注类别 3 的 precision/recall/f1-score
   ```

4. **业务对齐**：根据类别 3 的业务重要性，设定最低 Precision 或 Recall 阈值

### Q6: Precision 和 Recall 都很高，但 F1 不高，可能是什么原因？

**A**: 这**不可能发生**，F1 是 P 和 R 的调和平均，当两者都高时 F1 必然高。

可能的情况：
1. **计算错误**：检查是否有分母为零的情况
2. **理解偏差**：可能看的是算术平均而非 F1
3. **数据泄露**：训练集和测试集重叠，指标虚高

```python
# 验证关系
P, R = 0.9, 0.9
F1 = 2 * P * R / (P + R)  # = 0.9（三者一致）

# 只有当 P 和 R 差距大时，F1 才会被拉低
P, R = 0.99, 0.5
F1 = 2 * 0.99 * 0.5 / 1.49  # ≈ 0.66（被低值拉低）
```

### Q7: 如果样本极其不均衡 1:10000，SMOTE 还有用吗？

**A**: SMOTE 可能失效，建议：

1. **问题**：
   - SMOTE 生成的样本在高维空间可能不合理
   - 极度不均衡时，合成样本密度不足
   - 可能引入噪声

2. **替代方案**：
   - **异常检测方法**：One-Class SVM、Isolation Forest
   - **深度学习**：Autoencoder 重建误差、对比学习
   - **集成方法**：EasyEnsemble（多数类分成多份，分别与少数类组合训练）
   - **主动学习**：人工标注关键样本

3. **业务调整**：
   - 降低不均衡度要求（如从 1:10000 降到 1:100）
   - 接受更高的 FP 率（降低阈值）
   - 后处理规则过滤

## 常见错误（至少 3 个）

### 错误 1：样本不均衡时仍用 Accuracy 作为主指标

**错误描述**：
```python
# 99% 负样本 + 1% 正样本
model.predict_all_negative()
accuracy = 0.99  # 看起来很好

# 实际上模型对正样本预测全错
# 正确做法
f1 = f1_score(y_true, y_pred)  # = 0（正确反映问题）
```

**正确做法**：
- 检查样本分布，计算类别比例
- 不均衡时禁用 Accuracy，使用 Precision/Recall/F1
- 结合业务场景选择合适的聚合方式（Micro/Macro/Weighted）

### 错误 2：混淆 Precision 和 Recall 的含义

**错误描述**：
```python
# 错误理解
# "Precision 高说明我找到了很多正样本" ← 错！这是 Recall
# "Recall 高说明我预测为正的样本很准" ← 错！这是 Precision

# 正确理解
Precision = TP / (TP + FP)  # 预测为正中真正为正的比例
Recall = TP / (TP + FN)     # 实际为正中被找到的比例
```

**正确做法**：
- 记忆口诀：
  - **Precision**：预测为正的**精确度**（准不准）
  - **Recall**：实际为正的**召回率**（全不全）
- 画混淆矩阵理解：行是实际，列是预测
- 业务场景对齐：推荐（Precision）、召回（Recall）

### 错误 3：直接用默认阈值 0.5 而不调优

**错误描述**：
```python
# 不均衡场景下，默认 0.5 阈值往往不优
y_pred = (y_prob > 0.5).astype(int)  # 可能全预测为负

# 导致 Recall = 0，模型对正类无效
```

**正确做法**：
```python
# 1. 根据业务需求选择阈值
# 如：医疗诊断需要高 Recall，阈值设为 0.3
y_pred = (y_prob > 0.3).astype(int)

# 2. 根据 PR 曲线选择最优阈值
from sklearn.metrics import precision_recall_curve

precisions, recalls, thresholds = precision_recall_curve(y_true, y_prob)
# 最大化 F1
f1_scores = 2 * precisions * recalls / (precisions + recalls + 1e-8)
best_idx = np.argmax(f1_scores[:-1])  # 最后一个值对应 threshold=1
best_threshold = thresholds[best_idx]

# 3. 或根据业务约束
# 如：Precision 必须 > 0.9
valid_idx = np.where(precisions[:-1] >= 0.9)[0]
if len(valid_idx) > 0:
    best_threshold = thresholds[valid_idx[np.argmax(recalls[valid_idx])]]
```

### 错误 4：Multi-class 场景混淆 Micro-F1 和 Macro-F1

**错误描述**：
```python
# 错误：不均衡场景用 Micro-F1 认为模型很好
# 实际：大类主导，小类表现差

# 错误：只看 Macro-F1 忽略样本量
# 实际：可能优化了无关紧要的小类
```

**正确做法**：
- 明确业务场景：每个类是否同等重要？
- 打印 `classification_report` 查看每个类的详细指标
- 根据业务重要性选择聚合方式或自定义权重

### 错误 5：ROC-AUC 在不均衡场景下过于乐观

**错误描述**：
```python
# 不均衡场景 99% 负样本 + 1% 正样本
# 模型：随机猜测（AUC=0.5）

# 如果模型稍微好一点
roc_auc = 0.85  # 看起来不错

# 但 PR-AUC 可能很低
pr_auc = 0.3  # 正确反映正类预测不好
```

**正确做法**：
- 样本严重不均衡时，优先看 PR-AUC
- 或同时报告 ROC-AUC 和 PR-AUC，对比分析
- PR-AUC 的基线是正类比例（如 0.01），更严格

2. **Precision 和 Recall 的区别是什么？什么业务场景优先 Precision，什么优先 Recall？**
   - 参考答案：
     - Precision：预测为正中真正为正的比例（准不准）
     - Recall：实际为正中被找到的比例（全不全）
     - Precision 优先：推荐系统、广告投放（误报成本高）
     - Recall 优先：医疗诊断、安全检测（漏报成本高）

3. **Micro-F1 和 Macro-F1 的计算方式和适用场景？**
   - 参考答案：
     - Micro：全局聚合 TP/FP/FN，适合关注整体性能
     - Macro：每类独立计算后平均，适合每类同等重要、样本不均衡

4. **样本不均衡 1:100 应该怎么处理？给出至少 3 种方法。**
   - 参考答案：
     - 数据：过采样（SMOTE）、欠采样、混合采样
     - 算法：类权重、Focal Loss、阈值调整
     - 评估：禁用 Accuracy，关注 P/R/F1，使用 PR-AUC

5. **ROC-AUC 和 PR-AUC 的区别？什么时候用哪个？**
   - 参考答案：
     - ROC-AUC：TPR vs FPR，样本均衡时可用
     - PR-AUC：Precision vs Recall，样本不均衡且关注正类时优先

### 手写（5 分钟能写出）

1. **手写 Precision、Recall、F1 的计算公式**

```python
# 基础指标
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 = 2 * Precision * Recall / (Precision + Recall)

# 多分类 Micro-F1
Micro_P = sum(TP_i) / sum(TP_i + FP_i)
Micro_R = sum(TP_i) / sum(TP_i + FN_i)
Micro_F1 = 2 * Micro_P * Micro_R / (Micro_P + Micro_R)

# 多分类 Macro-F1
Macro_P = mean(Precision_i)
Macro_R = mean(Recall_i)
Macro_F1 = mean(F1_i)
```

2. **给定混淆矩阵，计算 Precision、Recall、F1**

```python
# 混淆矩阵
#         预测负  预测正
# 实际负    950     50
# 实际正     20     80

TN, FP, FN, TP = 950, 50, 20, 80

Precision = TP / (TP + FP) = 80 / 130 = 0.615
Recall = TP / (TP + FN) = 80 / 100 = 0.8
F1 = 2 * 0.615 * 0.8 / (0.615 + 0.8) = 0.695

Accuracy = (TP + TN) / (TP + TN + FP + FN) = 1030 / 1100 = 0.936
```

3. **写一个函数，根据 PR 曲线选择最优阈值**

```python
from sklearn.metrics import precision_recall_curve
import numpy as np

def find_best_threshold(y_true, y_prob, metric='f1'):
    """
    根据 PR 曲线选择最优阈值
    
    Args:
        y_true: 真实标签
        y_prob: 预测概率
        metric: 'f1', 'precision', 'recall'
    """
    precisions, recalls, thresholds = precision_recall_curve(y_true, y_prob)
    
    if metric == 'f1':
        f1_scores = 2 * precisions * recalls / (precisions + recalls + 1e-8)
        best_idx = np.argmax(f1_scores[:-1])
    elif metric == 'precision':
        best_idx = np.argmax(precisions[:-1])
    elif metric == 'recall':
        best_idx = np.argmax(recalls[:-1])
    
    return thresholds[best_idx], precisions[best_idx], recalls[best_idx]

# 使用示例
best_thresh, best_p, best_r = find_best_threshold(y_true, y_prob, metric='f1')
print(f"最佳阈值: {best_thresh:.3f}, P={best_p:.3f}, R={best_r:.3f}")
```
