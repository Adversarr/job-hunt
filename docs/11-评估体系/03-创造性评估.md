# 创造性评估：新颖度、多样性与实用性的权衡

## 一句话结论
创造性评估需从**新颖度**（与训练集/现有知识的差异）、**多样性**（生成样本间的差异）、**实用性**（创造性是否有价值）三个维度综合判断；自动指标（Self-BLEU/Distinct-n）可量化多样性但无法评估"有价值的创造"，人工评估仍是核心。

## 核心定义/公式

### 1. 创造性的三维评估框架

### 2. 自动评估指标

#### Self-BLEU（多样性指标）
```python
# Self-BLEU: 生成样本之间的相似度（越低越多样）
# 思路：用一个样本做 reference，其他样本做 candidate

import numpy as np
from nltk.translate.bleu_score import sentence_bleu

def compute_self_bleu(samples: list) -> float:
    """
    计算 Self-BLEU
    Args:
        samples: 同一 prompt 生成的多个样本列表
    Returns:
        self_bleu: 平均 BLEU 分数（越低越好）
    """
    if len(samples) < 2:
        return 0.0
    
    bleu_scores = []
    for i, candidate in enumerate(samples):
        # 其他样本作为 references
        references = [samples[j].split() for j in range(len(samples)) if j != i]
        candidate_tokens = candidate.split()
        
        # 计算该样本的 BLEU 分数
        score = sentence_bleu(references, candidate_tokens, weights=(0.25, 0.25, 0.25, 0.25))
        bleu_scores.append(score)
    
    return np.mean(bleu_scores)

# Self-BLEU 的局限：
# 1. 只衡量字面多样性，不衡量语义多样性
# 2. 对同义改写不敏感（"我喜欢" vs "我喜爱" 字面不同但语义相同）
# 3. 无法区分"有价值的多样性"和"随机噪声"
```

#### Distinct-n（多样性指标）
```python
# Distinct-n: 唯一 n-gram 占总 n-gram 的比例（越高越多样）

from collections import Counter

def compute_distinct_n(texts: list, n: int = 2) -> float:
    """
    计算 Distinct-n
    Args:
        texts: 生成文本列表
        n: n-gram 的 n 值
    Returns:
        distinct_score: Distinct-n 分数（0-1，越高越好）
    """
    all_ngrams = []
    
    for text in texts:
        tokens = text.split()
        if len(tokens) < n:
            continue
        
        # 提取 n-grams
        ngrams = [tuple(tokens[i:i+n]) for i in range(len(tokens) - n + 1)]
        all_ngrams.extend(ngrams)
    
    if not all_ngrams:
        return 0.0
    
    # 唯一 n-gram 数量 / 总 n-gram 数量
    unique_ngrams = set(all_ngrams)
    distinct_score = len(unique_ngrams) / len(all_ngrams)
    
    return distinct_score

# Distinct-1: 词汇多样性（unigram）
# Distinct-2: 短语多样性（bigram）
# Distinct-3: 句式多样性（trigram）

# 局限性：
# 1. 受文本长度影响（短文本天然 Distinct 高）
# 2. 无法区分"有价值的新颖"和"语法错误的新颖"
# 3. 对领域术语敏感（技术文档 Distinct 低但不代表不创造）
```

#### Entropy-based Diversity（基于熵的多样性）
```python
# 利用生成概率分布的熵衡量多样性
import torch
import torch.nn.functional as F

def compute_entropy_diversity(model, tokenizer, prompt: str, num_samples: int = 10, temperature: float = 1.0):
    """
    基于生成概率分布熵的多样性评估
    Args:
        model: 语言模型
        tokenizer: 分词器
        prompt: 输入提示
        num_samples: 生成样本数
        temperature: 采样温度
    Returns:
        avg_entropy: 平均 token 级熵（越高越多样）
    """
    inputs = tokenizer(prompt, return_tensors="pt")
    entropies = []
    
    with torch.no_grad():
        outputs = model(**inputs)
        logits = outputs.logits / temperature  # [seq_len, vocab_size]
        
        # 计算每个位置的熵
        probs = F.softmax(logits, dim=-1)
        entropy = -torch.sum(probs * torch.log(probs + 1e-10), dim=-1)  # [seq_len]
        
        avg_entropy = torch.mean(entropy).item()
    
    return avg_entropy

# 高熵 = 高不确定性 = 模型在该位置"创造力"高
# 但也可能是：模型不确定（训练不充分）/ 该位置本就多样（合理）
# 需要结合任务判断
```

#### Novelty Score（新颖度指标）
```python
# 新颖度：生成内容与训练集的差异

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

def compute_novelty(generated_text: str, training_corpus: list, top_k: int = 10) -> float:
    """
    计算新颖度
    Args:
        generated_text: 生成文本
        training_corpus: 训练集样本
        top_k: 取最相似的 k 个样本计算
    Returns:
        novelty: 新颖度分数（0-1，越高越新颖）
    """
    # 使用 TF-IDF 或 embedding 计算相似度
    vectorizer = TfidfVectorizer()
    
    # 训练集 + 生成文本
    all_texts = training_corpus + [generated_text]
    tfidf_matrix = vectorizer.fit_transform(all_texts)
    
    # 生成文本的向量（最后一个）
    gen_vec = tfidf_matrix[-1]
    train_vecs = tfidf_matrix[:-1]
    
    # 计算与所有训练样本的相似度
    similarities = cosine_similarity(gen_vec, train_vecs).flatten()
    
    # 取最相似的 top_k 个
    top_similarities = sorted(similarities, reverse=True)[:top_k]
    
    # 新颖度 = 1 - 平均最大相似度
    novelty = 1 - np.mean(top_similarities)
    
    return novelty

# 问题：
# 1. 高新颖度可能意味着"幻觉"或"错误"
# 2. 低新颖度不一定不好（标准答案场景）
# 3. 需要结合质量判断
```

## 为什么（3 个因果链）

### 1. 为什么自动指标难以评估"有价值的创造性"？

```
创造性本质 = 新颖性 + 价值性
        ↓
自动指标只能量化"新颖性"（与已知数据的差异）
        ↓
"价值性"需要人类判断：是否解决问题？是否有洞察？是否合理？
        ↓
高新颖性 + 低价值性 = 幻觉/错误/胡言乱语（自动指标无法区分）
```

**具体例子**：
- **场景**：让模型生成"如何提高团队效率"
- **创造性输出**："引入 OKR 目标管理，每周同步进度，建立反馈文化..."（新颖且有价值）
- **胡乱输出**："采用量子管理法，使用火星时间制，建立心灵感应网络..."（新颖但无价值）
- **自动指标**：两者新颖度都高，但只有前者有价值

**根本原因**：
1. 自动指标缺乏"常识"和"领域知识"
2. 无法判断创新是否合理、可行
3. 无法理解"解决问题"这一核心目标

### 2. 为什么 Self-BLEU 和 Distinct-n 不足以评估创造性？

```
Self-BLEU/Distinct-n 设计初衷 = 量化文本多样性
            ↓
多样性 ≠ 创造性（必要但不充分条件）
            ↓
问题1：字面多样性 vs 语义多样性
    "我喜欢吃苹果" vs "我喜爱吃苹果" → 字面不同，语义相同
            ↓
问题2：高质量多样性 vs 低质量多样性
    "方案A...方案B...方案C..." vs "乱码1...乱码2...乱码3..." → 都高多样性，但后者无意义
            ↓
问题3：任务相关多样性 vs 任务无关多样性
    代码生成：语法正确的不同实现 vs 语法错误的"多样"输出
            ↓
结论：自动指标需要配合质量评估（流畅性、正确性、相关性）
```

**量化证据**：
- 研究表明，Self-BLEU 与人类判断的创造性相关性仅 0.2-0.3
- 高 Distinct-n 可能来自语法错误、错别字等噪声
- 创造性任务中，自动指标与人评相关性普遍低于 0.4

### 3. 为什么创造性评估比事实性评估更难？

```
事实性评估 → 有客观标准（真假、对错）→ 可用知识库/NLI 验证 → 自动化程度高
        ↓
创造性评估 → 无客观标准（主观判断）→ 需要评估"新颖+有价值" → 主观性强
        ↓
核心挑战：
1. 标准：什么是"好的创造"？（艺术创作 vs 技术方案 vs 商业建议）
2. 权衡：新颖性 vs 可行性 vs 实用性
3. 文化：不同文化/领域对创造性的定义不同
4. 评估者：需要领域专家，普通人无法判断
        ↓
结果：创造性评估高度依赖人工，且评估者一致性低
```

**具体差异**：
| 维度 | 事实性评估 | 创造性评估 |
|------|----------|----------|
| 标准 | 客观（真/假） | 主观（好/坏/新颖） |
| 自动化 | 高（知识库、NLI） | 低（需人类判断） |
| 一致性 | 高（Kappa > 0.8） | 低（Kappa < 0.6） |
| 评估者 | 普通人即可 | 需要领域专家 |
| 指标可靠性 | 高 | 低 |

#### Step 2: 自动指标计算（多样性与新颖度）
```python
import numpy as np
from typing import List, Dict
from collections import Counter
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

class CreativityAutoEvaluator:
    """创造性自动评估器"""
    
    def __init__(self):
        self.vectorizer = TfidfVectorizer(max_features=5000)
    
    def evaluate_diversity(self, samples: List[str]) -> Dict[str, float]:
        """
        评估多样性
        Args:
            samples: 同一 prompt 生成的多个样本
        Returns:
            diversity_scores: 多样性指标字典
        """
        if len(samples) < 2:
            return {"distinct_1": 0, "distinct_2": 0, "self_bleu": 0, "entropy": 0}
        
        # 1. Distinct-n
        distinct_1 = self._compute_distinct_n(samples, n=1)
        distinct_2 = self._compute_distinct_n(samples, n=2)
        distinct_3 = self._compute_distinct_n(samples, n=3)
        
        # 2. Self-BLEU
        self_bleu = self._compute_self_bleu(samples)
        
        # 3. Embedding-based Diversity
        embedding_diversity = self._compute_embedding_diversity(samples)
        
        return {
            "distinct_1": distinct_1,
            "distinct_2": distinct_2,
            "distinct_3": distinct_3,
            "self_bleu": self_bleu,  # 越低越好
            "embedding_diversity": embedding_diversity  # 越高越好
        }
    
    def evaluate_novelty(
        self, 
        generated_texts: List[str], 
        training_corpus: List[str],
        top_k: int = 5
    ) -> Dict[str, float]:
        """
        评估新颖度
        Args:
            generated_texts: 生成文本列表
            training_corpus: 训练集样本
            top_k: 取最相似的 k 个样本
        Returns:
            novelty_scores: 新颖度指标字典
        """
        if not training_corpus:
            return {"avg_novelty": 0, "max_overlap": 0}
        
        # 构建 TF-IDF 矩阵
        all_texts = training_corpus + generated_texts
        tfidf_matrix = self.vectorizer.fit_transform(all_texts)
        
        # 分离训练集和生成集
        train_vecs = tfidf_matrix[:len(training_corpus)]
        gen_vecs = tfidf_matrix[len(training_corpus):]
        
        # 计算相似度
        similarities = cosine_similarity(gen_vecs, train_vecs)  # [num_gen, num_train]
        
        # 对每个生成文本，取最相似的 top_k 个训练样本
        novelty_scores = []
        max_overlaps = []
        
        for i in range(len(generated_texts)):
            top_k_sims = sorted(similarities[i], reverse=True)[:top_k]
            novelty = 1 - np.mean(top_k_sims)
            max_overlap = 1 - max(similarities[i])
            
            novelty_scores.append(novelty)
            max_overlaps.append(max_overlap)
        
        return {
            "avg_novelty": np.mean(novelty_scores),
            "max_overlap": np.mean(max_overlaps),  # 与训练集的最大相似度（越低越新颖）
            "novelty_std": np.std(novelty_scores)  # 新颖度的稳定性
        }
    
    def _compute_distinct_n(self, texts: List[str], n: int) -> float:
        """计算 Distinct-n"""
        all_ngrams = []
        
        for text in texts:
            tokens = text.split()
            if len(tokens) < n:
                continue
            ngrams = [tuple(tokens[i:i+n]) for i in range(len(tokens) - n + 1)]
            all_ngrams.extend(ngrams)
        
        if not all_ngrams:
            return 0.0
        
        return len(set(all_ngrams)) / len(all_ngrams)
    
    def _compute_self_bleu(self, samples: List[str]) -> float:
        """计算 Self-BLEU（简化版）"""
        from nltk.translate.bleu_score import sentence_bleu
        
        if len(samples) < 2:
            return 0.0
        
        bleu_scores = []
        for i, candidate in enumerate(samples):
            references = [samples[j].split() for j in range(len(samples)) if j != i]
            candidate_tokens = candidate.split()
            
            try:
                score = sentence_bleu(references, candidate_tokens, weights=(0.25, 0.25, 0.25, 0.25))
                bleu_scores.append(score)
            except:
                bleu_scores.append(0)
        
        return np.mean(bleu_scores)
    
    def _compute_embedding_diversity(self, samples: List[str]) -> float:
        """基于 embedding 的多样性（简化版，使用 TF-IDF）"""
        if len(samples) < 2:
            return 0.0
        
        # 使用 TF-IDF 作为简易 embedding
        tfidf_matrix = self.vectorizer.fit_transform(samples)
        
        # 计算两两余弦距离
        from sklearn.metrics.pairwise import cosine_similarity
        sim_matrix = cosine_similarity(tfidf_matrix)
        
        # 取上三角（不含对角线）
        upper_tri = sim_matrix[np.triu_indices(len(samples), k=1)]
        
        # 平均余弦距离
        avg_distance = 1 - np.mean(upper_tri)
        
        return avg_distance
    
    def comprehensive_evaluation(
        self,
        samples: List[str],
        training_corpus: List[str] = None,
        reference: str = None
    ) -> Dict[str, float]:
        """
        综合评估
        Args:
            samples: 生成样本
            training_corpus: 训练集（可选）
            reference: 参考答案（可选）
        Returns:
            comprehensive_scores: 综合评分
        """
        results = {}
        
        # 1. 多样性评估
        diversity_scores = self.evaluate_diversity(samples)
        results.update(diversity_scores)
        
        # 2. 新颖度评估
        if training_corpus:
            novelty_scores = self.evaluate_novelty(samples, training_corpus)
            results.update(novelty_scores)
        
        # 3. 质量检查（必须项）
        quality_checks = self._check_quality(samples, reference)
        results.update(quality_checks)
        
        # 4. 综合得分（加权平均）
        # 注意：自动指标只能参考，真正创造性需要人评
        results["auto_overall"] = self._compute_auto_overall(results)
        
        return results
    
    def _check_quality(self, samples: List[str], reference: str = None) -> Dict[str, float]:
        """质量检查：流畅性、相关性等"""
        # 流畅性检查（简化：句子长度、标点）
        fluency_scores = []
        for sample in samples:
            # 基本检查：是否有明显错误
            has_error = (
                len(sample.split()) < 3 or  # 过短
                sample.count("。") > 10 or  # 过多句号
                any(char in sample for char in ["�", "???", "!!!"])  # 异常字符
            )
            fluency_scores.append(0 if has_error else 1)
        
        return {
            "fluency_ratio": np.mean(fluency_scores),
            "avg_length": np.mean([len(s.split()) for s in samples])
        }
    
    def _compute_auto_overall(self, scores: Dict[str, float]) -> float:
        """计算自动指标综合得分（仅供参考）"""
        # 注意：这是启发式规则，实际应用需要调整
        diversity_score = scores.get("distinct_2", 0)
        novelty_score = scores.get("avg_novelty", 0)
        fluency_score = scores.get("fluency_ratio", 1)
        
        # 加权平均
        overall = 0.3 * diversity_score + 0.3 * novelty_score + 0.4 * fluency_score
        
        return overall
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **Self-BLEU** | 快速量化字面多样性、无需额外模型 | 忽略语义多样性、对同义改写不敏感、无法区分质量 | 快速筛选、监控训练、需配合质量指标 |
| **Distinct-n** | 简单直观、可解释、无参数 | 受文本长度影响、无法区分有价值多样性 vs 噪声 | 词汇级多样性评估、辅助指标 |
| **Entropy-based** | 理论基础强、捕捉模型不确定性 | 需要模型 logits、无法区分"合理不确定"vs"训练不足" | 研究场景、配合其他指标 |
| **Novelty Score** | 量化与训练集差异、检测过拟合 | 需要训练集、高新颖度可能意味着幻觉/错误 | 有训练集场景、检测抄袭/过拟合 |
| **人工评估** | 最接近真实创造力判断、可评估"价值性" | 成本高、一致性难保证、需要领域专家 | 关键决策、模型对比、最终验证 |
| **混合评估** | 兼顾成本与质量、自动指标筛选+人评验证 | 流程复杂、需要协调自动指标与人评 | 生产环境、持续迭代 |

### 关键权衡决策树

```
需要评估创造性
    │
    ├─ 任务类型？
    │   ├─ 问题解决 → 优先级：实用性 > 新颖性 > 多样性
    │   ├─ 创意写作 → 优先级：新颖性 > 引人入胜 > 多样性
    │   └─ 代码生成 → 优先级：正确性 > 效率 > 新颖性
    │
    ├─ 是否有训练集？
    │   ├─ 是 → 计算 Novelty Score（检测过拟合）
    │   └─ 否 → 无法计算新颖度，依赖人评
    │
    ├─ 评估预算？
    │   ├─ 高 → 完整人评（3-5 标注员/样本）+ 自动指标辅助
    │   ├─ 中 → 自动指标筛选 + 小规模人评（重点样本）
    │   └─ 低 → 仅自动指标 + LLM-as-Judge
    │
    └─ 决策？
        ├─ 自动指标筛选 → 高多样性 + 高新颖度 + 高质量 → 候选
        ├─ 人评验证 → 综合得分 > 阈值 → 通过
        └─ 线上 A/B → 最终验证
```

### 1. Q: Self-BLEU 和 ROUGE 在评估创造性时有什么局限性？

A:

**Self-BLEU 的局限性**：
1. **只衡量字面多样性，忽略语义多样性**：
   - "我喜欢吃苹果" vs "我喜爱吃苹果" → 字面不同但语义相同，Self-BLEU 会认为多样
   - 反之，语义不同但字面相似的表达可能被误判

2. **无法区分"有价值的多样性"和"低质量多样性"**：
   - 高 Self-BLEU 可能来自：语法错误、错别字、语义混乱
   - 自动指标无法判断"这样的多样性是否有价值"

3. **受样本数量影响**：
   - 样本数少时，Self-BLEU 不稳定
   - 需要至少 5-10 个样本才能得到有意义的分数

**ROUGE 的局限性**：
1. **设计初衷与创造性冲突**：
   - ROUGE 用于评估字面匹配（摘要、翻译）
   - 创造性评估的核心是"与参考不同"
   - 高 ROUGE 反而意味着低新颖性

2. **惩罚创造性改写**：
   - 创造性输出往往字面不同但语义相关
   - ROUGE 只看字面重叠，会低估创造性输出

3. **无法评估多样性**：
   - ROUGE 是单一参考对单一输出
   - 无法衡量多个输出之间的多样性

**正确使用方式**：
- Self-BLEU：用于快速筛选，配合质量指标（流畅性、相关性）
- ROUGE：用于评估"相关性/质量"，不用于评估创造性
- 创造性评估：Novelty Score + 多样性指标 + 人工评估

```python
def check_inter_annotator_agreement(annotations):
    """
    检查标注员间一致性
    Args:
        annotations: List[List[int]], 每行是一个样本，每列是一个标注员的评分
    Returns:
        kappa: Fleiss' Kappa 系数
    """
    from statsmodels.stats.inter_rater import fleiss_kappa
    
    # 转换为 Fleiss' Kappa 需要的格式
    # annotations: [num_samples, num_annotators]
    # 需要转换为：[num_samples, num_categories]
    
    num_samples = len(annotations)
    num_categories = 5  # 1-5 分
    
    category_counts = np.zeros((num_samples, num_categories))
    
    for i, sample_annotations in enumerate(annotations):
        for score in sample_annotations:
            category_counts[i, score - 1] += 1
    
    kappa = fleiss_kappa(category_counts)
    
    return kappa

# 解释
# Kappa > 0.8: 优秀一致性
# Kappa 0.6-0.8: 良好一致性
# Kappa 0.4-0.6: 中等一致性
# Kappa < 0.4: 一致性差，需要重新培训
```

```python
    def __init__(self):
        self.metrics_buffer = []
        self.alert_thresholds = {
            "diversity_drop": 0.2,  # 多样性下降 20% 触发告警
            "novelty_drop": 0.15,
            "quality_drop": 0.1
        }

        """
        监控一批生成结果
        """
        # 1. 实时计算自动指标
        diversity = self._compute_diversity(generated_texts)
        novelty = self._compute_novelty(generated_texts)
        quality = self._check_quality(generated_texts)
        
        # 2. 记录指标
        metrics = {
            "timestamp": time.time(),
            "diversity": diversity,
            "novelty": novelty,
            "quality": quality,
            "num_samples": len(generated_texts)
        }
        self.metrics_buffer.append(metrics)
        
        # 3. 异常检测
        alerts = self._detect_anomalies(metrics)

        if self._should_sample_for_human_eval():
            samples_for_human = self._select_samples(prompts, generated_texts)
            self._trigger_human_eval(samples_for_human)
        
        return alerts
    
    def _detect_anomalies(self, current_metrics):

        """
        # 定期采样：每天 1%
        # 或检测到异常时立即采样
        return True  # 简化
```

```python
    """
    使用 LLM 作为 judge 评估创造性
    """
    
    # 设计评估 prompt
    evaluation_prompt = f"""
请评估以下生成文本的创造性。

输入提示：{prompt}

生成文本：{generated_text}

请从以下维度打分（1-5分）：

1. 新颖性（Novelty）：是否提供新观点/新方法/新角度
   - 5分：独特新颖，超出预期
   - 4分：有一定新意
   - 3分：常规
   - 2分：略显陈旧
   - 1分：抄袭或陈词滥调

2. 实用性（Utility）：创新是否有价值
   - 5分：创新且有效解决问题
   - 4分：创新且部分有效
   - 3分：创新但实用性一般
   - 2分：创新但无价值
   - 1分：胡乱创新

3. 连贯性（Coherence）：是否逻辑自洽
   - 5分：逻辑严密
   - 4分：基本连贯
   - 3分：部分跳跃
   - 2分：逻辑混乱
   - 1分：不知所云

请以 JSON 格式输出：
{{
    "novelty_score": <1-5>,
    "utility_score": <1-5>,
    "coherence_score": <1-5>,
    "overall_score": <1-5>,
    "rationale": "<简要说明>"
}}
"""
    
    # 调用 judge 模型
    response = judge_model.generate(evaluation_prompt)
    
    # 解析结果
    result = parse_json(response)
    
    return result


# 使用多个 judge 模型投票
def ensemble_llm_judge(judge_models, prompt, generated_text):
    """
    多个 LLM judge 投票
    """
    results = []
    
    for model in judge_models:
        result = llm_judge_creativity(model, prompt, generated_text)
        results.append(result)
    
    # 聚合
    aggregated = {
        "novelty": np.mean([r["novelty_score"] for r in results]),
        "utility": np.mean([r["utility_score"] for r in results]),
        "coherence": np.mean([r["coherence_score"] for r in results]),
        "overall": np.mean([r["overall_score"] for r in results])
    }
    
    return aggregated
```

**优势**：
1. **成本低**：相比人工评估，成本降低 10-100 倍
2. **一致性好**：同一模型评估一致性好
3. **可扩展**：可以大规模评估

**局限性**：
1. **Self-preference**：模型可能偏好自己风格
2. **与人类 gap**：约 70-80% 一致率
3. **创造力判断难度**：LLM 本身对"创造性"的理解有限
4. **无法评估超越自身能力**：如果生成内容比 judge 模型更先进，judge 无法正确评估

**缓解方法**：
1. **使用更强的 judge 模型**：GPT-4 评估 GPT-3.5 的输出
2. **多模型投票**：减少单一模型偏见
3. **人工校准**：用小规模人评校准 LLM judge
4. **结合自动指标**：LLM judge + 传统自动指标

## 常见错误（5 个）

### 1. 错误：只用自动指标评估创造性

**描述**：认为 Self-BLEU/Distinct-n 足以评估创造性，忽略人工评估

**问题**：
- Self-BLEU 高可能意味着语法错误、语义混乱
- 无法区分"有价值的创造"和"胡编乱造"
- 自动指标与人类判断相关性低（< 0.4）

**正确做法**：
- 自动指标用于快速筛选和监控
- 关键决策必须有人评验证
- 明确自动指标的局限性

### 2. 错误：混淆"多样性"和"创造性"

**描述**：认为高多样性 = 高创造性

**问题**：
- 多样性是创造性的必要条件，不是充分条件
- 高多样性 + 低质量 = 噪声
- 忽略了创造性的核心："有价值的新颖"

**正确做法**：
- 明确创造性 = 新颖性 + 价值性
- 分离评估维度：多样性、新颖性、实用性
- 综合判断，不是单一指标

### 3. 错误：人评标准不清晰或不一致

**描述**：让标注员"整体评估创造性"，没有明确的 rubric

**问题**：
- 标注员理解不一致，一致性低（Kappa < 0.4）
- 无法区分不同维度的问题
- 结果不可复现

**正确做法**：
- 分离评估维度（新颖性、实用性、连贯性）
- 提供详细的 rubric 和示例
- 进行标注培训和一致性检查
- Fleiss' Kappa > 0.6 才接受

### 4. 错误：忽略创造性与质量的权衡

**描述**：追求高新颖性，忽略正确性、流畅性、相关性

**问题**：
- 高新颖性 + 低质量 = 幻觉/错误
- 模型为了"创新"而牺牲正确性
- 用户体验差

**正确做法**：
- 设置质量阈值（流畅性、相关性必须达标）
- 新颖性是加分项，不是必须项
- 综合评估，权重根据任务调整

### 5. 错误：忽略任务场景差异

**描述**：对所有任务使用相同的创造性评估标准

**问题**：
- 代码生成需要正确性优先，创造性是次要的
- 创意写作需要新颖性优先
- 对话需要平衡相关性和有趣性

**正确做法**：
- 根据任务类型定制评估维度和权重
- 明确哪些任务需要高创造性，哪些需要稳定性
- 分级策略：高风险场景低创造，低风险场景高创造
