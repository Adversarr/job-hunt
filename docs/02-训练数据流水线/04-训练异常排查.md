# 常见训练异常排查

## 一句话结论
Loss 震荡先查学习率和数据分布，NaN/inf 优先检查 softmax 的 max-trick、loss scaling 和异常样本；定位顺序是"数值→数据→模型→并行"。

## 核心定义/公式

### Loss 震荡诊断框架
```
震荡特征 → 可能原因 → 排查优先级
├── 高频小振幅 → 学习率过大 / batch 过小 / 数据噪声
├── 低频大振幅 → 数据分布突变 / 异常样本 / 梯度爆炸
└── 阶段性震荡 → 学习率调度 / 数据泄露 / 混合精度溢出
```

### Softmax 数值稳定公式
```python
# 标准 softmax（不稳定）
softmax(x) = exp(x) / sum(exp(x))  # 易溢出

# 数值稳定版本（max-trick）
def stable_softmax(x):
    x_max = x.max(dim=-1, keepdim=True)
    exp_x = torch.exp(x - x_max)
    return exp_x / exp_x.sum(dim=-1, keepdim=True)
```

### NaN/Inf 排查优先级
```
1. 数值层面（最快定位）
   ├── softmax 是否做了 max-trick
   ├── layernorm/rmsnorm 输入是否含 NaN
   ├── loss scaling 是否过大（混合精度）
   └── 梯度裁剪是否生效

2. 数据层面
   ├── 异常样本（空文本、超长序列、特殊字符）
   ├── 标签错误（index out of range）
   └── 数据分布突变（训练/验证集不一致）

3. 模型层面
   ├── 权重初始化是否合理
   ├── 梯度爆炸/消失（网络过深）
   └── 激活函数饱和（sigmoid/tanh 梯度为 0）

4. 并行层面
   ├── 分布式同步是否正确
   ├── 梯度聚合顺序问题
   └── 混合精度转换时机
```

## 为什么（2-3 个因果链）

### 1. Softmax 为何易产生 NaN/Inf
**现象**：直接计算 `exp(logits)` 时，大值导致 inf，小值导致 underflow
**根因**：FP16 最大值约 65536，`exp(20) ≈ 4.8e8` 已超限；softmax 输入范围不可控
**结果**：未做 max-trick 时，任何大 logit 都会触发 inf，反向传播产生 NaN

**因果链**：
```
logit 值过大（如 50）
    ↓
exp(50) ≈ 5.18e21（超出 FP16 范围）
    ↓
softmax 输出 = inf / sum(inf) = nan
    ↓
loss = nan，梯度无法计算，训练崩溃
```

### 2. Loss 震荡为何优先查学习率和数据
**现象**：Loss 呈锯齿状波动或周期性跳变
**根因**：学习率过大 → 梯度更新幅度超过损失曲面曲率；数据突变 → 优化方向冲突
**结果**：模型在局部极小值附近反复跳跃，无法收敛

**因果链（学习率）**：
```
学习率 η 过大（如 1e-3 对大模型）
    ↓
梯度更新 Δθ = -η∇L 幅度过大
    ↓
越过局部极小值，Loss 上升
    ↓
下一次更新方向相反，再次越过
    ↓
形成震荡，无法稳定收敛
```

**因果链（数据）**：
```
训练数据分布不均（如 domain shift）
    ↓
Batch 内样本来自不同分布
    ↓
梯度方向冲突，优化轨迹抖动
    ↓
Loss 呈现高频震荡
```

### 3. 混合精度训练为何需要 Loss Scaling
**现象**：FP16 训练时梯度经常下溢为 0
**根因**：FP16 动态范围小（6e-8 到 65536），小梯度 `≈1e-7` 会被截断为 0
**结果**：梯度消失，权重不更新，训练停滞

**因果链**：
```
FP16 最小正数 ≈ 6e-8
    ↓
深层网络梯度 ≈ 1e-7 到 1e-9（梯度消失）
    ↓
FP16 表示不了 → 截断为 0
    ↓
权重不更新，训练无效
    ↓
Loss Scaling 放大 loss → 梯度同步放大 → 反向后还原
```

## 怎么做（可落地步骤）

### 标准 NaN/Inf 排查流程

```python
# Step 1: 全局 NaN 检测 Hook
def register_nan_hook(model):
    def hook(module, input, output):
        if isinstance(output, torch.Tensor):
            if torch.isnan(output).any():
                print(f"[NaN] {module.__class__.__name__}")
                print(f"  input: min={input[0].min()}, max={input[0].max()}")
                raise ValueError("NaN detected, stopping training")
        elif isinstance(output, tuple):
            for i, o in enumerate(output):
                if isinstance(o, torch.Tensor) and torch.isnan(o).any():
                    print(f"[NaN] {module.__class__.__name__}[{i}]")
                    raise ValueError("NaN detected")
    
    for name, module in model.named_modules():
        module.register_forward_hook(hook)

# Step 2: 梯度检查
def check_gradients(model):
    for name, param in model.named_parameters():
        if param.grad is not None:
            if torch.isnan(param.grad).any():
                print(f"[Grad NaN] {name}")
            if torch.isinf(param.grad).any():
                print(f"[Grad Inf] {name}: max={param.grad.max()}")

# Step 3: 数据检查
def check_batch(batch):
    input_ids = batch['input_ids']
    labels = batch['labels']
    
    # 检查越界
    if input_ids.max() >= vocab_size:
        print(f"[Data Error] input_ids max={input_ids.max()}, vocab_size={vocab_size}")
    
    # 检查空序列
    if (input_ids == pad_token_id).all(dim=-1).any():
        print("[Data Error] Found empty sequence")
    
    # 检查异常长度
    seq_len = (input_ids != pad_token_id).sum(dim=-1)
    if seq_len.max() > 8192:
        print(f"[Data Warning] Long sequence: {seq_len.max()}")

# Step 4: Loss 监控
class LossMonitor:
    def __init__(self, window=100):
        self.window = window
        self.losses = []
    
    def update(self, loss):
        self.losses.append(loss)
        if len(self.losses) > self.window:
            self.losses.pop(0)
        
        if len(self.losses) == self.window:
            mean = sum(self.losses) / self.window
            std = (sum((x - mean)**2 for x in self.losses) / self.window) ** 0.5
            
            # 震荡检测：标准差大于均值的 20%
            if std > mean * 0.2:
                return f"[Warning] Loss oscillating: mean={mean:.4f}, std={std:.4f}"
        
        # NaN/Inf 检测
        if math.isnan(loss) or math.isinf(loss):
            return f"[Error] Loss is {loss}"
        
        return None
```

## 高频追问（至少 5 个）

### 1. Q: 训练时遇到过 loss 震荡吗？怎么定位是数据问题还是模型问题？

A: 定位流程分四步：
1. **固定数据**：用一小批固定数据反复训练，若仍有震荡则排除数据问题
2. **检查梯度**：打印梯度范数，若梯度爆炸（范数持续增大）则是模型/超参问题
3. **学习率消融**：降低学习率 10 倍，若震荡消失则是学习率问题
4. **数据消融**：分模块验证数据集，找出导致震荡的子集

**判断标准**：
- 固定数据仍震荡 → 模型/超参问题
- 固定数据不震荡 → 数据分布问题
- 降低 LR 后震荡消失 → 学习率问题
- 降低 LR 后仍震荡 → 模型结构或数值问题

### 2. Q: Softmax 稳定性怎么保证？

A: 三个层面保证：

1. **max-trick（必须）**：
```python
# 减去最大值，避免 exp 溢出
x = x - x.max(dim=-1, keepdim=True)
exp_x = torch.exp(x)
softmax = exp_x / exp_x.sum(dim=-1, keepdim=True)
```

2. **输入范围控制**：
   - LayerNorm/RMSNorm 后 logits 通常在 ±5 范围
   - 若 logits 过大，检查权重初始化和梯度爆炸

3. **混合精度处理**：
   - Softmax 计算保持 FP32
   - DeepSpeed/PyTorch AMP 自动处理

**为什么 max-trick 有效**：
```
softmax(x_i) = exp(x_i) / sum(exp(x_j))
             = exp(x_i - max) / sum(exp(x_j - max))  # 分子分母同乘 exp(-max)
             = softmax(x_i - max)  # 数值等价但更稳定
```

### 3. Q: 出现 NaN/inf 先查什么？

A: 按优先级排查（从快到慢）：

**第一优先：数值层面（5 分钟定位）**
```python
# 1. 检查是否开启 AMP 但未正确使用
with torch.cuda.amp.autocast():
    output = model(input)  # 确认 autocast 正确包裹

# 2. 检查 loss scaling
print(f"Loss scale: {scaler.get_scale()}")

# 3. 检查输入数据
print(f"Input: min={input.min()}, max={input.max()}, isnan={torch.isnan(input).any()}")

# 4. 检查梯度
for name, p in model.named_parameters():
    if p.grad is not None and torch.isnan(p.grad).any():
        print(f"NaN in gradient: {name}")
```

**第二优先：数据层面（15 分钟）**
- 检查是否有 `nan` 或 `inf` 样本
- 检查 label 是否越界（如 label > vocab_size）
- 检查是否有空序列或超长序列

**第三优先：模型层面（30 分钟）**
- 禁用部分层，二分定位
- 检查权重初始化
- 检查是否有除零操作（如 LayerNorm 方差为 0）

### 4. Q: 混合精度训练时 loss scale 怎么调？

A: 自动调整策略 + 手动干预：

**自动调整（推荐）**：
```python
scaler = torch.cuda.amp.GradScaler(
    init_scale=65536,      # 初始值 2^16
    growth_factor=2.0,     # 成功时翻倍
    backoff_factor=0.5,    # 失败时减半
    growth_interval=2000   # 2000 步未失败则增长
)
```

**手动调整信号**：
- `loss_scale` 频繁回退 → init_scale 太大
- `loss_scale` 持续增长不回退 → 可以增大 init_scale 加速
- 梯度频繁为 0 → scale 太大导致反向传播溢出

**调试技巧**：
```python
# 监控 scale 变化
if step % 100 == 0:
    print(f"Step {step}: scale={scaler.get_scale()}")
    
# 如果 scale 频繁回退，打印梯度
if scaler.get_scale() < prev_scale:
    for name, p in model.named_parameters():
        if p.grad is not None:
            print(f"{name}: grad_max={p.grad.abs().max()}")
```

### 5. Q: 如何区分梯度爆炸和梯度消失？

A: 通过梯度统计判断：

```python
def diagnose_gradient(model):
    total_norm = 0
    max_grad = 0
    min_grad = float('inf')
    zero_count = 0
    
    for name, param in model.named_parameters():
        if param.grad is not None:
            grad_norm = param.grad.norm().item()
            total_norm += grad_norm ** 2
            max_grad = max(max_grad, param.grad.abs().max().item())
            min_grad = min(min_grad, param.grad.abs().min().item())
            if grad_norm < 1e-8:
                zero_count += 1
    
    total_norm = total_norm ** 0.5
    
    # 梯度爆炸：范数持续 > 100 或出现 inf
    # 梯度消失：大量参数梯度 < 1e-7 或 min_grad 接近 0
    
    print(f"Total norm: {total_norm:.4f}")
    print(f"Max grad: {max_grad:.6f}")
    print(f"Min grad: {min_grad:.2e}")
    print(f"Near-zero params: {zero_count}/{len(list(model.parameters()))}")
```

**判断标准**：
| 现象 | 梯度爆炸 | 梯度消失 |
|------|----------|----------|
| 总范数 | > 100 或 inf | < 0.01 |
| 最大值 | > 1000 | < 1e-6 |
| 最小值 | 正常 | ≈ 0 |
| 近零参数 | < 10% | > 50% |
| Loss 表现 | 突然变 inf 或 NaN | 收敛极慢或停滞 |

### 6. Q: 学习率 warmup 为什么能稳定训练？

A: 三个原因：

1. **优化器状态预热**：Adam 的一阶/二阶矩估计需要足够样本才能稳定
   ```
   m_t = β₁ * m_{t-1} + (1-β₁) * g_t
   v_t = β₂ * v_{t-1} + (1-β₂) * g_t²
   ```
   初期 m、v 未稳定，大学习率会导致剧烈波动

2. **避免初始化敏感**：权重初始值随机，初期梯度方向不可靠
   - Warmup 期间小步探索，逐渐找到可靠优化方向
   - 之后正常学习率快速收敛

3. **配合 batch size 变化**：大 batch 训练初期样本多样性高
   - 梯度方差大，需要小学习率平滑
   - 典型配置：warmup_steps = 总步数的 1-3%

**经验值**：
- 7B 模型：warmup 2000 steps
- 70B 模型：warmup 2000-5000 steps
- 微调阶段：warmup 100-500 steps

### 7. Q: 为什么有的层容易产生 NaN？

A: 敏感层及原因：

| 层类型 | 易产生 NaN 原因 | 解决方案 |
|--------|----------------|----------|
| **Softmax** | exp 溢出 | max-trick + FP32 计算 |
| **LayerNorm** | 方差为 0 时除零 | 加 eps=1e-5 + FP32 计算 |
| **CrossEntropy** | label 错误 / logits 过大 | 检查 label 范围 + label smoothing |
| **Embedding** | index 越界 | 检查 input_ids < vocab_size |
| **Attention** | QK^T 数值过大 | 缩放 √d + FP32 softmax |
| **深层网络** | 梯度消失/爆炸 | 梯度裁剪 + residual + pre-norm |

**排查优先级**：
1. Attention softmax（最先检查）
2. LayerNorm/RMSNorm
3. CrossEntropy loss
4. Embedding lookup
5. FFN 激活函数

## 常见错误（至少 3 个）

### 1. 错误：直接在 logits 上计算 cross entropy 未检查范围
**描述**：CrossEntropyLoss 内部会做 softmax，若 logits 过大仍会溢出
**正确做法**：
```python
# 错误：logits 过大时直接计算
loss = F.cross_entropy(logits, labels)

# 正确：检查 logits 范围
logits_max = logits.max().item()
if logits_max > 50:
    print(f"Warning: logits max = {logits_max}, may cause overflow")
    # 可能需要检查权重初始化或学习率

# CrossEntropy 内部已做 log_softmax 数值稳定处理，但仍需关注输入范围
```

### 2. 错误：混合精度训练时未正确使用 autocast
**描述**：autocast 未包裹前向传播，导致数值问题
**正确做法**：
```python
# 错误：autocast 未正确包裹
output = model(input)
with torch.cuda.amp.autocast():
    loss = criterion(output, labels)  # 只有 loss 计算 autocast

# 正确：整个前向传播包裹
with torch.cuda.amp.autocast():
    output = model(input)
    loss = criterion(output, labels)
    
# 正确：训练循环
with torch.cuda.amp.autocast():
    output = model(input)
    loss = criterion(output, labels)

scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
```

### 3. 错误：梯度裁剪放在 backward 之前
**描述**：反向传播前梯度还未计算，裁剪无效
**正确做法**：
```python
# 错误顺序
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
loss.backward()  # 梯度在这之后才计算
optimizer.step()

# 正确顺序
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```

### 4. 错误：忽略 DeepSpeed 中的 loss scaling 自动调整日志
**描述**：loss scale 频繁回退但未注意到，导致训练不稳定
**正确做法**：
```python

if step % 100 == 0:
    scale = scaler.get_scale()
    print(f"Step {step}: loss_scale={scale}")
    
    # 如果 scale 频繁下降，可能是：
    # 1. 学习率过大
    # 2. 数据异常
    # 3. 模型不稳定
```

### 5. 错误：数据清洗时只检查文本，忽略 label 和 input_ids 越界
**描述**：label 值超过 vocab_size 或 input_ids 为负数
**正确做法**：
```python
def validate_batch(batch, vocab_size, pad_token_id):
    input_ids = batch['input_ids']
    labels = batch['labels']
    
    # 检查 input_ids
    assert input_ids.min() >= 0, f"Negative input_id: {input_ids.min()}"
    assert input_ids.max() < vocab_size, f"input_id overflow: {input_ids.max()} >= {vocab_size}"
    
    # 检查 labels
    assert labels.min() >= -100, f"Invalid label: {labels.min()}"  # -100 is ignore index
    valid_labels = labels[labels != -100]
    if len(valid_labels) > 0:
        assert valid_labels.max() < vocab_size, f"label overflow: {valid_labels.max()}"
    
    # 检查是否全为 pad
    if (input_ids == pad_token_id).all():
        raise ValueError("All tokens are padding")
```
