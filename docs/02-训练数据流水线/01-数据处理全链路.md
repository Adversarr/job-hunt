# 数据处理全链路

## 一句话结论

数据处理是 LLM 训练的基石，核心是"清洗 → 结构化 → 配比"三步走：去重去脏保证质量，统一格式（instruction/input/output + role 标记）适配训练框架，按任务难度和能力域科学配比避免过拟合和遗忘，Qwen 微调实践中常用 10-100 条高质量 instruction 数据即可显著提升指令遵循能力。

## 核心定义/公式

class DataCleaningPipeline:
    """数据清洗标准流程"""
    
    def __init__(self):
        self.min_length = 10          # 最小 token 数
        self.max_length = 8192         # 最大 token 数
        self.duplicate_threshold = 0.8  # 去重阈值（Jaccard 相似度）
        self.sensitive_words = [...]    # 脏词列表
        
    def clean(self, samples: List[Dict]) -> List[Dict]:
        """
        标准清洗流程：
        1. 格式校验
        2. 长度过滤
        3. 去重（MinHash + LSH）
        4. 脏词过滤
        5. 编码修复

### 数据配比公式

```python
# 任务能力域配比策略
class DataMixer:
    def calculate_weights(self, datasets: Dict[str, List]) -> Dict[str, float]:
        """
        能力域配比原则：
        1. 核心能力（指令遵循）: 40-50%
        2. 通用知识: 20-30%
        3. 专业领域: 10-20%
        4. 安全对齐: 5-10%
        """
        total = sum(len(v) for v in datasets.values())
        weights = {}
        
        for task_type, data in datasets.items():
            # 核心能力优先
            if task_type in ["instruction_following", "reasoning"]:
                weights[task_type] = 0.45 * (len(data) / total)
            # 通用知识保底
            elif task_type == "general_knowledge":
                weights[task_type] = 0.25 * (len(data) / total)
            # 专业领域按需
            elif task_type == "domain_specific":
                weights[task_type] = 0.20 * (len(data) / total)
            # 安全对齐
            elif task_type == "safety":
                weights[task_type] = 0.10 * (len(data) / total)
                
        # 归一化
        weight_sum = sum(weights.values())
        return {k: v/weight_sum for k, v in weights.items()}
    
    def dynamic_sampling(self, weights: Dict[str, float], 
                         epoch: int, total_epochs: int):
        """
        动态调整策略：
        - 前期：重基础能力，轻难样本
        - 中期：均衡采样
        - 后期：增加难样本权重
        """
        difficulty_scale = min(1.0, epoch / (total_epochs * 0.5))
        # 逐步增加难样本
        if "hard_samples" in weights:
            weights["hard_samples"] *= (1 + difficulty_scale)
        return weights
```

## 为什么（2-3 个因果链）

### 1. 数据质量决定模型上限

**因果链**：脏数据/重复数据 → 模型记住噪声 → 泛化能力下降 → 下游任务性能退化

- **去重必要性**：重复样本会导致模型过拟合特定模式，降低对未见数据的泛化能力。N-gram 去重和语义去重（MinHash + LSH）可将重复率从 30% 降至 < 1%。
- **脏词过滤**：训练数据中的错误标注、低质量回答会污染模型，导致幻觉和安全问题。清洗后数据质量提升可使 C-Eval 平均分提升 3-5 分。
- **Test Contamination**：训练集包含测试集样本会导致虚高评估分数，实际部署效果差。使用 N-gram 匹配可检测 95%+ 的泄露样本。

### 2. 结构化格式统一训练框架

**因果链**：格式不一致 → 需要额外适配代码 → 增加训练复杂度 → 易出 Bug 且难复现

- **统一模板**：采用 `instruction/input/output` + `history` + `system` 的标准格式，适配 HuggingFace Transformers、DeepSpeed、vLLM 等主流框架，减少 50%+ 的数据预处理代码。
- **Role 标记**：明确的 `system/user/assistant` 角色区分，让模型学习对话轮次和角色行为，对多轮对话场景至关重要（Qwen 在 MT-Bench 多轮对话分数提升 10%+）。
- **兼容性**：ChatML 格式被 Qwen、Baichuan、DeepSeek 等主流中文模型采用，可直接使用 `tokenizer.apply_chat_template()`，无需手动拼接。

### 3. 数据配比平衡能力与稳定性

**因果链**：配比不合理 → 某类任务过拟合 → 其他能力退化 → 模型综合性能下降

- **能力覆盖**：单一任务类型数据占比 > 60% 会导致灾难性遗忘（Catastrophic Forgetting），其他能力下降 20-40%。科学配比（核心 40-50%，通用 20-30%，专业 10-20%，安全 5-10%）可保持各能力均衡。
- **难度梯度**：早期训练集中简单样本可加速收敛，后期增加难样本（长上下文、推理、多步任务）可突破性能瓶颈。
- **样本多样性**：同质化数据（如大量相似指令）会让模型学会"套路"而非理解本质，数据增强（改写、反向翻译、模板化生成）可提升鲁棒性。

# 1. 长度过滤
def filter_length(samples: List[Dict], 
                   min_len: int = 10, 
                   max_len: int = 8192) -> List[Dict]:
    """过滤异常长度样本"""
    tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2-7B")
    filtered = []
    for sample in samples:
        text = sample["instruction"] + sample.get("input", "") + sample["output"]
        tokens = tokenizer.encode(text)
        if min_len <= len(tokens) <= max_len:
            filtered.append(sample)
    return filtered

# 2. 去重（MinHash + LSH）
from datasketch import MinHash, MinHashLSH

def deduplicate(samples: List[Dict], 
                threshold: float = 0.8) -> List[Dict]:
    """基于 MinHash 的近似去重"""
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    unique_samples = []
    
    for idx, sample in enumerate(samples):
        text = sample["instruction"] + sample["output"]
        
        # 生成 MinHash
        mh = MinHash(num_perm=128)
        for ngram in get_ngrams(text, n=3):
            mh.update(ngram.encode('utf8'))
        
        # 查询相似样本
        result = lsh.query(mh)
        if not result:  # 无重复
            lsh.insert(idx, mh)
            unique_samples.append(sample)
    
    return unique_samples

# 3. 脏词与敏感内容过滤
def filter_sensitive(samples: List[Dict], 
                     sensitive_words: List[str]) -> List[Dict]:
    """过滤包含脏词的样本"""
    filtered = []
    for sample in samples:
        text = sample["instruction"] + sample["output"]
        if not any(word in text for word in sensitive_words):
            filtered.append(sample)
    return filtered

# 4. 编码修复
def fix_encoding(samples: List[Dict]) -> List[Dict]:
    """修复编码问题"""
    for sample in samples:
        for key in ["instruction", "input", "output"]:
            if key in sample:
                # 修复乱码
                sample[key] = sample[key].encode('utf-8', errors='ignore').decode('utf-8')
                # 统一换行符
                sample[key] = sample[key].replace('\r\n', '\n').replace('\r', '\n')
                # 移除不可见字符
                sample[key] = ''.join(c for c in sample[key] if c.isprintable() or c in '\n\t')
    return samples

# 5. Test Contamination 检测
def remove_test_leak(samples: List[Dict], 
                     test_benchmarks: List[str]) -> List[Dict]:
    """移除测试集泄露样本"""
    from transformers import AutoTokenizer
    
    tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2-7B")
    clean_samples = []
    
    for sample in samples:
        text = sample["instruction"] + sample["output"]
        is_leak = False
        
        for benchmark_path in test_benchmarks:
            # 加载测试集
            test_data = load_json(benchmark_path)
            for test_sample in test_data:
                # N-gram 重叠检测
                if has_high_overlap(text, test_sample["question"], n=8, threshold=0.8):
                    is_leak = True
                    break
            if is_leak:
                break
        
        if not is_leak:
            clean_samples.append(sample)
    
    return clean_samples

```python
# Qwen 微调实践中的配比策略
class QwenDataMixer:
    def __init__(self):
        self.task_weights = {
            "instruction_following": 0.35,  # 指令遵循（核心）
            "reasoning": 0.15,              # 推理能力
            "knowledge_qa": 0.20,           # 知识问答
            "code": 0.10,                  # 代码能力
            "math": 0.10,                  # 数学能力
            "safety": 0.05,                # 安全对齐
            "multilingual": 0.05          # 多语言
        }
        
    def mix_datasets(self, 
                     datasets: Dict[str, List],
                     total_samples: int = 10000) -> List[Dict]:
        """按权重混合数据集"""
        mixed_data = []
        
        for task_type, weight in self.task_weights.items():
            if task_type not in datasets:
                continue
            
            task_data = datasets[task_type]
            n_samples = int(total_samples * weight)
            
            # 按需采样（可重复采样）
            sampled = random.choices(task_data, k=n_samples)
            mixed_data.extend(sampled)
        
        # 打乱顺序
        random.shuffle(mixed_data)
        return mixed_data
    
    def curriculum_sampling(self,
                           datasets: Dict[str, List],
                           current_epoch: int,
                           total_epochs: int):
        """
        课程学习：逐步增加难度
        - Epoch 1-2: 简单样本（短指令、单轮对话）
        - Epoch 3-5: 中等样本（推理、多轮）
        - Epoch 6+: 困难样本（长上下文、复杂推理）
        """
        if current_epoch <= 2:
            # 简单样本：过滤掉长文本和复杂推理
            return self.filter_difficulty(datasets, max_length=512, complexity="simple")
        elif current_epoch <= 5:
            # 中等难度
            return self.filter_difficulty(datasets, max_length=2048, complexity="medium")
        else:
            # 全量样本，增加困难样本权重
            return self.upweight_hard_samples(datasets, factor=1.5)
    
    def balance_single_multi_turn(self, 
                                   datasets: Dict[str, List]) -> Dict:
        """
        单轮/多轮对话配比策略：
        - 单轮对话: 60-70%（快速学习指令遵循）
        - 多轮对话: 30-40%（学习上下文理解）
        """
        single_turn = []
        multi_turn = []
        
        for task_type, data in datasets.items():
            for sample in data:
                if "history" in sample and len(sample["history"]) > 0:
                    multi_turn.append(sample)
                else:
                    single_turn.append(sample)
        
        # 按比例采样
        n_single = int(len(single_turn) * 0.65)
        n_multi = int(len(multi_turn) * 0.35)
        
        return {
            "single_turn": random.sample(single_turn, min(n_single, len(single_turn))),
            "multi_turn": random.sample(multi_turn, min(n_multi, len(multi_turn)))
        }
```

# 1. 自动质量评估
def quality_check(samples: List[Dict]) -> Dict:
    """数据质量评估"""
    tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2-7B")
    
    metrics = {
        "avg_instruction_len": 0,
        "avg_output_len": 0,
        "empty_rate": 0,
        "duplicate_rate": 0,
        "format_error_rate": 0
    }
    
    instruction_lens = []
    output_lens = []
    empty_count = 0
    format_errors = 0
    seen = set()
    duplicates = 0
    
    for sample in samples:
        # 格式检查
        if not validate_format(sample):
            format_errors += 1
            continue
        
        # 长度统计
        inst_len = len(tokenizer.encode(sample["instruction"]))
        out_len = len(tokenizer.encode(sample["output"]))
        instruction_lens.append(inst_len)
        output_lens.append(out_len)
        
        # 空值检查
        if not sample["instruction"].strip() or not sample["output"].strip():
            empty_count += 1
        
        # 去重检查
        text_hash = hash(sample["instruction"] + sample["output"])
        if text_hash in seen:
            duplicates += 1
        seen.add(text_hash)
    
    n = len(samples)
    metrics["avg_instruction_len"] = np.mean(instruction_lens)
    metrics["avg_output_len"] = np.std(output_lens)
    metrics["empty_rate"] = empty_count / n
    metrics["duplicate_rate"] = duplicates / n
    metrics["format_error_rate"] = format_errors / n
    
    return metrics

### 关键配置/参数

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| 最小长度 | 10 tokens | 过短样本噪声大，无法学到有效模式 |
| 最大长度 | 4096-8192 tokens | 超过模型窗口会被截断，浪费计算；需匹配模型窗口 |
| 去重阈值 | 0.8 (Jaccard) | 平衡去重效果和召回率，避免误删相似但不同的样本 |
| 单轮/多轮比例 | 65%/35% | 单轮建立基础指令遵循，多轮提升上下文理解 |
| 核心能力占比 | 40-50% | 保证指令遵循等核心能力，防止灾难性遗忘 |
| 困难样本比例 | 前期 10%，后期 30% | 课程学习：逐步增加难度，加速收敛且避免早期崩溃 |

                sample["messages"],
                tokenize=False,
                add_generation_prompt=False
            )
            
            # Tokenize
            tokens = self.tokenizer(
                formatted_text,
                max_length=4096,
                truncation=True,
                padding=False,
                return_tensors=None
            )
            
            # 标签 = 输入移位（下一个 token 预测）

            # Loss mask: 只计算 assistant 回复的 loss

            assistant_start = formatted_text.find("<|im_start|>assistant\n")
            assistant_prefix_len = len("<|im_start|>assistant\n")
            
            # 计算 assistant 开始的 token 位置
            prefix_text = formatted_text[:assistant_start + assistant_prefix_len]
            prefix_tokens = self.tokenizer.encode(prefix_text, add_special_tokens=False)
            start_idx = len(prefix_tokens)
            
            # 前面的 token 不计算 loss（设为 -100）
            tokens["labels"][:start_idx] = [-100] * start_idx
            
            tokenized_data.append(tokens)
        
        # Step 4: 转换为 Dataset
        dataset = Dataset.from_list(tokenized_data)
        
        return dataset

            "avg_length": np.mean(lengths),
            "max_length": max(lengths),
            "min_length": min(lengths),
            "std_length": np.std(lengths)
        }

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **严格去重（阈值 0.95）** | 彻底去除重复，提升泛化 | 可能误删相似但不同的样本（如同一问题的不同表述） | 数据量充足（> 100K 样本）时适用；小数据集用宽松阈值（0.8） |
| **宽松去重（阈值 0.7）** | 保留更多样本，召回率高 | 仍存在部分重复，可能过拟合 | 数据稀缺场景，需人工复核 |
| **全量人工清洗** | 质量最高，噪声最少 | 成本极高，耗时长（1000 样本约需 10 人时） | 小规模精品数据集（< 10K），对质量要求极高的场景 |
| **自动清洗 + 抽样质检** | 平衡成本与质量 | 可能遗漏部分噪声 | 大规模数据集（> 50K），工业界标准做法 |
| **只用单轮对话** | 收敛快，训练简单 | 缺失多轮上下文理解能力，MT-Bench 多轮对话分数低 15% | 初期快速验证，后续需补充多轮数据 |
| **单轮多轮均衡（50/50）** | 上下文理解强 | 训练慢，单轮指令遵循可能弱化 | 对话系统，需强多轮能力 |
| **固定配比训练** | 实现简单 | 可能配比不合理，后期难调整 | 数据分布均匀，任务边界清晰 |
| **动态课程学习配比** | 收敛更稳，最终性能优 | 实现复杂，需精细调参 | 大规模训练（> 10K 样本，> 10 epochs），有调优资源 |
| **合成数据为主** | 数据量可快速扩展 | 可能引入模型偏差，质量参差 | 领域数据稀缺，需人工校验 + 混合真实数据 |
| **真实数据为主** | 质量可靠，无合成偏差 | 数据量受限，成本高 | 有充足预算和标注资源，质量优先 |

## 高频追问（至少 5 个）

### 1. Q: 数据清洗时，如何平衡去重的严格程度？

A: 
- **策略**：根据数据规模调整阈值。数据充足（> 100K）用严格阈值（0.9-0.95），数据稀缺（< 10K）用宽松阈值（0.7-0.8）。
- **实践**：先用宽松阈值去重，再用人工抽样复核相似样本，判断是否真重复。
- **指标**：监控验证集性能，若训练集 loss 远低于验证集，说明过拟合，需加强去重。

### 2. Q: 多轮对话的 loss 应该怎么计算？

A: 
- **方案 1（推荐）**：只计算 assistant 回复部分的 loss，user 输入部分设为 -100（ignore index）。这样模型只学习如何生成回答，不学习模仿用户输入。
- **方案 2**：计算全部 token 的 loss，但可能导致模型学习生成 user 风格文本。
- **方案 3**：加权计算，assistant 部分权重高，user 部分权重低。
- **Qwen 实践**：采用方案 1，只计算 assistant token 的 loss，user 和 system 部分 mask 掉。

### 3. Q: 数据配比不合理会有什么问题？怎么诊断？

A: 
- **问题表现**：
  1. 某类任务过拟合：训练集分数高，测试集分数低。
  2. 灾难性遗忘：新任务数据占比过高，旧能力退化（如推理数据过多，指令遵循变差）。
  3. 收敛震荡：数据分布不均，梯度方向不一致。
  
- **诊断方法**：
  1. 分任务评估：在 C-Eval、GSM8K、HumanEval 等不同 benchmark 上分别测试，看能力是否均衡。
  2. 训练曲线分析：观察各任务 loss 曲线，若某类 loss 长期不降，说明数据不足或配比过低。
  3. 梯度分析：计算各任务数据的梯度方向，若冲突严重（余弦相似度 < 0），说明任务干扰，需调整配比或分组训练。

### 4. Q: Test Contamination 怎么检测？如何避免？

A: 
- **检测方法**：
  1. **N-gram 重叠**：提取测试集的 8-gram，在训练集中搜索，重叠率 > 80% 判定为泄露。
  2. **语义相似度**：用 embedding 模型计算训练样本与测试样本的相似度，cos > 0.95 判定为疑似泄露。
  3. **模型行为异常**：训练集准确率远高于验证集，或测试集分数虚高但实际表现差。
  
- **避免策略**：
  1. **严格隔离**：训练数据收集阶段排除所有公开测试集来源。
  2. **定期扫描**：在数据清洗阶段加入 contamination 检测模块，自动移除泄露样本。
  3. **留出验证集**：从训练数据中分出一部分做验证，避免用公开测试集做早停。

### 5. Q: 合成数据（如 Self-Instruct）的质量如何保证？

A: 
- **质量保证措施**：
  1. **多样性验证**：计算合成数据的词汇多样性、指令类型分布，避免同质化。
  2. **人工抽样质检**：随机抽取 5-10% 样本人工评估，质量分 < 3/5 的批次废弃。
  3. **混合真实数据**：合成数据占比不超过 50%，与人工标注数据混合训练。
  4. **过滤低质量样本**：用奖励模型或 GPT-4 对合成样本打分，过滤低分样本。
  5. **迭代优化**：第一轮合成后评估，分析失败案例，改进 prompt 模板，再次合成。

- **Qwen 实践**：Self-Instruct 生成 50K 初始数据，人工筛选 + GPT-4 打分后保留 10K 高质量样本，再混合人工标注数据训练。

### 6. Q: 如何处理数据中的错误标注？

A: 
- **自动检测**：
  1. **规则过滤**：输出长度异常（< 5 字或 > 2000 字）、格式错误（如要求 JSON 但输出文本）、截断（结尾不完整）。
  2. **模型打分**：用已训练的 reward model 或 GPT-4 对 instruction-output 对打分，过滤低分样本（< 2/5）。
  3. **一致性检查**：同一 instruction 多次采样，若输出差异大，说明标注不稳定。
  
- **修复策略**：
  1. **重标注**：对低质量样本人工重标。
  2. **改写修复**：用模型改写低质量输出（如 GPT-4 rewrite）。
  3. **直接丢弃**：若错误率 > 10%，直接丢弃该批次。

### 7. Q: 数据清洗的优先级顺序是什么？

A: 
- **推荐顺序**：
  1. **格式校验**（最高优先级）：确保必需字段存在，格式正确，否则后续处理会报错。
  2. **长度过滤**：过长样本会被截断浪费计算，过短样本噪声大。
  3. **去重**：避免过拟合重复样本，优先级高于内容清洗。
  4. **脏词与敏感内容**：避免安全问题和模型污染。
  5. **Test Contamination**：避免评估虚高，但成本较高，可放在后期。
  6. **质量打分**：精细化清洗，可迭代进行。

- **理由**：先快速过滤明显问题（格式、长度），再精细清洗（去重、敏感词），最后高成本检测，逐步提升质量。

## 常见错误（至少 3 个）

### 1. 错误：去重时只比较原文，忽略了语义重复

**描述**：使用简单的字符串匹配或哈希去重，导致语义相同但表述不同的样本（如"请解释量子计算"和"帮我讲讲量子计算是什么"）仍被保留。

**后果**：
- 模型学到重复模式，泛化能力下降。
- 浪费训练计算资源。

**正确做法**：
```python
# 错误做法：只用字符串哈希
def bad_dedup(samples):
    seen = set()
    unique = []
    for s in samples:
        text = s["instruction"] + s["output"]
        if text not in seen:
            seen.add(text)
            unique.append(s)
    return unique

# 正确做法：MinHash + LSH 语义去重
from datasketch import MinHash, MinHashLSH

def correct_dedup(samples, threshold=0.8):
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    unique = []
    
    for idx, s in enumerate(samples):
        text = s["instruction"] + s["output"]
        mh = MinHash(num_perm=128)
        for ngram in get_ngrams(text, n=3):
            mh.update(ngram.encode('utf8'))
        
        if not lsh.query(mh):
            lsh.insert(idx, mh)
            unique.append(s)
    
    return unique
```

### 2. 错误：多轮对话 loss 计算包含 user 部分

**描述**：将整个对话（user + assistant）都作为训练目标，导致模型学习生成 user 风格的文本。

**后果**：
- 模型可能学会"自问自答"，生成 user 输入。
- 浪费计算资源在无意义的 user token 上。

**正确做法**：
```python
# 错误做法：计算全部 token 的 loss
def bad_loss_computation(tokens):
    tokens["labels"] = tokens["input_ids"].copy()  # 全部计算 loss
    return tokens

# 正确做法：只计算 assistant 部分
def correct_loss_computation(tokens, formatted_text, tokenizer):
    # 找到 assistant 开始位置
    assistant_start = formatted_text.find("<|im_start|>assistant\n")
    prefix_len = len("<|im_start|>assistant\n")
    
    # 计算 token 索引
    prefix_text = formatted_text[:assistant_start + prefix_len]
    prefix_tokens = tokenizer.encode(prefix_text, add_special_tokens=False)
    start_idx = len(prefix_tokens)
    
    # mask 掉前面的 token
    tokens["labels"] = tokens["input_ids"].copy()
    tokens["labels"][:start_idx] = [-100] * start_idx  # -100 = ignore index
    
    return tokens
```

### 3. 错误：数据配比完全按样本数量，忽略任务难度

**描述**：简单按样本数均分（如每个任务 1000 条），导致简单任务过拟合，困难任务欠拟合。

**后果**：
- 简单任务（如单轮问答）训练集满分，测试集也高，但困难任务（如长程推理、代码生成）性能差。
- 模型能力不均衡。

**正确做法**：
```python
# 错误做法：按样本数均分
def bad_mix(datasets):
    n_per_task = 1000
    mixed = []
    for data in datasets.values():
        mixed.extend(random.sample(data, n_per_task))
    return mixed

# 正确做法：按任务难度和能力域加权
def correct_mix(datasets):
    # 定义任务难度权重
    task_weights = {
        "simple_qa": 0.2,        # 简单问答，不需要复杂推理
        "reasoning": 0.25,       # 推理任务，较难
        "code": 0.20,            # 代码生成，难
        "math": 0.15,            # 数学推理，难
        "safety": 0.10,          # 安全对齐
        "multiturn": 0.10        # 多轮对话
    }
    
    total_samples = 10000
    mixed = []
    
    for task, weight in task_weights.items():
        if task in datasets:
            n_samples = int(total_samples * weight)
            sampled = random.choices(datasets[task], k=n_samples)
            mixed.extend(sampled)
    
    random.shuffle(mixed)
    return mixed
```

### 4. 错误：忽视数据中的截断和格式错误

**描述**：直接使用未经清洗的原始数据，包含截断的输出、格式错误的 JSON、编码乱码等。

**后果**：
- 模型学习生成不完整或格式错误的输出。
- 训练时 loss 震荡或 NaN。

**正确做法**：
```python
# 错误做法：直接使用原始数据
raw_data = load_json("raw_data.json")
train_dataset = Dataset.from_list(raw_data)

# 正确做法：格式校验 + 清洗
def validate_and_clean(sample):
    # 1. 必需字段检查
    if "instruction" not in sample or "output" not in sample:
        return None
    
    # 2. 空值检查
    if not sample["instruction"].strip() or not sample["output"].strip():
        return None
    
    # 3. 截断检查（输出结尾不完整）
    if sample["output"].endswith("...") or len(sample["output"]) < 10:
        return None
    
    # 4. 格式检查（如要求 JSON 输出）
    if sample.get("format") == "json":
        try:
            json.loads(sample["output"])
        except:
            return None
    
    # 5. 编码修复
    for key in ["instruction", "output"]:
        sample[key] = sample[key].encode('utf-8', errors='ignore').decode('utf-8')
    
    return sample

clean_data = [validate_and_clean(s) for s in raw_data]
clean_data = [s for s in clean_data if s is not None]
train_dataset = Dataset.from_list(clean_data)
```

### 5. 错误：训练集包含验证集样本

**描述**：数据收集时未排除公开 benchmark 的测试集，导致评估虚高。

**后果**：
- 模型在 benchmark 上分数虚高，实际部署效果差。
- 无法客观评估模型真实能力。

**正确做法**：
```python
# 错误做法：直接混合所有数据
all_data = load_json("all_data.json")  # 包含公开数据集
train_data = all_data

# 正确做法：先检测泄露，再训练
from transformers import AutoTokenizer

def detect_contamination(train_data, test_benchmarks):
    tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2-7B")
    clean_data = []
    
    for sample in train_data:
        text = sample["instruction"] + sample["output"]
        is_leak = False
        
        for benchmark in test_benchmarks:
            for test_sample in benchmark:
                # 8-gram 重叠检测
                if has_high_ngram_overlap(text, test_sample, n=8, threshold=0.8):
                    is_leak = True
                    break
            
            if is_leak:
                break
        
        if not is_leak:
            clean_data.append(sample)
    
    return clean_data

# 加载公开测试集
test_benchmarks = [
    load_json("c-eval_test.json"),
    load_json("gsm8k_test.json"),
    load_json("humaneval_test.json")
]

# 清洗
train_data = detect_contamination(all_data, test_benchmarks)
```
