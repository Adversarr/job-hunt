# LLM 算法面试知识库

## 一句话结论
本知识库采用 15 章 58 篇文档覆盖 LLM 面试全链路，统一模板+标签体系确保知识可检索、可关联、可复用。

## 核心定义/公式

知识库采用 **主题域（Domain）→ 页面（Page）→ 卡片（Cards）** 三层组织结构：

```python
# 文档结构定义
knowledge_base = {
    "total_chapters": 15,
    "total_documents": 58,
    "companies": ["字节", "阿里", "腾讯", "美团", "百度"],
    "question_types": ["handwrite", "derive", "leetcode", "system_design"]
}

# 标签体系（强制执行）
tag_system = {
    "train": ["#pretrain", "#SFT", "#DPO", "#PPO", "#GRPO", "#RM", "#RLHF"],
    "architecture": ["#Transformer", "#MHA", "#GQA", "#MoE", "#RoPE", "#YaRN"],
    "engineering": ["#FlashAttention", "#DeepSpeed", "#ZeRO", "#3DParallel", "#TP", "#PP", "#DP"],
    "inference": ["#prefill", "#decode", "#kv_cache", "#paged_attention", "#spec_decode"],
    "numeric": ["#fp16", "#bf16", "#int8", "#int4", "#quant"],
    "type": ["#handwrite", "#derive", "#leetcode", "#system_design"],
    "company": ["#字节", "#阿里", "#腾讯", "#美团", "#百度"]
```

## 为什么（2-3 个因果链）

1. **知识碎片化 → 无法系统复习 → 面试效率低**：
   - 传统面试准备依赖零散博客/论文，知识点之间缺乏关联
   - 本知识库通过主题域划分，确保相关知识点集中，便于对比学习

2. **标准缺失 → 内容质量参差 → 难以评估覆盖度**：
   - 不同来源的面试资料格式不统一，深度不一致
   - 本知识库强制执行统一模板（11 个核心章节）+ 标签体系，确保每个主题都有完整回答

3. **问题分散 → 无法快速定位 → 临场检索慢**：
   - 真实面试问题来自多个公司，分散在不同平台
   - 本知识库建立 `questions.md → docs/` 映射，通过标签和覆盖索引实现快速定位

## 怎么做（可落地步骤）

### 标准使用流程

1. **快速查找问题**：
   - 按 company 标签检索：`#字节` → 查看 DPO、FlashAttention 相关题
   - 按章节浏览：第 4 章第 3 节 → DPO 算法详解
   - 按问题索引：`15-覆盖索引/README.md` → 问题到文档映射

2. **准备面试话题**：
   - 阅读"一句话结论" → 15 秒快速复习
   - 理解"为什么" → 应对原理追问
   - 掌握"怎么做" → 准备工程细节
   - 记忆"高频追问" → 预判追问场景
   - 避免"常见错误" → 规避典型坑

3. **构建知识关联**：
   - 查看文档末尾"相关文档" → 发现关联主题
   - 通过标签跳转 → 同类主题对比学习

### 关键配置/参数

- **必考章节**：第 1、2、3 章（Transformer、训练流水线、SFT）

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **统一模板** | 确保每个主题都有完整回答，可检索 | 灵活性降低，某些主题可能不需要所有章节 | 大部分面试主题适用 |
| **标签体系** | 快速定位相关主题，公司/题型交叉检索 | 需要强制执行，初期文档需校验 | 中大规模知识库必须 |
| **覆盖索引** | 问题到文档快速映射，验证覆盖度 | 维护成本高，需持续更新 | 面试准备场景必选 |

## 高频追问（至少 5 个）

1. **Q: 这个知识库的核心价值是什么？**
   A: 统一模板+标签体系确保知识可检索、可关联，覆盖真实面试题，避免知识碎片化。

   A: 7 大类标签：train（训练）、architecture（架构）、engineering（工程）、inference（推理）、numeric（数值）、type（题型）、company（公司），强制执行确保可检索。

   A: 确保每个主题都能回答：是什么（核心定义）、为什么（原理）、怎么做（实践）、权衡（系统思维）、常见错误（避坑），覆盖面试全流程。

   A: 基于 `questions.md` 持续收录新题，通过 AGENTS.md 定义生成规范，新增文档自动映射到章节和标签体系。

## 常见错误（至少 3 个）

1. **错误：不看标签体系，随意添加新标签**
   正确做法：严格遵循本文档定义的 7 大类标签，新增标签需在 AGENTS.md 中更新标准。

2. **错误：只看"一句话结论"，忽略"为什么"和"怎么做"**
   正确做法：面试前通读完整文档，理解原理和工程细节，"一句话结论"仅用于临场快速复习。

3. **错误：不看相关文档，知识点孤立学习**
   正确做法：查看文档末尾"相关文档"链接，构建知识关联，例如学习 DPO 时应关联 PPO、GRPO 文档。
