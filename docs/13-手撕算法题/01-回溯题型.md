# 回溯题型：括号生成

## 一句话结论

回溯 = 决策树 + 剪枝 + 回退，括号生成的核心是维护"左括号数 ≥ 右括号数"的合法性不变量，通过两个剪枝条件提前终止无效分支，时间复杂度为第 n 个卡特兰数 O(C_n)。

## 核心定义/公式

### 回溯算法框架

```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        result.append(路径)
        return
    
    for 选择 in 选择列表:
        # 剪枝：跳过无效选择
        if 不满足约束:
            continue
        
        做选择
        backtrack(路径, 选择列表)
        撤销选择
```

### 括号生成问题（LeetCode 22）

**题目**：给定 n，生成所有包含 n 对括号的有效组合。

**合法性不变量**：
- 任意前缀，左括号数 ≥ 右括号数
- 最终：左括号数 = 右括号数 = n

**剪枝条件**：
1. `left < n`：左括号数量不能超过 n
2. `right < left`：右括号数量不能超过已放置的左括号

**卡特兰数公式**：
$$
C_n = \frac{1}{n+1}\binom{2n}{n} = \frac{(2n)!}{(n+1)!n!}
$$

时间复杂度：O(C_n × n)，其中 C_n 是第 n 个卡特兰数
空间复杂度：O(n)（递归深度）

## 为什么（3 个因果链）

### 1. 为什么用回溯而非暴力枚举？

**现象**：2n 个位置，每个位置选左或右括号，暴力枚举 2^(2n) 种组合，但大部分不合法。

**根因**：
- 暴力枚举生成所有可能，包括无效组合
- 回溯在构建过程中就剪枝，避免进入无效分支
- 合法性不变量"左 ≥ 右"可以在 O(1) 内判断

**结果**：
- 暴力：O(4^n)（实际搜索空间），后处理过滤无效 → 浪费大量计算
- 回溯：O(C_n)，只生成有效组合，搜索空间减少 2 个数量级（n=4 时，C_4=14 vs 2^8=256）

### 2. 为什么剪枝条件是 `right < left` 而非其他？

**现象**：剪枝条件直接影响搜索效率和正确性。

**根因**：
- 有效性定义：任意前缀左括号数 ≥ 右括号数
- 若 `right >= left`，再放右括号会破坏不变量，产生无效序列
- `right < left` 确保右括号"有匹配的左括号"

**结果**：
- 正确剪枝：每个生成的序列都有效，无需后处理
- 错误剪枝（如只检查 `right < n`）：生成大量无效序列，如 `")(()"`

### 3. 为什么时间复杂度是卡特兰数？

**现象**：有效括号组合数量等于第 n 个卡特兰数。

**根因**：
- 卡特兰数的组合意义：合法括号序列计数
- 每个有效序列对应一个卡特兰路径（从 (0,0) 到 (n,n) 不越过对角线）
- 递归树叶子节点数 = 有效解数量 = C_n

**结果**：
- 时间：O(C_n × n)，每个解构造 O(n)
- 空间：O(n)，递归深度
- C_n ≈ 4^n/(n^(3/2)√π)，指数增长但远小于 2^(2n)

## 怎么做（可落地步骤）

### 标准回溯解法

```python
from typing import List

class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        result = []
        
        def backtrack(path: List[str], left: int, right: int):
            """
            path: 当前路径（已放置的括号序列）
            left: 已放置的左括号数
            right: 已放置的右括号数
            """
            # 结束条件：放置了 2n 个括号
            if len(path) == 2 * n:
                result.append(''.join(path))
                return
            
            # 选择 1：放左括号（剪枝：left < n）
            if left < n:
                path.append('(')
                backtrack(path, left + 1, right)
                path.pop()  # 撤销选择
            
            # 选择 2：放右括号（剪枝：right < left）
            if right < left:
                path.append(')')
                backtrack(path, left, right + 1)
                path.pop()  # 撤销选择
        
        backtrack([], 0, 0)
        return result
```

### 关键实现要点

1. **路径表示**：使用 `List[str]` 方便 append/pop，最后 `''.join()`
2. **参数设计**：
   - `left/right`：已放置数量，避免重复计算
   - 不传 `n`：外层闭包捕获
3. **剪枝位置**：在 `if` 条件中剪枝，而非进入递归后判断
4. **撤销操作**：必须与"做选择"对称，`pop()` 放在递归返回后

### 变体：返回第 k 个合法序列

```python
def generate_kth_parenthesis(n: int, k: int) -> str:
    """生成第 k 个合法括号序列（按字典序）"""
    result = []
    
    def count_sequences(left: int, right: int) -> int:
        """计算从 (left, right) 状态出发的合法序列数"""
        if left == n and right == n:
            return 1
        count = 0
        if left < n:
            count += count_sequences(left + 1, right)
        if right < left:
            count += count_sequences(left, right + 1)
        return count
    
    def backtrack(path: List[str], left: int, right: int, remaining_k: int):
        if len(path) == 2 * n:
            return remaining_k
        
        # 尝试放左括号
        if left < n:
            cnt = count_sequences(left + 1, right)
            if remaining_k <= cnt:
                path.append('(')
                return backtrack(path, left + 1, right, remaining_k)
            remaining_k -= cnt
        
        # 尝试放右括号
        if right < left:
            path.append(')')
            return backtrack(path, left, right + 1, remaining_k)
        
        return remaining_k
    
    backtrack(result, 0, 0, k)
    return ''.join(result)
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **标准回溯** | 代码简洁、易理解、只生成有效解 | 递归栈开销、n 较大时慢 | n ≤ 10（面试、竞赛） |
| **暴力枚举+过滤** | 实现简单，无需思考剪枝 | 时间 O(4^n)、大量无效计算 | 不推荐，仅作对比 |
| **动态规划** | 可预处理所有 C_n，查询 O(1) | 空间 O(n×C_n)、构造复杂 | 需要多次查询不同 n |
| **迭代法（BFS）** | 避免递归栈溢出 | 队列内存占用大、代码复杂 | n 很大时（>15） |
| **数学公式（卡特兰）** | 直接计数，不生成序列 | 只计数不生成具体组合 | 只需要数量时 |

## 高频追问（8 个）

### 1. Q: 为什么剪枝条件是 `right < left` 而不是 `right < n`？

**A**: 
- `right < n` 只保证右括号总数不超过 n，但无法保证局部合法性
- 例如 `")(()"` 满足 `right=2 < n=3`，但第一个 `)` 没有匹配的左括号，序列无效
- `right < left` 保证每放一个右括号时，前面至少有一个未匹配的左括号，维护"左 ≥ 右"不变量

### 2. Q: 时间复杂度为什么是卡特兰数？能口述证明吗？

**A**: 
- 合法括号序列与卡特兰路径一一对应（网格路径不越过对角线）
- 递归树的叶子节点数 = 有效解数量 = 第 n 个卡特兰数 C_n
- C_n = C(2n, n) / (n+1)，斯特林近似得 C_n ≈ 4^n / (n^(3/2) √π)
- 每个解构造 O(n)，总时间 O(C_n × n)

### 3. Q: 如果只要生成第 k 个序列（按字典序），如何优化？

**A**: 
- 预处理每个状态的子问题规模（从该状态出发的合法序列数）
- 对于当前选择，若 `k > 左子树规模`，则跳过左子树，进入右子树
- 时间 O(n²)（每个节点预处理）或 O(n)（预计算所有卡特兰数）

### 4. Q: n=10 时有多少个解？能在 1 秒内生成吗？

**A**: 
- C_10 = 16796 个解
- 生成时间 < 1ms（实测 Python 约 0.5ms）
- n=15 时 C_15=9694845，仍可在 < 1s 内完成
- n=20 时 C_20≈6.5×10^9，会超时（需要改用迭代生成或限制输出）

### 5. Q: 如何用迭代（非递归）实现？

**A**: 
```python
def generate_iterative(n: int) -> List[str]:
    """迭代法：使用栈模拟递归"""
    result = []
    stack = [('', 0, 0)]  # (路径, left, right)
    
    while stack:
        path, left, right = stack.pop()
        
        if len(path) == 2 * n:
            result.append(path)
            continue
        
        # 注意顺序：先压右，后压左（保证左优先遍历）
        if right < left:
            stack.append((path + ')', left, right + 1))
        if left < n:
            stack.append((path + '(', left + 1, right))
    
    return result
```
- 优点：避免递归栈溢出
- 缺点：显式栈管理，代码稍复杂

### 6. Q: 这个问题和其他回溯题（如全排列、子集）有什么共性？

**A**: 
**共性框架**：
- 决策树构建 + 剪枝 + 回退
- 用路径数组记录状态，用参数标记当前位置

**差异点**：
| 问题 | 选择列表 | 剪枝条件 | 时间复杂度 |
|------|----------|----------|------------|
| 全排列 | 剩余元素 | 已选过的跳过 | O(n×n!) |
| 子集 | 每个元素选或不选 | 无需剪枝 | O(2^n) |
| 括号生成 | 左或右括号 | `left<n`, `right<left` | O(C_n×n) |

### 7. Q: 如果改成生成 3 种括号 `()[]{}`，如何扩展？

**A**: 
```python
def generate_three_parens(n: int) -> List[str]:
    """生成 n 对三种括号的所有有效组合"""
    result = []
    pairs = {'(': ')', '[': ']', '{': '}'}
    
    def backtrack(path: List[str], stack: List[str]):
        if len(path) == 2 * n:
            if not stack:  # 栈为空才合法
                result.append(''.join(path))
            return
        
        # 选择开括号
        for open_bracket in pairs:
            if path.count(open_bracket) < n:
                path.append(open_bracket)
                stack.append(open_bracket)
                backtrack(path, stack)
                stack.pop()
                path.pop()
        
        # 选择闭括号（必须匹配栈顶）
        if stack:
            last = stack[-1]
            close_bracket = pairs[last]
            path.append(close_bracket)
            stack.pop()
            backtrack(path, stack)
            stack.append(last)
            path.pop()
    
    backtrack([], [])
    return result
```
- 核心变化：用栈维护匹配关系，闭括号必须匹配栈顶开括号
- 时间复杂度：指数级（更复杂，无简单封闭形式）

### 8. Q: 如果要验证一个括号序列是否合法，最优方法是什么？

**A**: 
```python
def is_valid(s: str) -> bool:
    """时间 O(n)，空间 O(n)"""
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    
    for char in s:
        if char in pairs.values():  # 开括号
            stack.append(char)
        elif char in pairs:  # 闭括号
            if not stack or stack.pop() != pairs[char]:
                return False
    
    return len(stack) == 0
```
- 时间 O(n)，空间 O(n)（栈）
- 优化：如果只有一种括号 `()`，可以用计数器代替栈，空间 O(1)

## 常见错误（5 个）

### 1. 剪枝条件写错，生成无效序列

**错误代码**：
```python
# 错误：只检查 right < n
if right < n:
    path.append(')')
    backtrack(path, left, right + 1)
    path.pop()
```

**结果**：生成 `")(()"`, `"())(("` 等无效序列

**正确做法**：`right < left` 确保右括号有匹配的左括号

### 2. 忘记撤销选择，路径污染

**错误代码**：
```python
def backtrack(path, left, right):
    if left < n:
        path.append('(')
        backtrack(path, left + 1, right)
        # 忘记 pop！
    
    if right < left:
        path.append(')')
        backtrack(path, left, right + 1)
        # 忘记 pop！
```

**结果**：路径累积所有选择，结果全是长度 > 2n 的序列

**正确做法**：每个选择分支后必须 `path.pop()`

### 3. 结束条件判断错误

**错误代码**：
```python
# 错误：只判断 left == n
if left == n:
    result.append(''.join(path))
    return
```

**结果**：收集到 `(((`, `(()` 等不完整序列

**正确做法**：`len(path) == 2 * n` 或 `left == n and right == n`

### 4. 重复添加结果

**错误代码**：
```python
def backtrack(path, left, right):
    if len(path) == 2 * n:
        result.append(path)  # 错误：传引用
        return
```

**结果**：所有结果都指向同一个 `path` 对象，最终全变成最后一个解

**正确做法**：`result.append(''.join(path))` 或 `result.append(path.copy())`

### 5. 时间复杂度分析错误

**错误说法**："时间复杂度是 O(2^(2n))"

**正确分析**：
- 搜索空间是卡特兰数 C_n，远小于 2^(2n)
- 例如 n=8：C_8=1430，2^16=65536，相差 45 倍
- 正确表述：O(C_n × n)，C_n ≈ 4^n / (n^(3/2) × √π)

```python
# 计数版参考
def catalan(n: int) -> int:
    """计算第 n 个卡特兰数"""
    if n <= 1:
        return 1
    
    dp = [0] * (n + 1)
    dp[0] = dp[1] = 1
    
    for i in range(2, n + 1):
        for j in range(i):
            dp[i] += dp[j] * dp[i - 1 - j]
    
    return dp[n]
```
