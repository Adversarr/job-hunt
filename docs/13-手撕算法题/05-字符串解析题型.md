# 字符串解析题型

## 一句话结论
字符串解析核心：**栈处理嵌套结构**（递归转迭代）+ **状态机处理多模式**（数字/字符/符号）+ **边界校验贯穿始终**（空串/溢出/前导零/多位数）。

## 核心定义/公式

### 典型题型分类
1. **解码类**：`"3[a]2[bc]"` → `"aaabcbc"`（LeetCode 394）
2. **计算器类**：`"3+2*2"` → `7`（LeetCode 227/224）
3. **路径类**：`"/home//foo/"` → `"/home/foo"`（LeetCode 71）
4. **表达式类**：中缀转后缀、括号匹配、逆波兰表达式

## 为什么（2-3 个因果链）

### 1. 为什么用栈而不是递归？
- **现象**：嵌套结构天然符合递归定义（如 `2[3[a]]`）
- **根因**：递归有栈深度限制（Python 默认 1000），嵌套层次过深会栈溢出；且迭代更易调试、空间可控
- **结果**：显式栈将递归转迭代，栈深度可控，支持超深嵌套

### 2. 为什么需要状态机？
- **现象**：一个字符可能属于不同模式（数字、字母、括号）
- **根因**：单一遍历无法处理多模式转换，需要记录"当前在解析什么"
- **结果**：状态机清晰分离解析逻辑，每个状态处理一种模式，降低复杂度

### 3. 为什么边界校验要在解析过程中做而非后处理？
- **现象**：前导零 `"01[a]"`、数字溢出 `999999999999[a]`、不匹配括号 `"[a"` 等异常
- **根因**：后处理无法定位错误位置，且可能引发运行时错误（如整数溢出）
- **结果**：边解析边校验，及时报错并定位，避免传播

## 怎么做（可落地步骤）

### 标准解题模板（解码类）

```python
def decodeString(s: str) -> str:
    """
    LeetCode 394. 字符串解码
    "3[a]2[bc]" -> "aaabcbc"
    "3[a2[c]]" -> "accaccacc"
    """
    stack = []  # 存储 (prev_str, repeat_num)
    curr_str = ""
    curr_num = 0
    
    for ch in s:
        if ch.isdigit():
            # 处理多位数："12[a]" → curr_num=12
            curr_num = curr_num * 10 + int(ch)
        elif ch == '[':
            # 压栈：保存当前状态，开始新的解码
            stack.append((curr_str, curr_num))
            curr_str = ""
            curr_num = 0
        elif ch == ']':
            # 出栈：结束当前层解码
            prev_str, repeat_num = stack.pop()
            curr_str = prev_str + curr_str * repeat_num
        else:
            # 字符直接追加
            curr_str += ch
    
    return curr_str
```

### 边界处理清单

```python
def decodeString_with_validation(s: str) -> str:
    """带完整边界校验的版本"""
    
    # 1. 空串处理
    if not s:
        return ""
    
    # 2. 初始化
    stack = []
    curr_str = ""
    curr_num = 0
    max_int = 2**31 - 1  # 防止溢出
    
    # 3. 前导零标记
    has_leading_zero = False
    
    for i, ch in enumerate(s):
        if ch.isdigit():
            # 边界：前导零检查（"01[a]" 非法，"10[a]" 合法）
            if curr_num == 0 and ch == '0' and not has_leading_zero:
                # 单独的 0 是合法的，如 "0[a]"
                has_leading_zero = True
            elif curr_num == 0 and ch != '0':
                # 正常数字开始
                has_leading_zero = False
            
            # 边界：数字溢出检查
            if curr_num > (max_int - int(ch)) // 10:
                raise ValueError(f"Integer overflow at position {i}")
            
            curr_num = curr_num * 10 + int(ch)
            
        elif ch == '[':
            # 边界：数字必须紧跟 [
            if curr_num == 0:
                raise ValueError(f"No number before '[' at position {i}")
            
            stack.append((curr_str, curr_num))
            curr_str = ""
            curr_num = 0
            has_leading_zero = False
            
        elif ch == ']':
            # 边界：栈不能为空（括号匹配）
            if not stack:
                raise ValueError(f"Unmatched ']' at position {i}")
            
            prev_str, repeat_num = stack.pop()
            # 边界：结果长度限制（防止内存爆炸）
            if len(curr_str) * repeat_num > 10**6:
                raise ValueError(f"Result too large at position {i}")
            
            curr_str = prev_str + curr_str * repeat_num
            
        else:
            # 字母
            curr_str += ch
    
    # 边界：栈必须为空（所有括号已匹配）
    if stack:
        raise ValueError(f"Unmatched '[' in string")
    
    return curr_str
```

### 状态机方案（复杂场景）

```python
class StateMachine:
    """状态机处理多模式字符串"""
    
    def __init__(self):
        self.state = 'START'
        self.stack = []
        self.buffer = ""
        self.num = 0
    
    def transition(self, ch: str):
        """状态转移"""
        if ch.isdigit():
            if self.state in ('START', 'NUM'):
                self.state = 'NUM'
                self.num = self.num * 10 + int(ch)
            elif self.state == 'STR':
                # 字符串后直接接数字：开始新的解码
                self.stack.append(self.buffer)
                self.buffer = ""
                self.state = 'NUM'
                self.num = int(ch)
        
        elif ch == '[':
            if self.state == 'NUM':
                self.stack.append(self.num)
                self.num = 0
                self.state = 'BRACKET_OPEN'
            else:
                raise ValueError("Invalid '[' position")
        
        elif ch == ']':
            if self.state == 'STR':
                repeat = self.stack.pop()
                prev = self.stack.pop() if self.stack else ""
                self.buffer = prev + self.buffer * repeat
                self.state = 'STR'
            else:
                raise ValueError("Invalid ']' position")
        
        else:  # 字母
            if self.state in ('START', 'STR', 'BRACKET_OPEN'):
                self.state = 'STR'
                self.buffer += ch
            else:
                raise ValueError(f"Unexpected char '{ch}' in state {self.state}")
```

### 计算器模板（加减乘除）

```python
def calculate(s: str) -> int:
    """
    LeetCode 227. 基本计算器 II
    "3+2*2" -> 7
    " 3/2 " -> 1
    """
    stack = []
    num = 0
    sign = '+'  # 上一个运算符
    
    for i, ch in enumerate(s):
        if ch.isdigit():
            num = num * 10 + int(ch)
        
        # 遇到运算符或末尾，处理上一个运算符
        if (not ch.isdigit() and ch != ' ') or i == len(s) - 1:
            if sign == '+':
                stack.append(num)
            elif sign == '-':
                stack.append(-num)
            elif sign == '*':
                stack.append(stack.pop() * num)
            elif sign == '/':
                # 向下取整
                stack.append(int(stack.pop() / num))
            
            sign = ch
            num = 0
    
    return sum(stack)
```

### 路径解析模板

```python
def simplifyPath(path: str) -> str:
    """
    LeetCode 71. 简化路径
    "/home//foo/" -> "/home/foo"
    "/../" -> "/"
    """
    stack = []
    parts = path.split('/')
    
    for part in parts:
        if part == '' or part == '.':
            continue
        elif part == '..':
            if stack:
                stack.pop()
        else:
            stack.append(part)
    
    return '/' + '/'.join(stack)
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **栈 vs 递归** | 避免栈溢出，空间可控，易调试 | 代码稍复杂，需手动管理状态 | 嵌套深度 > 1000 或需精确控制内存 |
| **状态机 vs 单遍扫描** | 清晰分离逻辑，易扩展新模式 | 代码量增加，简单题过度设计 | 多模式混合（数字/字符/符号）、复杂规则 |
| **边解析边校验 vs 后处理校验** | 及时发现错误，定位准确 | 性能略低（频繁判断） | 用户输入、边界敏感场景 |
| **双栈（数字栈+字符栈）vs 单栈** | 逻辑清晰，易理解 | 空间开销略大 | 新手友好，面试现场编码 |

## 高频追问（至少 5 个）

### Q1: 字符串 `"100[a]"` 怎么处理多位数？
A: 使用 `curr_num = curr_num * 10 + int(ch)` 累加，遇到 `[` 才压栈。每个数字字符都会更新 `curr_num`，`"100"` → `1 → 10 → 100`。

### Q2: 前导零 `"01[a]"` 合法吗？怎么判断？
A: 通常不合法（类似编程语言整数）。判断逻辑：
- `curr_num == 0` 且当前字符是 `'0'`：标记 `has_leading_zero = True`
- `has_leading_zero == True` 且遇到非零数字：报错
- 单独的 `"0[a]"` 合法（重复 0 次，结果为空串）

### Q3: 数字太大导致结果字符串过长怎么办？
A: 两层防护：
1. **解析时**：检查 `curr_num > max_int`，防止整数溢出
2. **构造时**：检查 `len(curr_str) * repeat_num > threshold`，防止内存爆炸
```python
MAX_LEN = 10**6  # 根据实际需求设定
if len(curr_str) * repeat_num > MAX_LEN:
    raise ValueError("Result exceeds maximum length")
```

### Q4: 嵌套括号 `"3[a2[b]]"` 怎么处理？
A: 栈天然支持嵌套：
- 遇到 `[`：压栈当前状态，开始新层
- 遇到 `]`：弹出栈顶，合并结果
- `"3[a2[b]]"` → 栈过程：`[] → [('' ,3)] → [('' ,3), ('a',2)] → [('abbabbabb',)]`

### Q5: 不匹配的括号 `"3[a"` 或 `"a]"` 怎么检测？
A: 
- **缺 `]`**：遍历结束后栈不为空，`if stack: raise`
- **缺 `[`**：遇到 `]` 时栈为空，`if not stack: raise`
- 记录位置 `i` 便于定位：`raise ValueError(f"Unmatched ']' at position {i}")`

### Q6: 如何处理空字符串输入和全空格字符串？
A: 
- **空串**：`if not s: return ""`
- **全空格**：计算器类题目先 `s = s.replace(' ', '')` 或在遍历时 `if ch == ' ': continue`
- 注意：路径类题目 `"/  /"` 需保留路径语义，不能简单去空格

### Q7: 字符串解码的时间复杂度和空间复杂度？
A:
- **时间**：O(N + M)，N 为输入长度，M 为输出长度（构造结果）
- **空间**：O(N + M)，栈深度 O(N)，结果字符串 O(M)
- 嵌套深度为 D 时，栈空间 O(D)，但 D ≤ N

## 常见错误（至少 3 个）

### 错误 1: 多位数处理错误
**描述**：只用 `curr_num = int(ch)`，遇到 `"12[a]"` 只取最后一位 2。
**正确做法**：`curr_num = curr_num * 10 + int(ch)` 累加。

```python
# 错误
if ch.isdigit():
    curr_num = int(ch)  # ❌ 每次覆盖，丢失高位

# 正确
if ch.isdigit():
    curr_num = curr_num * 10 + int(ch)  # ✅ 累加
```

### 错误 2: 栈元素顺序混乱
**描述**：压栈 `(num, str)`，出栈时用 `(str, num)`，导致类型错误。
**正确做法**：统一顺序，如 `(prev_str, repeat_num)`，并在出栈时解包到对应变量。

```python
# 错误
stack.append((curr_num, curr_str))  # 压栈 (num, str)
prev_str, repeat_num = stack.pop()  # ❌ 解包顺序反了

# 正确
stack.append((curr_str, curr_num))  # 压栈 (str, num)
prev_str, repeat_num = stack.pop()  # ✅ 顺序一致
```

### 错误 3: 忘记重置 curr_num
**描述**：遇到 `[` 后没有重置 `curr_num = 0`，导致下一个数字从上一个数字继续累加。
**正确做法**：每次压栈后立即重置。

```python
elif ch == '[':
    stack.append((curr_str, curr_num))
    curr_str = ""
    curr_num = 0  # ✅ 必须重置
```

### 错误 4: 括号匹配检查遗漏
**描述**：只检查栈是否为空，不检查不匹配的 `]`，导致 `pop()` 时 IndexError。
**正确做法**：
```python
elif ch == ']':
    if not stack:  # ✅ 先检查
        raise ValueError("Unmatched ']'")
    prev_str, repeat_num = stack.pop()
    # ...
```

### 错误 5: 结果字符串过长未检查
**描述**：`"999999[a]"` 导致内存爆炸或超时。
**正确做法**：
```python
MAX_LEN = 10**6
if len(curr_str) * repeat_num > MAX_LEN:
    raise ValueError("Result too large")
```

### 错误 6: 计算器类题目除法精度错误
**描述**：`int(stack.pop() / num)` 在负数时向零取整，但题目可能要求向下取整。
**正确做法**：
```python
# Python 3 中 // 是向下取整
# 题目若要求向零取整（截断）
result = int(a / b)

# 题目若要求向下取整
result = a // b
```
