# DP 题型：乘积子数组与背包问题

## 一句话结论
DP 核心是**状态定义**和**状态转移**：乘积子数组需同时维护 max/min 应对负数，完全背包需区分"求极值"和"求计数"两套转移方程，时间复杂度 O(n²) 但可通过状态压缩优化空间。

## 核心定义/公式

### 1. 最大乘积子数组（LeetCode 152）

**状态定义**：
```
dp_max[i] = 以 nums[i] 结尾的最大乘积
dp_min[i] = 以 nums[i] 结尾的最小乘积（处理负数）
```

**状态转移**：
```python
dp_max[i] = max(nums[i], dp_max[i-1] * nums[i], dp_min[i-1] * nums[i])
dp_min[i] = min(nums[i], dp_max[i-1] * nums[i], dp_min[i-1] * nums[i])
```

**完整代码**：
```python
def maxProduct(nums):
    if not nums:
        return 0
    
    n = len(nums)
    dp_max = [0] * n
    dp_min = [0] * n
    dp_max[0] = dp_min[0] = nums[0]
    
    result = nums[0]
    for i in range(1, n):
        dp_max[i] = max(nums[i], dp_max[i-1] * nums[i], dp_min[i-1] * nums[i])
        dp_min[i] = min(nums[i], dp_max[i-1] * nums[i], dp_min[i-1] * nums[i])
        result = max(result, dp_max[i])
    
    return result

# 空间优化版 O(1)
def maxProduct_optimized(nums):
    if not nums:
        return 0
    
    max_prod = min_prod = result = nums[0]
    
    for i in range(1, len(nums)):
        # 先保存旧值，因为会被覆盖
        prev_max, prev_min = max_prod, min_prod
        max_prod = max(nums[i], prev_max * nums[i], prev_min * nums[i])
        min_prod = min(nums[i], prev_max * nums[i], prev_min * nums[i])
        result = max(result, max_prod)
    
    return result
```

### 2. 零钱兑换（LeetCode 322 - 完全背包求最小）

**状态定义**：
```
dp[i] = 凑成金额 i 所需的最少硬币数
```

**状态转移**：
```python
dp[i] = min(dp[i], dp[i - coin] + 1) for coin in coins if coin <= i
```

**完整代码**：
```python
def coinChange(coins, amount):
    # 初始化：dp[0] = 0，其余为无穷大
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    
    # 完全背包：外层遍历金额，内层遍历硬币
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != float('inf') else -1

# 优化版：外层遍历硬币，内层遍历金额
def coinChange_optimized(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    
    for coin in coins:
        for i in range(coin, amount + 1):
            dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != float('inf') else -1
```

### 3. 零钱兑换 II（LeetCode 518 - 完全背包求计数）

**状态定义**：
```
dp[i] = 凑成金额 i 的组合数
```

**状态转移**：
```python
dp[i] += dp[i - coin] for coin in coins if coin <= i
```

**完整代码**：
```python
def change(amount, coins):
    # 初始化：dp[0] = 1（一种方法：不选任何硬币）
    dp = [0] * (amount + 1)
    dp[0] = 1
    
    # 完全背包求计数：外层遍历硬币（保证组合而非排列）
    for coin in coins:
        for i in range(coin, amount + 1):
            dp[i] += dp[i - coin]
    
    return dp[amount]

# 对比：如果是求排列数，需要外层遍历金额
def change_permutation(amount, coins):
    dp = [0] * (amount + 1)
    dp[0] = 1
    
    # 外层遍历金额 → 得到的是排列数
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] += dp[i - coin]
    
    return dp[amount]
```

## 为什么（3 个因果链）

### 1. 为什么乘积子数组需要同时维护 max 和 min？

**因果链**：存在负数 → 负负得正 → 最小值可能变最大值

- **现象**：`[-2, 3, -4]` 中，`(-2) * 3 * (-4) = 24` 是最大乘积
- **根因**：负数乘以最小值（很负）会得到最大值（很正）
- **结果**：必须同时维护 `dp_max` 和 `dp_min`，每次从三个候选项中选最大/最小

### 2. 为什么完全背包内外层循环顺序影响结果？

**因果链**：遍历顺序决定选择顺序 → 组合 vs 排列 → 题目需求匹配

- **外层物品（硬币），内层容量（金额）**：
  - 每个硬币按顺序处理，保证 `[1, 2]` 和 `[2, 1]` 不会重复计数
  - 得到的是**组合数**（顺序无关）
  
- **外层容量，内层物品**：
  - 每个金额会考虑所有硬币，`[1, 2]` 和 `[2, 1]` 分别计数
  - 得到的是**排列数**（顺序有关）

### 3. 为什么 DP 能将指数级搜索降为多项式复杂度？

**因果链**：重叠子问题 → 记忆化存储 → 避免重复计算

- **现象**：暴力搜索会重复计算相同子问题（如 `amount=10` 被多次计算）
- **根因**：DP 通过填表存储中间结果，每个子问题只计算一次
- **结果**：时间复杂度从 O(2^n) 降到 O(n * amount)

#   - 完全背包：内层正序遍历（每个物品可以用多次）
#   - 组合问题：外层物品，内层容量
#   - 排列问题：外层容量，内层物品

```python
# 滚动数组：将二维 DP 降为一维
# 条件：当前状态只依赖于有限的几个前驱状态
# 注意：必须确保覆盖顺序正确（倒序/正序）
```

def maxProduct(nums):

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| 暴力搜索 | 实现简单，容易理解 | 时间 O(2^n)，数据稍大就超时 | n < 20 的小规模问题 |
| 记忆化递归 | 避免重复计算，思路清晰 | 有递归栈开销，空间 O(n) | 需要自顶向下思考的问题 |
| 二维 DP | 状态转移清晰，容易调试 | 空间 O(n²)，大数据集易 OOM | 中小规模问题 |
| 一维 DP | 空间 O(n)，适合面试手撕 | 需要注意遍历顺序，不易扩展 | 完全背包/0-1背包标准问题 |
| BFS（最短路） | 天然求最短路径，代码简洁 | 空间消耗大，不适用于计数问题 | 零钱兑换等求最小步数问题 |

**关键权衡点**：
- **时间 vs 空间**：一维 DP 节省空间，但需注意覆盖顺序
- **组合 vs 排列**：循环顺序决定结果类型，必须根据题目要求选择
- **最优化 vs 计数**：最优化用 `min/max`，计数用累加

## 高频追问（至少 5 个）

1. **Q: 为什么乘积子数组不能只维护最大值？**
   A: 因为负数存在时，`最小值 × 负数 = 最大值`。例如 `[2, -3, -4]`，`(-3) * (-4) = 12` 是最大乘积。必须同时维护 `dp_max` 和 `dp_min` 来处理负负得正的情况。

2. **Q: 零钱兑换中，为什么外层遍历硬币得到组合数，外层遍历金额得到排列数？**
   A: 外层遍历硬币时，每个硬币只被处理一次，`[1, 2]` 和 `[2, 1]` 被视为同一种组合。外层遍历金额时，每个金额会尝试所有硬币，顺序不同会被分别计数，得到排列数。

3. **Q: 完全背包和 0-1 背包的遍历顺序有什么区别？为什么？**
   A: 0-1 背包内层倒序遍历，确保每个物品只被使用一次；完全背包内层正序遍历，允许同一物品被多次选择。正序时 `dp[i]` 会使用已更新的 `dp[i-coin]`，相当于用了多次同一物品。

4. **Q: 如果零钱兑换要求恰好用 k 个硬币，怎么修改 DP？**
   A: 状态定义改为二维：`dp[i][j]` 表示凑成金额 i 恰好用 j 个硬币的组合数。转移方程：`dp[i][j] = sum(dp[i-coin][j-1]) for coin in coins`。初始化 `dp[0][0] = 1`。

5. **Q: 乘积最大子数组有 O(1) 空间的解法吗？**
   A: 有，用两个变量 `max_prod` 和 `min_prod` 代替数组，每次更新前保存旧值。空间复杂度从 O(n) 降到 O(1)。面试中推荐写优化版展示空间优化能力。

6. **Q: DP 题目如何判断是否可以空间优化？**
   A: 观察状态转移方程，如果 `dp[i]` 只依赖于有限个前驱状态（如 `dp[i-1]`，`dp[i-2]`），则可以降维。用变量保存需要的前驱值，原地更新当前状态。

7. **Q: 零钱兑换如果 coins 很大（如 10^5），怎么优化？**
   A: 可以用 BFS 做"最短路径"问题，因为每步选择硬币相当于走一步。但计数问题仍需 DP。如果硬币数量远小于金额，可以预处理硬币间的整除关系优化搜索。

## 常见错误（至少 3 个）

1. **乘积子数组只维护最大值，忽略负数情况**
   - **错误做法**：`dp[i] = max(nums[i], dp[i-1] * nums[i])`
   - **正确做法**：同时维护 `dp_max` 和 `dp_min`，每次从三个候选项中选择
   - **典型失败案例**：`[2, -5, -2, -4, 3]` 只维护最大值得 3，正确答案 24（`-5 * -2 * -4 * 3` 的负数乘积）

2. **完全背包求计数时，循环顺序写反（得到排列数而非组合数）**
   - **错误做法**：外层遍历金额，内层遍历硬币
   - **正确做法**：外层遍历硬币，内层遍历金额
   - **典型失败案例**：`amount=5, coins=[1, 2, 5]`，错误顺序得到 9（排列数），正确答案 4（组合数）

3. **零钱兑换初始化错误，dp[0] 没有设置为合法值**
   - **错误做法**：`dp = [float('inf')] * (amount + 1)` 不设置 `dp[0] = 0`
   - **正确做法**：`dp[0] = 0`（凑成金额 0 不需要任何硬币）
   - **典型失败案例**：所有 `dp[i]` 都保持 `inf`，返回 -1

4. **空间优化时覆盖顺序错误（0-1 背包用正序，完全背包用倒序）**
   - **错误做法**：0-1 背包 `for i in range(coin, amount+1)` 正序遍历
   - **正确做法**：0-1 背包 `for i in range(amount, coin-1, -1)` 倒序遍历
   - **典型失败案例**：正序遍历时，`dp[i]` 会使用已更新的 `dp[i-coin]`，相当于同一物品被多次使用

5. **状态定义不清楚，dp[i] 的物理含义模糊**
   - **错误做法**：不明确 `dp[i]` 是"以 i 结尾"还是"前 i 个元素"
   - **正确做法**：明确定义并写在注释中，如 `dp[i] = 以 nums[i] 结尾的最大乘积`
   - **典型失败案例**：混淆两种定义，导致转移方程写错边界
