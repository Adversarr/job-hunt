# 工程手撕题型

## 一句话结论
工程手撕考察代码规范性、边界处理、状态机思维和复杂度分析：井字棋核心是状态合法性校验+获胜线枚举；MHA 手撕关键是形状推导+数值稳定性；任务调度器用贪心+填充槽位；路径转换需要处理相对/绝对路径、冗余路径、边界情况。

## 核心定义/公式

### 1. 井字棋胜负判断

**状态表示**：
```
'X' = 玩家1, 'O' = 玩家2, '.' = 空
```

**获胜条件（8条线）**：
- 横向：3条 (行0, 行1, 行2)
- 纵向：3条 (列0, 列1, 列2)
- 对角线：2条 (主对角、副对角)

**合法性约束**：
```
count(X) == count(O) 或 count(X) == count(O) + 1
若 X 获胜，则 count(X) == count(O) + 1
若 O 获胜，则 count(X) == count(O)
```

### 2. NumPy MHA 核心公式

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
MultiHead = Concat(head_1, ..., head_h) W^O
```

**关键形状变换**：
```python
# Input: [batch, seq_len, d_model]
# QKV投影: [batch, seq_len, d_model]
# 分头: [batch, num_heads, seq_len, head_dim]
# Attention: [batch, num_heads, seq_len, seq_len]
# Output: [batch, seq_len, d_model]
```

### 3. 任务调度器 (LC-621)

**核心公式**：
```
最短时间 = max((max_count - 1) * (n + 1) + num_max, len(tasks))
```
其中：
- `max_count`: 出现次数最多的任务的次数
- `num_max`: 出现次数等于 max_count 的任务种类数
- `n`: 冷却时间

### 4. 路径转换

**关键操作**：
- `.` 表示当前目录
- `..` 表示上级目录
- 连续 `/` 规约为单个 `/`
- 相对路径 → 绝对路径：基于当前工作目录拼接

## 为什么（2-3 个因果链）

### 1. 为什么井字棋需要状态合法性检查？

**因果链**：
- 现象：用户可能输入非法状态（如双方都赢、棋子数不对）
- 根因：题目通常给定"当前状态"需判断"是否获胜"，但状态本身可能矛盾
- 结果：必须先校验合法性才能给出正确答案

**示例非法状态**：
```
X X X
O O O    # 双方都赢，非法
. . .

X X X
. . .    # X赢但count(X)==count(O)+2，非法
O O .
```

### 2. 为什么 MHA 手撕重点考察形状推导？

**因果链**：
- 现象：MHA 涉及多次 reshape、transpose、矩阵乘法
- 根因：维度错误是最常见 bug，形状推导体现对计算流程的深度理解
- 结果：面试官通过"能否准确推导每步形状"判断是否真正掌握 MHA 原理

### 3. 为什么任务调度器用贪心而非 DP？

**因果链**：
- 现象：任务调度看似需要排列组合
- 根因：冷却时间是固定约束，瓶颈在于出现最多的任务类型
- 结果：贪心策略（先安排最多任务，填充空隙）直接得到最优解，复杂度从 O(n!) 降到 O(n)

### 一、井字棋胜负判断

#### 标准做法

```python
def check_tictactoe_winner(board):
    """
    判断井字棋当前玩家是否获胜
    
    Args:
        board: 3x3 的二维列表，包含 'X', 'O', '.'
    
    Returns:
        str: 'X' 或 'O' 表示获胜方，'Draw' 表示平局，'Pending' 表示未结束，'Invalid' 表示非法状态
    """
    # 1. 输入解析与合法性校验
    if len(board) != 3 or any(len(row) != 3 for row in board):
        return 'Invalid'
    
    # 2. 统计棋子数量
    count_x = sum(row.count('X') for row in board)
    count_o = sum(row.count('O') for row in board)
    
    # 3. 状态合法性检查
    # X 先手，所以 X 数量等于 O 或比 O 多 1
    if not (count_x == count_o or count_x == count_o + 1):
        return 'Invalid'
    
    # 4. 定义所有获胜线
    win_lines = [
        # 横向
        [(0, 0), (0, 1), (0, 2)],
        [(1, 0), (1, 1), (1, 2)],
        [(2, 0), (2, 1), (2, 2)],
        # 纵向
        [(0, 0), (1, 0), (2, 0)],
        [(0, 1), (1, 1), (2, 1)],
        [(0, 2), (1, 2), (2, 2)],
        # 对角线
        [(0, 0), (1, 1), (2, 2)],
        [(0, 2), (1, 1), (2, 0)],
    ]
    
    # 5. 检查是否有玩家获胜
    def check_win(player):
        for line in win_lines:
            if all(board[i][j] == player for i, j in line):
                return True
        return False
    
    x_wins = check_win('X')
    o_wins = check_win('O')
    
    # 6. 双方同时获胜，非法
    if x_wins and o_wins:
        return 'Invalid'
    
    # 7. X 获胜但数量不对
    if x_wins and count_x != count_o + 1:
        return 'Invalid'
    
    # 8. O 获胜但数量不对
    if o_wins and count_x != count_o:
        return 'Invalid'
    
    # 9. 返回结果
    if x_wins:
        return 'X'
    if o_wins:
        return 'O'
    
    # 10. 检查是否还有空位
    if any('.' in row for row in board):
        return 'Pending'
    
    return 'Draw'


# 从文件读取井字棋状态
def read_tictactoe_from_file(filepath):
    """
    从 txt 文件读取井字棋状态
    
    文件格式示例：
    X O X
    O X O
    X . .
    """
    board = []
    with open(filepath, 'r') as f:
        for line in f:
            # 处理可能的分隔符（空格、逗号等）
            row = line.strip().replace(',', ' ').split()
            if len(row) == 3:
                board.append(row)
    return board


# 完整流程
def solve_tictactoe(filepath):
    """读取文件并判断胜负"""
    board = read_tictactoe_from_file(filepath)
    return check_tictactoe_winner(board)
```

```python
        lines.append([(i, j) for j in range(n)])

        lines.append([(i, j) for i in range(n)])
```

### 二、只用 NumPy 写 MHA

#### 标准实现

```python
import numpy as np

def softmax(x, axis=-1):
    """数值稳定的 softmax"""
    x_max = np.max(x, axis=axis, keepdims=True)
    exp_x = np.exp(x - x_max)
    return exp_x / np.sum(exp_x, axis=axis, keepdims=True)

def multi_head_attention_numpy(X, W_Q, W_K, W_V, W_O, num_heads, mask=None):
    """
    只用 NumPy 实现多头注意力
    
    Args:
        X: [batch, seq_len, d_model] 输入
        W_Q, W_K, W_V: [d_model, d_model] 投影矩阵
        W_O: [d_model, d_model] 输出投影
        num_heads: 头数
        mask: [seq_len, seq_len] causal mask（可选）
    
    Returns:
        output: [batch, seq_len, d_model]
        attn_weights: [batch, num_heads, seq_len, seq_len]
    """
    batch_size, seq_len, d_model = X.shape
    head_dim = d_model // num_heads
    
    assert d_model % num_heads == 0, "d_model 必须能被 num_heads 整除"
    
    # 1. 线性投影 [batch, seq_len, d_model]
    Q = X @ W_Q
    K = X @ W_K
    V = X @ W_V
    
    # 2. 分头 [batch, seq_len, d_model] -> [batch, num_heads, seq_len, head_dim]
    Q = Q.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)
    K = K.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)
    V = V.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)
    
    # 3. 计算注意力分数 [batch, num_heads, seq_len, seq_len]
    scores = Q @ K.transpose(0, 1, 3, 2) / np.sqrt(head_dim)
    
    # 4. 应用 mask
    if mask is not None:
        scores = scores + mask  # broadcast 到所有 batch 和 head
    
    # 5. Softmax
    attn_weights = softmax(scores, axis=-1)
    
    # 6. 加权求和 [batch, num_heads, seq_len, head_dim]
    context = attn_weights @ V
    
    # 7. 合并头 [batch, seq_len, d_model]
    context = context.transpose(0, 2, 1, 3).reshape(batch_size, seq_len, d_model)
    
    # 8. 输出投影
    output = context @ W_O
    
    return output, attn_weights


def create_causal_mask(seq_len):
    """创建因果 mask"""
    mask = np.triu(np.ones((seq_len, seq_len)), k=1) * (-1e9)
    return mask


# 完整测试用例
def test_mha():
    np.random.seed(42)
    batch_size, seq_len, d_model, num_heads = 2, 4, 64, 8
    
    X = np.random.randn(batch_size, seq_len, d_model)
    W_Q = np.random.randn(d_model, d_model)
    W_K = np.random.randn(d_model, d_model)
    W_V = np.random.randn(d_model, d_model)
    W_O = np.random.randn(d_model, d_model)
    
    mask = create_causal_mask(seq_len)
    
    output, attn_weights = multi_head_attention_numpy(X, W_Q, W_K, W_V, W_O, num_heads, mask)
    
    print(f"Input shape: {X.shape}")
    print(f"Output shape: {output.shape}")
    print(f"Attention weights shape: {attn_weights.shape}")
    
    # 验证因果性：attention weights 上三角应为 0
    for h in range(num_heads):
        assert np.allclose(attn_weights[0, h, :, :], np.tril(attn_weights[0, h, :, :])), \
            f"Head {h} 上三角不为 0"
    
    print("✓ 测试通过")
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| QKV 投影 | O(batch × seq × d²) | O(batch × seq × d) |
| 分头 reshape | O(batch × seq × d) | 同上（view） |
| QK^T 矩阵乘 | O(batch × heads × seq² × d_k) | O(batch × heads × seq²) |
| Softmax | O(batch × heads × seq²) | 同上 |
| Attention @ V | O(batch × heads × seq² × d_k) | O(batch × seq × d) |
| 输出投影 | O(batch × seq × d²) | O(batch × seq × d) |
| **总计** | O(batch × d² × seq + batch × d × seq²) | O(batch × seq × d + batch × seq²) |

**关键点**：
- 当 `seq << d` 时，投影主导
- 当 `seq >> d` 时，注意力矩阵主导（O(seq²)）

### 三、任务调度器 (LC-621)

#### 标准解法

```python
def least_interval(tasks, n):
    """
    任务调度器：最短时间完成所有任务
    
    Args:
        tasks: 任务列表，如 ['A', 'A', 'A', 'B', 'B', 'B']
        n: 冷却时间
    
    Returns:
        int: 最短时间
    
    核心思想：
    1. 出现次数最多的任务是瓶颈
    2. 以最多任务构建框架，其他任务填充空隙
    3. 空隙不够时需要 idle
    """
    if n == 0:
        return len(tasks)
    
    # 统计每个任务的出现次数
    from collections import Counter
    task_counts = Counter(tasks)
    counts = sorted(task_counts.values(), reverse=True)
    
    # 最多出现次数
    max_count = counts[0]
    
    # 出现次数等于 max_count 的任务数
    num_max = counts.count(max_count)
    
    # 核心公式
    # (max_count - 1) 个完整周期，每个周期长度 (n + 1)
    # 最后一个周期填充 num_max 个任务
    result = (max_count - 1) * (n + 1) + num_max
    
    # 如果任务总数更大，说明空隙不够用，直接返回任务总数
    return max(result, len(tasks))


# 详细图解版本（帮助理解）
def least_interval_detailed(tasks, n):
    """
    详细模拟过程帮助理解
    
    示例：tasks = ['A', 'A', 'A', 'B', 'B', 'B'], n = 2
    
    任务计数：A:3, B:3
    max_count = 3, num_max = 2
    
    框架构建（以 A 为主）：
    A _ _ | A _ _ | A B
    
    第 1 个周期：A, _, _ （A 后面有 n=2 个空隙）
    第 2 个周期：A, _, _ 
    最后：A, B （B 也是出现 3 次，所以放在最后）
    
    填充其他任务：
    A B _ | A B _ | A B
    
    总长度：8
    公式验证：(3-1) * (2+1) + 2 = 2*3 + 2 = 8 ✓
    """
    from collections import Counter
    
    if n == 0:
        return len(tasks)
    
    task_counts = Counter(tasks)
    max_count = max(task_counts.values())
    num_max = sum(1 for c in task_counts.values() if c == max_count)
    
    # 方法 1：公式法
    result_formula = (max_count - 1) * (n + 1) + num_max
    
    # 方法 2：模拟法（验证用）
    # 构建调度序列
    schedule = []
    task_list = sorted(task_counts.items(), key=lambda x: -x[1])
    
    while any(count > 0 for _, count in task_list):
        # 每次选择冷却可用且剩余次数最多的任务
        added = False
        for i, (task, count) in enumerate(task_list):
            if count > 0:
                # 检查冷却
                if len(schedule) >= n + 1:
                    # 检查前 n 个位置是否有相同任务
                    recent = schedule[-(n):]
                    if task in recent:
                        continue
                
                schedule.append(task)
                task_list[i] = (task, count - 1)
                added = True
                break
        
        if not added:
            schedule.append('idle')
    
    result_simulate = len(schedule)
    
    assert result_formula == result_simulate, f"公式法和模拟法结果不一致: {result_formula} vs {result_simulate}"
    
    return max(result_formula, len(tasks))
```

#### 贪心策略证明

**为什么贪心是最优的？**

1. **瓶颈分析**：出现次数最多的任务决定了最少需要多少个时间槽
2. **框架构建**：以 `max_count` 个任务为骨架，每两个之间需要 `n` 个冷却槽
3. **填充策略**：其他任务填充到冷却槽中，如果不够则用 idle
4. **特殊情况**：如果任务总数 > 框架大小，说明冷却槽已被填满，无需 idle

```
示例：tasks = ['A', 'A', 'A', 'B', 'B', 'B', 'C', 'C', 'D'], n = 2

框架（以 A 为主）：
A _ _ | A _ _ | A B

填充其他任务：
A B C | A B C | A B D

长度 = 9 = max(8, 9) = max(公式结果, 任务总数)
```

### 四、路径转换题

#### 标准解法

```python
def simplify_path(path):
    """
    简化 Unix 风格路径
    
    Args:
        path: 如 "/home/", "/a/./b/../../c/"
    
    Returns:
        str: 简化后的规范路径
    
    示例：
    "/home/" -> "/home"
    "/a/./b/../../c/" -> "/c"
    "/../" -> "/"
    "/home//foo/" -> "/home/foo"
    """
    stack = []
    
    # 按 '/' 分割
    parts = path.split('/')
    
    for part in parts:
        if part == '' or part == '.':
            # 空字符串或当前目录，跳过
            continue
        elif part == '..':
            # 返回上一级
            if stack:
                stack.pop()
        else:
            # 正常路径
            stack.append(part)
    
    # 重新构建路径
    return '/' + '/'.join(stack)


def relative_to_absolute(relative_path, current_dir):
    """
    相对路径转绝对路径
    
    Args:
        relative_path: 相对路径，如 "./subdir/file.txt", "../parent/file.txt"
        current_dir: 当前工作目录，如 "/home/user"
    
    Returns:
        str: 绝对路径
    """
    # 如果已经是绝对路径
    if relative_path.startswith('/'):
        return simplify_path(relative_path)
    
    # 拼接当前目录和相对路径
    if relative_path.startswith('./'):
        relative_path = relative_path[2:]
    
    combined = current_dir.rstrip('/') + '/' + relative_path
    return simplify_path(combined)


def absolute_to_relative(absolute_path, base_dir):
    """
    绝对路径转相对路径
    
    Args:
        absolute_path: 绝对路径
        base_dir: 基准目录
    
    Returns:
        str: 相对路径
    """
    abs_parts = absolute_path.strip('/').split('/')
    base_parts = base_dir.strip('/').split('/')
    
    # 找到公共前缀
    common_len = 0
    for i in range(min(len(abs_parts), len(base_parts))):
        if abs_parts[i] == base_parts[i]:
            common_len += 1
        else:
            break
    
    # 计算需要返回的层级
    up_levels = len(base_parts) - common_len
    
    # 构建相对路径
    relative_parts = ['..'] * up_levels
    relative_parts.extend(abs_parts[common_len:])
    
    if not relative_parts:
        return '.'
    
    return '/'.join(relative_parts)


# 综合路径转换器
class PathConverter:
    """路径转换工具类"""
    
    @staticmethod
    def normalize(path):
        """规范化路径（处理冗余分隔符、.、..）"""
        return simplify_path(path)
    
    @staticmethod
    def join(*paths):
        """拼接多个路径"""
        result = []
        for path in paths:
            if path.startswith('/'):
                result = [path]
            else:
                if result:
                    result.append(path)
                else:
                    result = [path]
        combined = '/'.join(result) if result else '/'
        return simplify_path(combined.replace('//', '/'))
    
    @staticmethod
    def dirname(path):
        """获取父目录"""
        normalized = simplify_path(path)
        if normalized == '/':
            return '/'
        parts = normalized.rsplit('/', 1)
        return parts[0] if parts[0] else '/'
    
    @staticmethod
    def basename(path):
        """获取文件名"""
        normalized = simplify_path(path)
        if normalized == '/':
            return ''
        return normalized.rsplit('/', 1)[-1]
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **井字棋：枚举获胜线 vs 按行/列/对角遍历** | 代码清晰，易扩展 | 需要预计算或存储获胜线 | 小规模（3x3）推荐枚举，大规模推荐遍历 |
| **MHA：NumPy vs PyTorch** | NumPy 暴露底层细节，展示理解深度 | 无 GPU 加速，生产环境不适用 | 面试手撕场景，考察原理理解 |

### Q1: 井字棋如果棋盘是 N×N，获胜需要连续 M 个棋子，怎么改？

**A**: 
1. **获胜线生成**：不再固定 8 条线，需要动态生成所有长度为 M 的连续线
2. **滑动窗口**：对每行、每列、对角线用滑动窗口检查是否有 M 个连续相同棋子
3. **复杂度**：O(N²) 遍历所有窗口

```python
def check_win_nxn(board, m):
    """N×N 棋盘，需要连续 M 个"""
    n = len(board)
    
    # 检查所有 M 连续窗口
    for i in range(n):
        for j in range(n - m + 1):
            # 横向
            if board[i][j] != '.' and all(board[i][j+k] == board[i][j] for k in range(m)):
                return board[i][j]
            # 纵向
            if board[j][i] != '.' and all(board[j+k][i] == board[j][i] for k in range(m)):
                return board[j][i]
    
    # 对角线（略）
    return None
```

### Q2: MHA 手撕时，batch 维度处理有什么坑？

**A**:
1. **矩阵乘法广播**：`[batch, seq, d] @ [d, d]` 自动广播，但 `[batch, seq, d] @ [batch, d, d]` 需要注意
2. **Transpose 顺序**：NumPy 的 transpose 参数顺序要写对：`(0, 2, 1, 3)` 而非默认
3. **Reshape vs View**：NumPy reshape 可能拷贝数据，要保证内存连续性
4. **Contiguous**：转置后必须 contiguous 才能正确 reshape

### Q3: 任务调度器为什么公式是 `max(result, len(tasks))`？

**A**:
- **框架大小**：`(max_count-1)*(n+1) + num_max` 是基于最多任务构建的框架
- **任务超额**：如果其他任务太多，把框架空隙填满后还有剩余，说明冷却约束不再是瓶颈
- **实际意义**：`len(tasks)` 是无冷却时最少时间，框架方法计算的是有冷却时最少时间，取最大值保证两个约束都满足

**示例**：
```
tasks = ['A', 'A', 'B', 'B', 'C', 'C', 'D', 'D', 'E', 'E', 'F', 'F'], n = 2
max_count = 2, num_max = 6
公式结果 = (2-1)*(2+1) + 6 = 9
任务总数 = 12
最终结果 = max(9, 12) = 12

说明：任务密度太高，冷却槽不够用，实际不需要 idle
```

### Q4: 路径转换中如何处理符号链接（symlink）？

**A**:
符号链接需要解析真实路径，涉及：
1. **系统调用**：`os.path.realpath()` 解析所有符号链接
2. **循环检测**：符号链接可能形成环，需要检测并报错
3. **面试简化**：通常只要求处理 `.` 和 `..`，不涉及符号链接

扩展实现：
```python
import os

def resolve_path_with_symlinks(path):
    """解析路径包括符号链接"""
    try:
        return os.path.realpath(path)
    except OSError as e:
        raise ValueError(f"无法解析路径: {e}")
```

### Q5: MHA 的 softmax 为什么需要 max-trick？

**A**:
1. **数值溢出**：QK^T 点积可能很大（如 100），`exp(100)` 直接溢出为 inf
2. **除零风险**：softmax 公式 `exp(x_i) / sum(exp(x_j))`，分母为 inf 导致 NaN
3. **Max-trick 原理**：
   - 减去最大值后，所有输入 ≤ 0
   - `exp(x - max) ≤ exp(0) = 1`，不会溢出
   - 分子分母同时除以 `exp(max)`，结果不变

```python
# 错误：可能溢出
softmax(x) = exp(x) / sum(exp(x))

# 正确：max-trick
softmax(x) = exp(x - max(x)) / sum(exp(x - max(x)))
```

### Q6: 井字棋如何处理从文件读取时的格式异常？

**A**:
常见异常处理：
1. **行数不对**：不是 3 行 → 返回错误
2. **列数不对**：某行不是 3 个 → 返回错误
3. **非法字符**：不是 X/O/. → 返回错误
4. **分隔符多样**：空格、逗号、制表符 → 统一处理

```python
def read_tictactoe_robust(filepath):
    """鲁棒的文件读取"""
    board = []
    valid_chars = {'X', 'O', '.', 'x', 'o'}  # 大小写兼容
    
    with open(filepath, 'r') as f:
        for line_num, line in enumerate(f, 1):
            # 去除空白
            line = line.strip()
            if not line:
                continue
            
            # 处理多种分隔符
            row = line.replace(',', ' ').replace('\t', ' ').split()
            
            # 检查列数
            if len(row) != 3:
                raise ValueError(f"第 {line_num} 行不是 3 列: {line}")
            
            # 检查字符
            for c in row:
                if c.upper() not in valid_chars:
                    raise ValueError(f"第 {line_num} 行有非法字符: {c}")
            
            # 统一大小写
            board.append([c.upper() for c in row])
    
    # 检查行数
    if len(board) != 3:
        raise ValueError(f"棋盘不是 3 行: {len(board)} 行")
    
    return board
```

### Q7: 任务调度器如果要输出具体调度顺序，怎么改？

**A**:
使用模拟法记录调度序列：

```python
def least_interval_with_schedule(tasks, n):
    """返回最短时间和具体调度顺序"""
    from collections import Counter, deque
    
    task_counts = Counter(tasks)
    # 按次数降序排列
    max_heap = [(-count, task) for task, count in task_counts.items()]
    
    import heapq
    heapq.heapify(max_heap)
    
    time = 0
    schedule = []
    # 冷却队列：[(可用时间, 任务, 剩余次数)]
    cooldown = deque()
    
    while max_heap or cooldown:
        time += 1
        
        # 检查冷却队列是否有任务可用
        if cooldown and cooldown[0][0] < time:
            avail_time, task, count = cooldown.popleft()
            heapq.heappush(max_heap, (-count, task))
        
        if max_heap:
            neg_count, task = heapq.heappop(max_heap)
            count = -neg_count
            schedule.append(task)
            
            if count > 1:
                # 加入冷却队列
                cooldown.append((time + n, task, count - 1))
        else:
            schedule.append('idle')
    
    return time, schedule
```

## 常见错误（至少 3 个）

### 错误 1: 井字棋只检查获胜，不检查状态合法性

**描述**：
```python
# 错误：直接检查获胜，不校验棋子数量
if check_win('X'):
    return 'X'
```

**正确做法**：
```python
# 先校验合法性
count_x = sum(row.count('X') for row in board)
count_o = sum(row.count('O') for row in board)

if not (count_x == count_o or count_x == count_o + 1):
    return 'Invalid'

# 再检查获胜
x_wins = check_win('X')
o_wins = check_win('O')

if x_wins and o_wins:
    return 'Invalid'  # 双方不能同时赢
if x_wins and count_x != count_o + 1:
    return 'Invalid'  # X 赢但数量不对
```
