# 滑窗题型：最长无重复子串与模板化思维

## 一句话结论
滑窗通过双指针维护一个动态窗口，左指针收缩（排除非法状态）、右指针扩展（探索新状态），用哈希表/数组记录窗口内元素频次，时间复杂度从暴力 $O(n^2)$ 优化到 $O(n)$，空间复杂度 $O(\min(n, m))$（m 为字符集大小）。

## 核心定义/公式

### 滑窗核心公式

```
窗口状态 = [left, right) 半开区间
窗口大小 = right - left
最大窗口 = max(max_window, right - left)
```

**关键不变式**：窗口内始终满足题目约束条件（如无重复字符）。

### 时间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 暴力枚举 | $O(n^2)$ | $O(\min(n, m))$ | 枚举所有子串，逐个检查 |
| 滑窗优化 | $O(n)$ | $O(\min(n, m))$ | 左右指针各最多移动 n 次 |

其中 $n$ 为字符串长度，$m$ 为字符集大小（如 ASCII 128、Unicode 65536）。

### 数据结构选择

```python
# 方案 1：哈希表（通用）
char_count = {}  # {字符: 出现次数}
char_index = {}  # {字符: 最新下标}

# 方案 2：数组（字符集已知且连续）
# ASCII: 128 / Extended ASCII: 256 / 小写字母: 26
char_count = [0] * 128  # 下标为字符 ASCII 码，值为出现次数
char_index = [-1] * 128  # 下标为字符 ASCII 码，值为最新下标
```

**选择原则**：
- 字符集小且连续（如小写字母、ASCII）→ 数组（$O(1)$ 访问更快）
- 字符集大或未知（如 Unicode）→ 哈希表（空间节省）

## 为什么（2-3 个因果链）

### 1. 为什么滑窗能优化到 $O(n)$？

**因果链**：暴力枚举每个子串起点和终点 → 大量重复计算（如子串 "abc" 和 "abcd" 重复检查 "abc"）→ 滑窗利用已计算信息（右指针移动时，左指针之前的窗口已合法，无需重新检查）→ 左右指针单调递增，各移动最多 n 次 → 总操作数 $O(2n) = O(n)$

**关键洞察**：
- **左指针单调性**：左指针只向右移动（从不回退），因为回退意味着重新检查已排除的字符
- **右指针单调性**：右指针只向右移动（从不回退），因为窗口大小只增不减或不变
- **状态继承**：右指针扩展时，继承上一个窗口的状态（哈希表/数组记录），只更新新增字符

### 2. 为什么用哈希表/数组记录字符频次？

**因果链**：判断窗口内是否有重复字符 → 需要快速查询某个字符是否在窗口内 → 暴力遍历窗口 $O(k)$（k 为窗口大小）→ 使用哈希表/数组 $O(1)$ 查询 → 整体复杂度从 $O(n^2)$ 降到 $O(n)$

**记录内容选择**：
- **记录频次**：判断是否有重复（频次 > 1），适用于需要知道重复次数的场景
- **记录下标**：快速跳转左指针（直接移到重复字符的下一个位置），适用于"跳过非法区间"优化

### 3. 为什么左指针移动策略有两种？

**因果链**：
- **策略 1：逐步移动（while 循环）**：每次 left++，逐步收缩窗口，保证窗口大小最大 → 适用于需要遍历所有合法子串的场景（如求所有最长子串）
- **策略 2：跳跃移动（直接定位）**：left = max(left, char_index[char] + 1)，直接跳到重复字符的下一个位置 → 适用于只求最大长度的场景（效率更高）

**性能差异**：
```python
# 逐步移动：适合需要记录所有合法窗口的场景
while char_count[s[right]] > 1:  # 有重复，逐步收缩
    char_count[s[left]] -= 1
    left += 1

# 跳跃移动：只求最大长度时更高效
if s[right] in char_index:  # 有重复，直接跳转
    left = max(left, char_index[s[right]] + 1)
```

#### 步骤 3：完整实现（两种方案对比）

**方案 1：记录下标（跳跃移动）**

```python
def length_of_longest_substring(s: str) -> int:
    """
    时间复杂度: O(n)，每个字符最多访问两次
    空间复杂度: O(min(n, m))，m 为字符集大小
    
    关键优化：
    - 用字典记录字符最新位置
    - 左指针直接跳跃到重复字符的下一个位置
    - 无需逐步收缩窗口
    """
    n = len(s)
    if n <= 1:
        return n
    
    char_index = {}  # 记录字符最新出现的位置
    left = 0
    max_len = 0
    
    for right in range(n):
        char = s[right]
        
        # 如果字符在窗口内出现过，更新左指针
        if char in char_index and char_index[char] >= left:
            left = char_index[char] + 1
        
        # 更新字符位置
        char_index[char] = right
        
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

**方案 2：记录频次（逐步移动）**

```python
def length_of_longest_substring_v2(s: str) -> int:
    """
    逐步收缩窗口的版本，适合需要遍历所有合法窗口的场景
    
    关键点：
    - 用字典记录字符出现次数
    - 左指针逐步移动，每次收缩一个字符
    - 适用于需要记录所有合法子串的情况
    """
    n = len(s)
    if n <= 1:
        return n
    
    char_count = {}  # 记录窗口内字符出现次数
    left = 0
    max_len = 0
    
    for right in range(n):
        char = s[right]
        char_count[char] = char_count.get(char, 0) + 1
        
        # 如果有重复，收缩窗口直到没有重复
        while char_count[char] > 1:
            char_count[s[left]] -= 1
            if char_count[s[left]] == 0:
                del char_count[s[left]]
            left += 1
        
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

**方案 3：数组优化（ASCII 字符集）**

```python
def length_of_longest_substring_ascii(s: str) -> int:
    """
    针对已知字符集（ASCII）的优化版本
    
    优势：
    - 数组访问比哈希表更快（O(1) 常数更小）
    - 无哈希冲突问题
    - 适合字符集小且连续的场景
    """
    n = len(s)
    if n <= 1:
        return n
    
    # ASCII 字符集：0-127
    char_index = [-1] * 128  # -1 表示未出现过
    left = 0
    max_len = 0
    
    for right in range(n):
        char = ord(s[right])  # 获取 ASCII 码
        
        # 如果字符在窗口内出现过
        if char_index[char] >= left:
            left = char_index[char] + 1
        
        # 更新字符位置
        char_index[char] = right
        
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

### 滑窗通用模板

```python
def sliding_window_template(s: str) -> int:
    """
    滑窗问题通用模板
    
    适用场景：
    - 最长子串/子数组问题
    - 包含/不包含特定元素的问题
    - 满足某种条件的连续区间问题
    
    时间复杂度：O(n)
    空间复杂度：O(m)，m 为元素种类数
    """
    n = len(s)
    if n == 0:
        return 0
    
    # 1. 初始化数据结构
    window = {}  # 或数组，根据元素类型选择
    left = 0
    result = 0
    
    # 2. 扩展右指针
    for right in range(n):
        # 2.1 添加右指针元素到窗口
        element = s[right]
        window[element] = window.get(element, 0) + 1
        
        # 2.2 检查是否满足条件（或违反约束）
        # 根据具体题目修改判断条件
        while condition_violated(window):  # 条件违反
            # 收缩窗口
            element = s[left]
            window[element] -= 1
            if window[element] == 0:
                del window[element]
            left += 1
        
        # 2.3 更新结果（窗口满足条件时）
        result = max(result, right - left + 1)
    
    return result

def condition_violated(window: dict) -> bool:
    """
    根据具体题目定义条件
    
    示例：
    - 无重复子串：any(count > 1 for count in window.values())
    - 最多包含 k 个不同字符：len(window) > k
    - 包含所有目标字符：不适用（需要反向判断）
    """
    return any(count > 1 for count in window.values())
```

### 关键配置/参数

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| `char_index` 初始值 | `-1`（数组）或不存在（字典） | 区分"未出现"和"出现位置 0" |
| `left` 初始值 | `0` | 从字符串开头开始 |
| `max_len` 初始值 | `0` 或 `min(1, len(s))` | 处理空字符串边界 |
| `窗口类型` | `[left, right]` 闭区间 | 更直观，长度 = right - left + 1 |

### 边界条件处理

```python
def length_of_longest_substring_edge_cases(s: str) -> int:
    """
    边界条件处理示例
    
    关键边界：
    1. 空字符串
    2. 单字符
    3. 全部相同字符
    4. 全部不同字符
    5. 最长子串在开头/中间/结尾
    """
    n = len(s)
    
    # 边界 1：空字符串
    if n == 0:
        return 0
    
    # 边界 2：单字符
    if n == 1:
        return 1
    
    # 正常处理
    char_index = {}
    left = 0
    max_len = 0
    
    for right in range(n):
        char = s[right]
        
        # 边界 5：更新左指针时确保单调性
        if char in char_index:
            # max 确保左指针不回退（关键！）
            left = max(left, char_index[char] + 1)
        
        char_index[char] = right
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用边界 |
|------|------|------|----------|
| **记录下标（跳跃）** | 时间复杂度严格 $O(n)$，左指针移动次数少 | 无法遍历所有合法窗口 | 只求最大长度/最优解 |
| **记录频次（逐步）** | 可遍历所有合法窗口，逻辑清晰 | 左指针移动次数多（均摊 $O(n)$） | 需要所有合法窗口/复杂条件 |
| **哈希表** | 通用性强，支持任意字符集 | 哈希开销，常数因子大 | Unicode/未知字符集 |
| **数组** | 访问快，无哈希冲突 | 空间固定，可能浪费 | 字符集小且连续（ASCII/小写字母） |
| **暴力枚举** | 实现简单，易于理解 | $O(n^2)$，大数据集超时 | n < 1000 或面试时间不够 |

**性能对比（n = 100000）**：
- 暴力枚举：$10^{10}$ 次操作，超时
- 滑窗（哈希表）：$2 \times 10^5$ 次操作，< 10ms
- 滑窗（数组）：$2 \times 10^5$ 次操作，< 5ms

## 高频追问（至少 5 个）

### 1. Q: 为什么时间复杂度是 $O(n)$ 而不是 $O(n^2)$？

**A**: 关键在于左右指针的单调性：
- **右指针**：从 0 到 n-1，每个位置只访问一次，共移动 n 次
- **左指针**：从 0 到 n-1，虽然可能跳跃，但从不回退，最多移动 n 次
- **总操作数**：$O(n + n) = O(2n) = O(n)$

**对比暴力**：暴力枚举需要两层循环，外层枚举起点（n 次），内层枚举终点（平均 n/2 次），总操作数 $O(n^2)$。

### 2. Q: 如果要求最长无重复子串的具体内容（而非长度），怎么做？

**A**: 记录最大窗口的左右端点：

```python
def longest_substring_content(s: str) -> str:
    n = len(s)
    if n == 0:
        return ""
    
    char_index = {}
    left = 0
    max_len = 0
    max_start = 0  # 记录最长子串的起点
    
    for right in range(n):
        char = s[right]
        
        if char in char_index:
            left = max(left, char_index[char] + 1)
        
        char_index[char] = right
        
        # 更新最大长度和起点
        if right - left + 1 > max_len:
            max_len = right - left + 1
            max_start = left
    
    return s[max_start:max_start + max_len]

# 示例
# 输入: "abcabcbb"
# 输出: "abc"（而非长度 3）
```

### 4. Q: 如果要求"最多包含 k 个不同字符的最长子串"，如何修改？

**A**: 修改条件判断逻辑：

```python
def longest_substring_k_distinct(s: str, k: int) -> int:
    """
    最多包含 k 个不同字符的最长子串
    
    关键修改：
    - window 记录字符频次
    - 条件：len(window) > k 时收缩
    """
    n = len(s)
    if n == 0 or k == 0:
        return 0
    
    window = {}  # {字符: 频次}
    left = 0
    max_len = 0
    
    for right in range(n):
        char = s[right]
        window[char] = window.get(char, 0) + 1
        
        # 收缩直到字符种类 <= k
        while len(window) > k:
            char = s[left]
            window[char] -= 1
            if window[char] == 0:
                del window[char]
            left += 1
        
        max_len = max(max_len, right - left + 1)
    
    return max_len

# 示例
# 输入: s = "eceba", k = 2
# 输出: 3（"ece"）
```

### 5. Q: 滑窗问题如何判断该用"记录下标"还是"记录频次"？

**A**: 
| 场景 | 选择方案 | 原因 |
|------|---------|------|
| 只求最大长度/最优解 | 记录下标 + 跳跃移动 | 效率最高，左指针移动次数少 |
| 需要遍历所有合法窗口 | 记录频次 + 逐步移动 | 可以访问每个合法状态 |
| 条件复杂（如包含所有目标字符） | 记录频次 | 需要精确知道窗口内元素状态 |
| 条件简单（如无重复） | 记录下标 | 可以快速定位重复位置 |

**经验法则**：
- 能跳跃就跳跃（记录下标）
- 需要精确控制窗口状态时用频次

### 6. Q: 为什么 `left = max(left, char_index[char] + 1)` 要用 max？

**A**: 防止左指针回退！

**场景**：字符串 "abba"
```
right=0, char='a': left=0, char_index={'a':0}
right=1, char='b': left=0, char_index={'a':0, 'b':1}
right=2, char='b': 重复！
  - char_index['b']=1
  - 不用 max: left = 1 + 1 = 2  ✓
right=3, char='a': 重复！
  - char_index['a']=0
  - 不用 max: left = 0 + 1 = 1  ✗ 回退了！
  - 用 max: left = max(2, 0+1) = 2  ✓
```

**结论**：max 确保左指针单调不减，保证时间复杂度 $O(n)$。

## 常见错误（至少 3 个）

### 1. **错误：左指针回退**

**描述**：`left = char_index[char] + 1` 没有用 max，导致左指针可能向左移动。

**场景**：字符串 "abba"，处理到第二个 'a' 时，`left` 会从 2 回退到 1。

**正确做法**：
```python
# 错误
left = char_index[char] + 1  # 可能回退

# 正确
left = max(left, char_index[char] + 1)  # 确保单调不减
```

### 2. **错误：忘记判断字符是否在窗口内**

**描述**：`if char in char_index` 判断字符是否出现过，但没有判断是否在**当前窗口**内。

**场景**：字符串 "tmmzuxt"，处理到第二个 't' 时，第一个 't' 已经不在窗口内。

**正确做法**：
```python
# 错误：只判断是否出现过
if char in char_index:
    left = char_index[char] + 1

# 正确：判断是否在当前窗口内
if char in char_index and char_index[char] >= left:
    left = char_index[char] + 1

# 或：直接用 max（推荐）
if char in char_index:
    left = max(left, char_index[char] + 1)
```

### 3. **错误：窗口长度计算错误**

**描述**：混淆闭区间和半开区间，导致长度计算错误。

**场景**：窗口 `[left, right]` 闭区间，长度应为 `right - left + 1`，但写成 `right - left`。

**正确做法**：
```python
# 闭区间 [left, right]
length = right - left + 1  # 正确

# 半开区间 [left, right)
length = right - left  # 正确（但需要调整循环）
```

**推荐**：统一使用闭区间，避免混淆。

### 4. **错误：忘记更新数据结构**

**描述**：更新左指针后，忘记更新哈希表/数组。

**场景**：记录频次的方案中，`left += 1` 后忘记 `char_count[s[left]] -= 1`。

**正确做法**：
```python
# 错误：只移动指针
while char_count[char] > 1:
    left += 1  # 忘记更新频次

# 正确：同时更新数据结构
while char_count[char] > 1:
    char_count[s[left]] -= 1
    left += 1
```

### 5. **错误：边界条件处理不当**

**描述**：空字符串、单字符等边界情况没有单独处理。

**场景**：`s = ""`，循环不执行，返回初始值 `max_len = 0`，但如果初始化为 `max_len = 1` 则错误。

**正确做法**：
```python
# 正确：初始化为 0
max_len = 0

# 或：单独处理边界
if n == 0:
    return 0
if n == 1:
    return 1
```

```python
def length_of_longest_substring(s: str) -> int:
    n = len(s)
    char_index = {}
    left = 0
    max_len = 0
    
    for right in range(n):
        char = s[right]
        if char in char_index:
            left = max(left, char_index[char] + 1)
        char_index[char] = right
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

```python
def length_of_longest_substring_ascii(s: str) -> int:
    n = len(s)
    char_index = [-1] * 128  # ASCII
    left = 0
    max_len = 0
    
    for right in range(n):
        char = ord(s[right])
        if char_index[char] >= left:

        char_index[char] = right
```

4. **最多包含 k 个不同字符的变种**

```python
def longest_substring_k_distinct(s: str, k: int) -> int:
    window = {}
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        window[s[right]] = window.get(s[right], 0) + 1
        
        while len(window) > k:
            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1
        
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

### 复杂度推导

**手写时间复杂度证明**：
```
设字符串长度为 n

右指针移动：
- 从 0 到 n-1，每个位置访问 1 次
- 总移动次数：n 次

左指针移动：
- 从 0 到 n-1，从不回退
- 每次移动至少 1 位
- 最大总移动次数：n 次

总操作数：O(n + n) = O(2n) = O(n)

空间复杂度：
- 哈希表/数组存储字符集
- 最坏情况：所有字符都不同
- 空间：O(min(n, m))，m 为字符集大小
```
