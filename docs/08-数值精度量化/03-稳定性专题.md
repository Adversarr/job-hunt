# 稳定性专题（训练中的数值稳定性问题）

## 一句话结论
训练不稳定主要源于数值溢出/下溢、梯度爆炸/消失、异常数据，排查顺序：先数值（softmax/layernorm/loss scaling）→ 再数据（异常样本）→ 最后模型（架构/初始化），关键保障是混合精度策略、梯度裁剪和稳定算子实现。

## 核心定义/公式

### 1. 数值稳定性问题分类
| 问题类型 | 表现 | 根因 | 典型位置 |
|----------|------|------|----------|
| Overflow | 值变 inf | 数值 > 类型上限 | softmax、梯度、attention score |
| Underflow | 值变 0 | 数值 < 类型下限 | softmax、小梯度、长序列累加 |
| NaN | 非数 | 0/0、inf/inf、log(0) | 除法、log、sqrt(负数) |
| 梯度爆炸 | 梯度极大 | 深层网络、异常数据 | 反向传播、RNN |
| 梯度消失 | 梯度极小 | 深层网络、sigmoid | 反向传播 |

### 2. Softmax 稳定性（Max-Trick）

**不稳定的 softmax 实现**：
```python
def softmax_unstable(x):
    return torch.exp(x) / torch.sum(torch.exp(x), dim=-1, keepdim=True)
    # 问题：exp(大数) → inf，导致 NaN
```

**稳定的 softmax 实现（max-trick）**：
```python
def softmax_stable(x):
    x_max = torch.max(x, dim=-1, keepdim=True).values
    exp_x = torch.exp(x - x_max)  # 减去最大值，防止溢出
    return exp_x / torch.sum(exp_x, dim=-1, keepdim=True)
```

**数学证明**：
```
softmax(x_i) = exp(x_i) / Σ exp(x_j)
             = exp(x_i - max) / Σ exp(x_j - max)  # 分子分母同除 exp(max)
```

### 3. LayerNorm/RMSNorm 数值稳定性

**LayerNorm 稳定实现**：
```python
def layer_norm_stable(x, weight, bias, eps=1e-5):
    mean = x.mean(dim=-1, keepdim=True)
    # 使用 var 而非直接 (x-mean)^2，避免精度损失
    var = x.var(dim=-1, keepdim=True, unbiased=False)
    x_norm = (x - mean) / torch.sqrt(var + eps)  # eps 防止除零
    return weight * x_norm + bias
```

**RMSNorm（更简洁稳定）**：
```python
def rms_norm(x, weight, eps=1e-6):
    rms = torch.sqrt(torch.mean(x ** 2, dim=-1, keepdim=True) + eps)
    return weight * (x / rms)
```

### 4. 混合精度训练稳定性保障

**PyTorch AMP 完整实现**：
```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler(
    init_scale=2**16,       # 初始缩放因子
    growth_factor=2.0,      # 无溢出时放大
    backoff_factor=0.5,     # 溢出时缩小
    growth_interval=2000,   # 检查间隔
    enabled=True
)

for batch in dataloader:
    optimizer.zero_grad()
    
    with autocast(dtype=torch.float16):  # 或 torch.bfloat16
        loss = model(batch)
    
    # 梯度缩放（FP16 必需，BF16 可选）
    scaler.scale(loss).backward()
    
    # 梯度裁剪（防止爆炸）
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    
    # 更新
    scaler.step(optimizer)
    scaler.update()
```

### 5. 梯度裁剪公式

**按范数裁剪（最常用）**：
```python
# 总梯度范数
total_norm = torch.nn.utils.clip_grad_norm_(
    model.parameters(), 
    max_norm=1.0,      # 最大范数
    norm_type=2.0      # L2 范数
)
# 实际执行：grad = grad * min(1, max_norm / ||grad||)
```

**按值裁剪**：
```python
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=1.0)
# 将梯度限制在 [-clip_value, clip_value]
```

## 为什么（2-3 个因果链）

### 1. 为什么 softmax 容易溢出
```
大 logits → exp(logits) 极大 → 超过 FP16 上限(65504) → inf
        → inf / inf 或 inf / 0 → NaN
        → 反向传播全链 NaN → 训练崩溃
```

**根因**：指数函数增长极快，exp(100) ≈ 2.7e43 已远超 FP16 范围。

### 2. 为什么 FP16 训练需要 loss scaling
```
小梯度（< 6e-8）→ FP16 underflow → 梯度变 0
            → 参数不更新 → 训练停滞
            → loss scaling 放大梯度 → 保持在 FP16 表示范围
```

**根因**：FP16 最小正常数约 6e-8，而深度网络梯度常小于此值。

### 3. 为什么深层网络容易梯度消失/爆炸
```
链式法则：∂L/∂W = ∂L/∂h_n × ∂h_n/∂h_{n-1} × ... × ∂h_1/∂W
每层梯度因子 < 1 → 连乘趋近于 0 → 梯度消失
每层梯度因子 > 1 → 连乘指数增长 → 梯度爆炸
```

## 怎么做（可落地步骤）

### NaN/Inf 排查流程

```python
def diagnose_nan_inf(model, batch):
    """NaN/Inf 系统排查流程"""
    
    # 1. 检查输入
    if torch.isnan(batch['input']).any() or torch.isinf(batch['input']).any():
        print("❌ 输入数据包含 NaN/Inf")
        return
    
    # 2. 逐层检查前向传播
    def check_activation_hook(module, input, output):
        if torch.isnan(output).any():
            print(f"❌ {module.__class__.__name__} 输出 NaN")
        if torch.isinf(output).any():
            print(f"❌ {module.__class__.__name__} 输出 Inf")
    
    hooks = []
    for name, module in model.named_modules():
        hooks.append(module.register_forward_hook(check_activation_hook))
    
    # 3. 执行前向传播
    with torch.no_grad():
        output = model(batch['input'])
    
    # 4. 清理 hooks
    for h in hooks:
        h.remove()
    
    # 5. 检查 loss
    loss = compute_loss(output, batch['label'])
    if torch.isnan(loss) or torch.isinf(loss):
        print("❌ Loss 为 NaN/Inf，检查：")
        print("  - label 是否正确")
        print("  - softmax/layernorm 实现")
        print("  - loss scaling 参数")

def diagnose_gradient(model):
    """梯度诊断"""
    for name, param in model.named_parameters():
        if param.grad is not None:
            if torch.isnan(param.grad).any():
                print(f"❌ {name} 梯度 NaN")
            if torch.isinf(param.grad).any():
                print(f"❌ {name} 梯度 Inf")
            
            grad_norm = param.grad.norm().item()
            if grad_norm > 100:
                print(f"⚠️ {name} 梯度过大: {grad_norm:.2f}")
            if grad_norm < 1e-8:
                print(f"⚠️ {name} 梯度过小: {grad_norm:.2e}")
```

### 异常数据检测

```python
def detect_anomalous_samples(dataset, tokenizer, max_length=2048):
    """检测异常样本"""
    anomalies = []
    
    for idx, sample in enumerate(dataset):
        issues = []
        
        # 1. 检查文本长度
        tokens = tokenizer.encode(sample['text'])
        if len(tokens) > max_length:
            issues.append(f"过长序列: {len(tokens)} tokens")
        
        # 2. 检查特殊字符
        if '\x00' in sample['text']:
            issues.append("包含 null 字符")
        
        # 3. 检查重复
        words = sample['text'].split()
        if len(words) > 0:
            repeat_ratio = len(set(words)) / len(words)
            if repeat_ratio < 0.3:
                issues.append(f"重复度过高: {1-repeat_ratio:.1%}")
        
        # 4. 检查标签
        if 'label' in sample:
            if sample['label'] not in valid_labels:
                issues.append(f"无效标签: {sample['label']}")
        
        if issues:
            anomalies.append({
                'index': idx,
                'issues': issues,
                'sample': sample['text'][:200]
            })
    
    return anomalies
```

## 权衡分析

| 方案 | 收益 | 代价 | 适用场景 |
|------|------|------|----------|
| BF16 混合精度 | 无需 loss scaling、训练稳定 | 硬件要求新（Ampere+） | 大模型训练首选 |
| FP16 + Loss Scaling | 硬件兼容性好 | 需调参、可能不稳定 | 旧 GPU（Volta/Turing） |
| FP32 全精度 | 数值最稳定 | 显存大、速度慢 | Debug、小模型 |
| 梯度裁剪 | 防止梯度爆炸 | 可能限制学习 | RNN、深层网络 |
| Warmup | 防止初期不稳定 | 增加调参复杂度 | 大 batch、大模型 |

## 高频追问（至少 5 个）

### Q1: Softmax 为什么需要 max-trick？
A: 防止 exp(大数) 溢出。减去最大值后，exp(x-max) ≤ exp(0) = 1，永远不会溢出。数学上等价，因为分子分母同除 exp(max)。

### Q2: FP16 和 BF16 哪个更稳定？为什么？
A: BF16 更稳定。BF16 动态范围与 FP32 相同（8位指数），不易溢出；FP16 动态范围小（5位指数），梯度容易 underflow/overflow，需要 loss scaling。

### Q3: 出现 NaN 后，应该按什么顺序排查？
A: 顺序：
1. **先检查数值**：softmax 实现、layernorm eps、loss scaling 参数
2. **再检查数据**：输入是否含 NaN、label 是否正确、异常样本
3. **最后检查模型**：架构设计、初始化方法、并行配置
4. **检查并行**：分布式训练时梯度同步是否正确

### Q4: 为什么梯度裁剪能提高稳定性？
A: 梯度裁剪限制单步更新的最大幅度，防止：
- 异常数据导致的梯度爆炸
- 深层网络的梯度累积效应
- 大学习率导致的参数剧烈变化
代价是可能限制模型的学习能力。

### Q5: LayerNorm 和 RMSNorm 哪个更稳定？
A: 实践中都稳定。RMSNorm 更简单（无 mean 中心化），计算量略小；LayerNorm 有 mean 中心化，理论上对分布变化更鲁棒。大模型（LLaMA、Qwen）普遍采用 RMSNorm。

### Q6: Loss 震荡一定是数值问题吗？
A: 不一定。Loss 震荡可能原因：
1. **数值问题**：梯度爆炸、NaN/Inf
2. **数据问题**：batch size 太小、数据分布不均、异常样本
3. **优化问题**：学习率过大、缺少 warmup
4. **架构问题**：深层网络、初始化不当
需要综合诊断。

### Q7: 如何判断是数据问题还是模型问题导致的训练不稳定？
A: 方法：
1. **换数据**：用合成数据/小数据集测试，稳定则数据问题
2. **换模型**：用简单模型（如 2 层 MLP）测试，稳定则模型复杂度过高
3. **检查梯度**：梯度异常大/小可能是模型问题
4. **检查 loss 曲线**：突然跳跃可能是异常数据

## 常见错误（至少 3 个）

### 错误 1：忘记在 softmax 中减去最大值
```python
# ❌ 错误
def softmax_bad(x):
    return torch.exp(x) / torch.sum(torch.exp(x), dim=-1, keepdim=True)

# ✅ 正确
def softmax_good(x):
    x_max = x.max(dim=-1, keepdim=True).values
    return torch.exp(x - x_max) / torch.exp(x - x_max).sum(dim=-1, keepdim=True)

# 实际使用 PyTorch 内置
torch.nn.functional.softmax(x, dim=-1)  # 已实现 max-trick
```

### 错误 2：FP16 训练忘记梯度缩放
```python
# ❌ 错误：直接用 FP16
with autocast(dtype=torch.float16):
    loss = model(input)
loss.backward()  # 小梯度会 underflow

# ✅ 正确：使用 GradScaler
scaler = GradScaler()
with autocast(dtype=torch.float16):
    loss = model(input)
scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
```

### 错误 3：梯度裁剪在 scaler.step 之后调用
```python
# ❌ 错误顺序
scaler.step(optimizer)
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)  # 已经更新了！

# ✅ 正确顺序
scaler.unscale_(optimizer)  # 先反缩放梯度
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)  # 再裁剪
scaler.step(optimizer)  # 最后更新
```

### 错误 4：LayerNorm 的 eps 设置太小
```python
# ❌ 可能不稳定
LayerNorm(hidden_size, eps=1e-12)  # 某些框架实现可能精度不够

# ✅ 推荐
LayerNorm(hidden_size, eps=1e-5)  # 或 1e-6
RMSNorm(hidden_size, eps=1e-6)
```
